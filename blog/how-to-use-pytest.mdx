> **Tóm tắt nhanh**
> - Pytest cho phép dùng `assert` statement thay vì special assertion methods
> - Fixtures cung cấp test data và setup/cleanup, có thể share giữa các tests
> - `@pytest.mark.parametrize` chạy cùng test với nhiều input values khác nhau
> - Exception testing: `with pytest.raises(ExpectedException)` — verify code raise đúng exception

## Installation

```bash
pip install pytest
```

Pytest là test framework phổ biến nhất cho Python — đơn giản, powerful, và có huge ecosystem của plugins.

## Assertions

Pytest dùng Python's standard `assert` statement thay vì special assertion methods:

```python
assert actual == expected
assert actual != expected
assert value in collection
assert value not in collection
```

Theo Python documentation, asserts được optimize away khi run với `-O` flag (optimize mode) — production deployments thường dùng exception-based validation thay vì asserts ([Python Docs](https://docs.python.org/3/reference/simple_stmts.html#assert), 2024).

## Basic Test

```python
def test_name():
    assert True
```

Pytest tự động discover tests với prefix `test_`. Chạy với `pytest` command.

## Exception Testing

Verify code raise đúng exception:

```python
import pytest

def test_zero_division():
    with pytest.raises(ZeroDivisionError):
        1 / 0
```

## Fixtures

Fixtures cung cấp test data và setup/cleanup. Definition:

```python
import pytest

@pytest.fixture
def user():
    return {
        'name': 'John Snow',
        'email': 'john@snow.west'
    }
```

Usage trong test function:

```python
def test_do_something(user):
    assert user['name'] == 'John Snow'
```

### Cleanup với yield

```python
@pytest.fixture
def database():
    db = Database()
    yield db  # Test runs here, db is available
    db.cleanup()  # Cleanup after test
```

### Fixture Scopes

| Scope | When recreated |
|-------|----------------|
| `function` | Mỗi test function (default) |
| `class` | Một lần cho mỗi test class |
| `module` | Một lần cho mỗi module |
| `package` | Một lần cho mỗi package |
| `session` | Một lần cho toàn bộ test session |

```python
@pytest.fixture(scope='module')
def db_connection():
    # Kết nối một lần, reuse cho tất cả tests trong module
    connection = create_connection()
    yield connection
    connection.close()
```

## Data Parameterization

Chạy cùng test với nhiều input values:

```python
import pytest

def is_even(number):
    return number % 2 == 0

@pytest.mark.parametrize("input,expected", [
    (2, True),
    (3, False),
    (11, False),
])
def test_is_even(input, expected):
    assert is_even(input) == expected
```

Output:

```
test_sample.py::test_is_even[2-True] PASSED
test_sample.py::test_is_even[3-False] PASSED
test_sample.py::test_is_even[11-False] PASSED
```

## FAQ

### Assert vs exception-based validation?

Dùng assert cho test assertions, exception cho production validation. Asserts có thể bị disable với `-O` flag, exceptions thì không.

### Fixtures vs conftest.py?

Fixtures defined trong conftest.py auto-import được trong tất cả tests trong cùng directory và subdirectories. Inline fixtures chỉ trong file đó.

### Mocking với pytest?

Dùng `pytest-mock` plugin hoặc `unittest.mock`:

```python
from unittest.mock import patch

@patch('module.function')
def test_with_mock(mock_func):
    mock_func.return_value = 42
    assert module.consumer() == 42
```

### Best practice cho test organization?

1. Test files: `test_*.py` hoặc `*_test.py`
2. Test functions: `test_*` prefix
3. Test classes: `Test*` prefix
4. Keep tests independent — không rely vào execution order
5. Use descriptive names: `test_user_login_success` thay vì `test1`
