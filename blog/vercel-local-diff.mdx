# Why Does My App Work on Vercel But Not Locally?

If your code runs fine on Vercel but breaks on your machine, you're not alone. This is one of the most common developer frustrations. The good news is that each cause has a clear fix.

The issues usually boil down to networking, DNS, firewall rules, or environment differences. Let's walk through each one.

## Key Takeaways

- Vercel uses IPv4 by default; your local machine might prefer IPv6
- Local firewalls and VPNs often block external API requests
- DNS resolution is faster and more reliable on Vercel
- Environment differences between local and production can cause issues
- Rate limiting might block your local IP but not Vercel's
- Node.js version mismatches can cause subtle problems

---

## IPv6 Networking

**The problem:** Your local machine tries to use IPv6 for network requests, but the service you're calling only works properly over IPv4. This causes timeouts.

**The fix:** Force your system to use IPv4.

```bash
sudo sysctl -w net.ipv6.conf.all.disable_ipv6=1
```

This is a temporary change. To make it permanent, add it to your sysctl config.

---

## Local Firewall, VPN, or Proxy

**The problem:** Your firewall, VPN, or proxy might be blocking outgoing requests to external services. This is especially common with corporate networks.

**The fix:** Test on a different network, like a mobile hotspot, to see if the issue is network-specific. Check your firewall rules and VPN settings. Temporarily disable them one at a time to isolate the culprit.

---

## DNS Resolution

**The problem:** Vercel uses a highly optimized DNS setup that resolves domains quickly. Your local machine might have slower DNS resolvers, causing delays or failures when looking up addresses.

**The fix:** Switch to a faster public DNS resolver.

Update your `/etc/resolv.conf`:

```
nameserver 8.8.8.8
nameserver 8.8.4.4
```

For Google DNS. Cloudflare is also a solid option:
- Cloudflare: `1.1.1.1` and `1.0.0.1`

This change takes effect immediately and usually fixes DNS-related timeouts.

---

## Environment Differences

**The problem:** Vercel runs on Linux servers with a specific Node.js setup. Your local environment might be MacOS or Windows with different networking behavior.

**The fix:** Use Docker to match Vercel's production environment locally.

```dockerfile
# Dockerfile
FROM node:16-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000
CMD ["npm", "run", "dev"]
```

This ensures your local development matches what runs on Vercel.

---

## Rate Limiting or IP Blocking

**The problem:** External APIs might block or throttle your local IP address, especially if you've made too many requests. Vercel's server IPs might be whitelisted or face fewer restrictions.

**The fix:** Add rate limiting to your API calls and consider using a proxy service.

```javascript
const axios = require('axios');
const rateLimit = require('axios-rate-limit');

const axiosInstance = rateLimit(axios.create(), { maxRequests: 2, perMilliseconds: 1000 });

axiosInstance.get('https://example.com/api')
  .then(response => console.log(response.data))
  .catch(error => console.error(error));
```

This prevents you from hitting rate limits that might get your IP blocked.

---

## Node.js Version Mismatch

**The problem:** Vercel uses a specific Node.js version. If your local version differs, subtle networking or DNS bugs can appear.

**The fix:** Match Vercel's Node.js version using `nvm`.

```bash
nvm install <version>
nvm use <version>
```

Check your Vercel dashboard to see which Node.js version is configured for your project.

## Frequently Asked Questions

**Why does my API work on Vercel but timeout locally?**
Your local machine might be on IPv6 while the API only responds properly over IPv4. Try disabling IPv6 or using Google's public DNS.

**How do I check if my VPN is blocking requests?**
Temporarily disable your VPN and test the request. If it works without the VPN, your VPN is the culprit.

**Can Docker really fix environment issues?**
Yes. Docker containers run the same Linux environment regardless of your host OS. This eliminates many "works on my machine" problems.

**Why would an API block my local IP but not Vercel's?**
APIs often track request patterns. High-volume traffic from Vercel's datacenter IPs might be expected, while requests from residential IPs trigger rate limiting or blocking.

**What if none of these fixes work?**
Check your local network's outbound rules. Corporate networks often have proxies that inspect traffic. Try testing from a different network entirely.
