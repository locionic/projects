# Redux: State Management Made Simple

Redux solves a real problem in React apps. When multiple components need access to the same data, things get messy fast.

Redux is a library that manages your application state in one central place. Instead of passing data through layers of components, you keep everything in a "store" that any component can access. State changes follow strict rules, so your app stays predictable.

When should you reach for Redux? If your app has lots of global state accessed by many components, Redux removes a lot of headaches. It scales well and keeps your data flow clean.

## Key Takeaways

- Redux provides a single source of truth for your app state
- State can only change through actions and reducers
- Redux Toolkit simplifies the original Redux setup
- The three core pieces are: Store, Actions, and Reducers

---

## The Three Core Pieces

Every Redux app revolves around three concepts. They work together in a one-way data flow.

### The Store

The store is where your application's global state lives. Think of it as a container that holds all the data your components need to share.

The store is the single source of truth for your app. It manages the state and notifies parts of your UI when something changes. You don't access the state directly. Instead, you use the store's methods to read and update it.

### Actions

An action is a plain JavaScript object. It describes what happened in your app, like "user clicked login" or "data loaded successfully."

Every action must have a `type` property with a string that describes the action. You can also add other properties, most commonly `payload`, which carries the data needed to complete the action.

Actions are the only way to trigger a state change in Redux.

### Reducers

A reducer is a function that takes the current state and an action, then calculates the new state. ItReads the action type and decides what to do.

Reducers are the only place where the store's state actually changes. They receive the current state and an action, then return the next state. This keeps everything predictable and easy to trace.

## Redux Toolkit Setup

Redux Toolkit makes setting up Redux much easier. It handles a lot of the boilerplate for you.

Install it with:

```
npm install @reduxjs/redux-toolkit
```

### Example: Login Slice

Here's how to create a slice for authentication:

```
import React, { useReducer, SyntheticEvent } from "react";
import { createSlice } from "@reduxjs/toolkit";

const authState = {
  isRequestingToken: "",
  username: "",
  token: "",
  error: ""
};

const authSlice = createSlice({
  name: "auth",
  reducers: {
    loginStart: (state, action) => {
      state.isRequestingToken = "yes";
      state.username = action.payload.username;
    },
    loginSuccess: (state, action) => {
      state.isRequestingToken = "no";
      state.token = action.payload.token;
    }
  },
  initialState: authState
});

const { loginStart, loginSuccess } = authSlice.actions;
const authReducer = authSlice.reducer;

export function SillyThings() {
  const [state, dispatch] = useReducer(authReducer, authState);

  function handleSubmit(event: SyntheticEvent) {
    event.preventDefault();
    const formData = new FormData(event.target as HTMLFormElement);
    const username = formData.get("username");

    dispatch(loginStart({ username }));

    fetch("https://api.valentinog.com/api/token-create/", {
      method: "POST",
      body: formData
    })
      .then(response => {
        if (!response.ok) throw Error(response.statusText);
        return response.json();
      })
      .then(json => dispatch(loginSuccess({ token: json.token })));
  }

  return state.token ? (
    <p>Welcome back {state.username}</p>
  ) : (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor="username"> Username</label>
        <input type="text" id="username" name="username" />
      </div>
      <div>
        <label htmlFor="password">Password</label>
        <input type="password" id="password" name="password" />
      </div>
      <button type="submit">LOGIN</button>
    </form>
  );
}
```

`createSlice` generates the actions and reducer for you. Define your initial state and the reducer functions, and it handles the rest.

## Frequently Asked Questions

**When should I use Redux instead of React's built-in state?**

Use Redux when you have state that many unrelated components need to access, or when state changes across many places in your app. For local component state, React useState or useReducer usually suffices.

**What's the difference between actions and reducers?**

Actions describe what happened (like "login clicked"), while reducers determine how the state changes in response. Actions are plain objects, reducers are functions.

**Why does Redux Toolkit matter?**

It removes boilerplate. You write less code to set up Redux, and it's the recommended approach now. It also includes utilities like createSlice that auto-generate actions and reducers.

**How does data flow in Redux?**

One-way flow: UI dispatches an action, the store runs the reducer with the current state and action, the reducer returns the new state, and the store notifies subscribers. Components then re-render with the new state.

**Is Redux only for React?**

No. Redux is a standalone library. It works with React but also integrates with other frameworks like Angular, Vue, or vanilla JavaScript.
