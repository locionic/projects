# Useful Python Stuff

Here's a collection of Python tricks I keep coming back to. Whether you're pulling data from Google Drive in Colab or setting up a specific Python version, these snippets save me time.

## Downloading Data from Google Drive in Colab

Want to grab files from a shared Google Drive link directly in Google Colab? Here's how to do it.

First, install gdown:

```
!pip uninstall gdown -y && pip install gdown
!gdown -V
```

### For Whole Shared Folders

Use the `--folder` flag to download an entire directory:

```
!gdown --folder https://drive.google.com/drive/folders/sdsldkfj...somefileid.. -O /some_parent_directory/some_child_directory
```

### For Individual Files

Download single files like this:

```
!gdown https://drive.google.com/drive/folders/sdsldkfj...somefileid.. -O /some_parent_directory/some_child_directory
```

## Key Takeaways

- Install `gdown` with `pip install gdown`
- Use `--folder` flag for directories, no flag for single files
- Alternatively, import gdown in Python for more control

## Downloading Python 3.9

Need Python 3.9 specifically? Install it on Ubuntu/Debian with:

```
sudo apt install python3.9 python3.9-venv python3.9-dev
```

### Fixing "Package Not Found" Errors

If you see "Couldn't find any package by glob 'python3.9'", update your packages first:

```
apt update
sudo apt install software-properties-common
sudo add-apt-repository ppa:deadsnakes/ppa
```

Then try the install again.

## Setting Up a Virtual Environment

Once Python 3.9 is installed, create a virtual environment:

```
mkdir venv
which python3.9
/usr/bin/python3.9 -m venv venv/
source venv/bin/activate
```

The `which python3.9` command shows you the exact path, so you know you're using the right version.

## Frequently Asked Questions

**How do I download a specific file from Google Drive in Colab?**

Use gdown with the file's shared link: `!gdown https://drive.google.com/file/d/FILE_ID -O /path/to/output`

**What's the difference between gdown Method 1 and Method 2?**

Method 1 uses shell commands (!gdown) while Method 2 imports gdown as a Python module. The module approach gives you more control in your notebook scripts.

**Can I download private files with gdown?**

gdown works best with shared/public files. For private files, you'll need to authenticate or use the Google Drive API directly.

**How do I switch between Python versions?**

Use `update-alternatives` or specify the full path like `/usr/bin/python3.9` when creating your virtual environment.

**Why use a virtual environment for Python 3.9?**

Isolating your project with a venv prevents dependency conflicts. Each project gets its own packages and version requirements.
