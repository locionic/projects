## Sự khác biệt giữa `id` và `class` trong HTML/CSS?

Một `id` là định danh duy nhất — chỉ một phần tử duy nhất được phép dùng. Một `class` có thể tái sử dụng trên nhiều phần tử cùng lúc.

Dùng `id` khi cần chọn một phần tử cụ thể qua CSS hoặc JavaScript. Dùng `class` khi muốn style một nhóm các phần tử DOM cùng lúc.

```css
#id { /* chọn đúng một phần tử có id này */ }
.class { /* chọn tất cả phần tử có class này */ }
```

## Mô hình hộp (box model) trong CSS là gì?

Mô hình hộp CSS mô tả các hình chữ nhật bao quanh mỗi phần tử DOM, gồm 4 lớp từ trong ra ngoài:

- **Content** — vùng chứa văn bản và hình ảnh
- **Padding** — khoảng đệm giữa content và đường viền
- **Border** — đường viền bao quanh phần tử
- **Margin** — khoảng cách bên ngoài border, tạo khoảng trống giữa các phần tử

Theo W3C, `box-sizing: border-box` giúp tính toán kích thước trực quan hơn vì padding và border được tính vào width/height thay vì cộng thêm ([W3C](https://www.w3.org/TR/css-box-3/), 2023).

## Sự khác biệt giữa `inline`, `inline-block` và `block`?

- **Inline** — không có width/height riêng, nằm trên cùng dòng với text. Ví dụ: `<span>`, `<a>`
- **inline-block** — giống inline nhưng đặt được width/height. Ví dụ: `<img>`
- **Block** — bắt đầu dòng mới, mặc định chiếm toàn bộ width. Ví dụ: `<div>`, `<p>`

## Thực hành SEO tốt nhất cho cấu trúc HTML?

### Về HTML:
- **Dùng semantic HTML:** `<header>`, `<nav>`, `<article>`, `<section>`, `<footer>`
- **Cấu trúc tiêu đề hợp lý:** một `<h1>` duy nhất mỗi trang, sau đó `<h2>`, `<h3>`...
- **Thẻ meta cần thiết:** description, keywords, viewport
- **Thuộc tính `alt` cho ảnh:** cải thiện accessibility và SEO

### Ngoài ra:
- **Liên kết nội bộ** — giúp search engine thu thập dữ liệu
- **Thiết kế mobile-first** — đảm bảo responsive trên mọi thiết bị
- **Tốc độ tải trang** — tối ưu ảnh, minify code, cache hiệu quả

## Document Object Model (DOM) là gì?

DOM là API đại diện cho cấu trúc trang HTML dưới dạng cây node. Mỗi node tương ứng một phần của document: thẻ HTML, thuộc tính, hoặc text node.

JavaScript dùng DOM API để đọc và thay đổi nội dung trang động — đây là cơ chế cốt lõi cho mọi ứng dụng web tương tác ([MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model), 2024).

## Sự khác biệt giữa `em` và `rem`?

- **`em`** — phụ thuộc font-size của phần tử cha gần nhất
- **`rem`** — phụ thuộc font-size của phần tử gốc (`<html>`)

Rem thường dễ quản lý hơn trong thiết kế có nhiều cấp lồng nhau vì không bị compounding như em.

## Cách tạo bố cục Flexbox?

1. Đặt `display: flex` trên phần tử cha
2. Định nghĩa thuộc tính flex cho các phần tử con

```css
.container {
  display: flex;
  justify-content: space-between;
}
.item {
  flex: 1;
}
```

Flexbox tối ưu cho navigation bar, card layout, và centering — nhưng khi cần lưới hai chiều phức tạp thì nên dùng CSS Grid.

## Tính đặc hiệu (specificity) trong CSS hoạt động thế nào?

CSS dùng specificity để phân giải xung đột khi nhiều rules cùng áp dụng. Thứ tự ưu tiên từ cao xuống thấp:

1. **Inline styles** — ưu tiên cao nhất
2. **ID selectors** (`#id`) — cao hơn class
3. **Class selectors** (`.class`) — cao hơn type selectors
4. **Type selectors** (`p`, `div`) — ưu tiên thấp nhất

## Closures trong JavaScript là gì?

Closure xảy ra khi một hàm đượng định nghĩa bên trong hàm khác — nó giữ quyền truy cập biến của hàm bên ngoài ngay cả khi hàm đó đã kết thúc.

```javascript
function counter() {
  let count = 0;  // private variable
  return function() {
    count++;
    return count;
  };
}
```

Closures tạo private state mà không cần class — pattern phổ biến trong module pattern và event handlers.

## Biến CSS là gì, khi nào nên dùng?

Biến CSS (`--variable-name`) lưu trữ giá trị tái sử dụng toàn file:

```css
:root {
  --main-color: #3498db;
  --spacing: 16px;
}
body {
  color: var(--main-color);
  padding: var(--spacing);
}
```

Dùng khi có giá trị lặp lại cần thay đổi tập trung — theme colors, spacing, breakpoints. Tránh hardcode giá trị lặp.

## SSR (Server-Side Rendering) là gì, khi nào cần?

SSR render HTML trên server thay vì trình duyệt. Cần dùng khi:

- **Nội dung nặng** — tin tức, blog, trang marketing
- **SEO quan trọng** — search engine crawl nội dung đã render sẵn
- **Initial load performance** — server mạnh hơn thiết bị client

Next.js, Nuxt.js, Angular Universal là các framework phổ biến cho SSR. Trade-off: complexity tăng, nhưng performance và SEO cải thiện rõ rệt.

## Tree Shaking là gì, giúp performance thế nào?

Tree shaking loại bỏ code JavaScript không sử dụng trong quá trình bundle. Kết quả: file nhỏ hơn, tải nhanh hơn.

Webpack và Vite thực hiện tree shaking tự động khi dùng ES6 modules (`import`/`export`). ES modules là static structure — build tool phân tích được imports thực sự cần thiết.

## FAQ

### Cần học những gì để pass phỏng vấn Frontend?

HTML/CSS/JavaScript vững là nền tảng. Sau đó học một framework như React hoặc Vue. Thực hành DOM manipulation, event handling, và async operations bằng vanilla JS trước khi dùng abstraction.

### Bao lâu để prepares cho frontend interview?

Với người đã có foundation vững: 2-4 tuần focus. Nếu mới bắt đầu: 2-3 tháng. Trọng tâm là hiểu sâu concepts, không chỉ học thuộc câu trả lời.

### Câu hỏi nào được hỏi thường xuyên nhất?

Variance theo công ty, nhưng các câu hỏi closures, hoisting, event bubbling, và API calls (fetch/axios) gần như luôn có mặt.
