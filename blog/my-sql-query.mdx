## Key Takeaways

- Connect with `mysql -u [user] -p [database]`
- Create databases and tables before storing anything
- Use `SELECT` with `WHERE` and `GROUP BY` to slice data
- CRUD: `INSERT`, `UPDATE`, `DELETE` for data changes
- `ALTER` modifies existing tables without rebuilding them

---

## Connecting to a MySQL Server

Log in from the terminal:

```bash
mysql -u [username] -p
mysql -u [username] -p [database]
mysqldump -u [username] -p [database] > data_backup.sql
quit
exit
help
```

Basic stuff. `-p` prompts for your password. Point it at a database name to connect directly.

## Creating and Displaying Databases

```sql
CREATE DATABASE zoo;
SHOW DATABASES;
USE zoo;
DROP DATABASE zoo;
SHOW TABLES;
DESCRIBE animal;
```

`USE` switches the active database. `DROP` deletes it—careful with that one.

## Creating Tables

Define your schema upfront:

```sql
CREATE TABLE habitat (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(64)
);
CREATE TABLE animal (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(64),
  species VARCHAR(64),
  age INT,
  habitat_id INT,
  FOREIGN KEY (habitat_id) REFERENCES habitat(id)
);
```

`AUTO_INCREMENT` gives each row a unique ID automatically. `FOREIGN KEY` links tables together.

## Modifying Tables

Need to change a table after creation? `ALTER` it:

```sql
ALTER TABLE animal RENAME pet;

ALTER TABLE animal
ADD COLUMN name VARCHAR(64);

ALTER TABLE animal
RENAME COLUMN id TO identifier;

ALTER TABLE animal
MODIFY COLUMN name VARCHAR(128);

ALTER TABLE animal
DROP COLUMN name;

DROP TABLE animal;
```

## Querying Data

Get data out with `SELECT`:

```sql
SELECT species, AVG(age) AS average_age
FROM animal
WHERE id != 3
GROUP BY species
HAVING AVG(age) > 3
ORDER BY AVG(age) DESC;
```

**Joins** pull data from multiple tables:

```sql
SELECT city.name, country.name
FROM city
[INNER | LEFT | RIGHT] JOIN country
  ON city.country_id = country.id;
```

- `INNER JOIN` — only rows that match both sides
- `LEFT JOIN` — all rows from the left table, matched or null
- `RIGHT JOIN` — all rows from the right table, matched or null

## Aggregation and Grouping

Count things:

```sql
SELECT COUNT(*)
FROM animal;

SELECT COUNT(name)
FROM animal;

SELECT COUNT(DISTINCT name)
FROM animal;
```

`COUNT(DISTINCT ...)` counts unique values only.

---

## Group By

Group rows to aggregate:

```sql
SELECT species, COUNT(id)
FROM animal
GROUP BY species;

SELECT habitat_id, AVG(age), MIN(age), MAX(age)
FROM animal
GROUP BY habitat;
```

`GROUP BY` bundles rows with the same value. Works with `COUNT`, `AVG`, `MIN`, `MAX`, and more.

## Inserting Data

Add rows:

```sql
INSERT INTO habitat VALUES (1, 'River'), (2, 'Forest');

INSERT INTO habitat (name) VALUES ('Savanna');
```

First form inserts all columns in order. Second form specifies column names explicitly.

---

## Updating Data

Change existing rows:

```sql
UPDATE animal
SET
  species = 'Duck',
  name = 'Quack'
WHERE id = 2;
```

**Always include `WHERE`.** Leave it out and you update every row in the table.

---

## Deleting Data

Remove rows:

```sql
DELETE FROM animal
WHERE id = 1;

TRUNCATE TABLE animal;
```

`DELETE` removes specific rows. `TRUNCATE` wipes the whole table—faster, but can't be undone.

## FAQ

**How do I back up a database?**

Run `mysqldump -u [user] -p [database] > backup.sql`. This exports everything to a file you can restore later with `mysql -u [user] -p [database] < backup.sql`.

**What's the difference between `DROP`, `DELETE`, and `TRUNCATE`?**

`DELETE` removes rows one by one and can use `WHERE`. `TRUNCATE` removes all rows at once. `DROP` deletes the whole table—structure and all.

**How do I add a primary key to an existing table?**

Use `ALTER TABLE animal ADD PRIMARY KEY (id);`. The column must have no duplicates and no nulls.

**Should I always use `WHERE` with `UPDATE` and `DELETE`?**

Yes. Except when you genuinely mean to change or remove every row. The typo that skips a `WHERE` clause is a very memorable mistake.

**What's a foreign key?**

A column that references the primary key of another table. In the example, `habitat_id` in `animal` points to `id` in `habitat`. It keeps your data consistent—if you try to insert a `habitat_id` that doesn't exist, MySQL rejects it.
