> **Tóm tắt nhanh**
> - `git config --get remote.origin.url` — xem remote URL repo hiện tại
> - `git checkout -b [branch]` — tạo branch mới và switch sang ngay
> - `git reset HEAD~1` — undo last commit, giữ changes staged
> - Token authentication: `git remote set-url origin https://token@github.com/user/repo.git`

## Các lệnh Git thường dùng

### Xem remote URL

```bash
git config --get remote.origin.url
```

Lệnh này output đường dẫn của origin remote — hữu ích khi cần verify repo hoặc thêm token.

### Set remote với token

```bash
git remote set-url origin https://token@github.com/username/repository.git
```

Dùng khi cần authenticate qua token thay vì SSH key. Thay `token` bằng GitHub personal access token của bạn.

### Tạo và switch branch

```bash
git checkout -b [name_of_your_new_branch]
```

Flag `-b` tạo branch mới và chuyển sang luôn, không cần chạy lệnh riêng cho mỗi bước.

### Push branch lên GitHub

```bash
git push origin [name_of_your_new_branch]
```

Sau khi push, GitHub sẽ hiển thị link để tạo Pull Request nếu muốn.

### Undo last commit

```bash
git reset HEAD~1   # Giữ changes staged
rm -rf myfolder    # Xóa folder không muốn
git add .          # Stage changes mới
git commit -m "Removed myfolder"
```

`--soft` flag giữ changes trong staging area, `--hard` xóa hoàn toàn. Dùng cẩn thận với `--hard` — data có thể mất vĩnh viễn.

## FAQ

### Làm sao xem lịch sử commit?

```bash
git log --oneline -10
```

Hiển thị 10 commit gần nhất với hash ngắn và message.

### Hủy Stash đã lưu?

```bash
git stash drop stash@{0}
```

### Tạo branch từ commit cũ?

```bash
git checkout -b new-branch 8a3c7d9
```

Dùng commit hash để tạo branch tại thời điểm đó.

### Cập nhật branch với latest main?

```bash
git fetch origin
git merge origin/main
```

Hoặc rebase để lịch sử gọn hơn: `git rebase origin/main`.
