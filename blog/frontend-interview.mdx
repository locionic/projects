# Frontend Interview Questions

Preparing for frontend interviews can feel overwhelming. Whether you're just starting out or aiming for a senior role, knowing what to expect makes all the difference. Based on actual interview patterns from top tech companies, these questions cover the concepts you'll likely face.

From HTML fundamentals to advanced topics like virtual DOM and server-side rendering, this guide walks through each concept clearly. Let's get you ready to impress.

> **Key Takeaways**
> - IDs are unique identifiers while classes can be reused across multiple elements
> - The CSS box model consists of content, padding, border, and margin layers
> - Flexbox and Grid are essential for modern layout systems
> - Closures give inner functions access to outer scope variables even after execution
> - The event loop enables JavaScript's non-blocking asynchronous operations

# Beginner

## What is the difference between an id and a class in HTML/CSS?

An id is a unique identifier for a single HTML element, while a class is a reusable identifier you can apply to multiple elements.

You'd use an id when you need to target one specific element through CSS or JavaScript, and a class when you need to target a group of DOM elements.

In CSS, `#id` selects a specific element while `.class` selects all elements with that class.

![id identifier](https://assets.roadmap.sh/guest/css-classes-vs-id-example-9ufsl.png)

## Can you explain the box model in CSS?

The CSS box model describes the rectangular boxes generated for elements in the DOM. It consists of four layers.

Content is the innermost part where text and images appear. Padding creates space between the content and border. The border surrounds the element, and margin separates it from neighboring elements.

By controlling each layer individually, you can precisely define how each element looks in your UI.

## What is the difference between inline, inline-block, and block elements?

CSS handles these elements differently based on how they flow in the layout.

Inline elements don't start on a new line and only take up as much width as their content requires. You can't set width or height on them. Think span and anchor tags.

Inline-block elements also don't start a new line, but you can set width and height on them. Images work this way.

Block elements always start on a new line and take full width by default. Divs and paragraphs are classic examples.

## What are some SEO best practices when structuring an HTML document?

Semantic HTML goes a long way here. Use proper tags like `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, and `<footer>` to give your document meaningful structure.

Also keep heading hierarchy correct, use your `<h1>` once per page, and always include relevant meta tags for description, keywords, and viewport.

Images need alt attributes for accessibility and search engine understanding. Internal links help crawlers navigate your site, and a mobile-friendly responsive design improves both user experience and rankings.

## What is the Document Object Model (DOM)?

[INTERNAL-LINK: JavaScript DOM manipulation → Working with the DOM]

The DOM is an API for web documents. It represents your HTML page as a tree of nodes, where each node corresponds to a document part: an element, an attribute, or text.

This tree structure lets JavaScript dynamically access and modify page content.

## How do you add an event listener to an element?

First, you grab the element using a document method like `getElementById`. Then you call `addEventListener` on that element.

The method takes the event name (like 'click', 'keyup', or 'mouseup'), a handler function, and optionally a boolean for the capture phase.

Simple enough once you do it a couple times.

## What is the difference between null and undefined?

In JavaScript, undefined is the default value for new variables. It means the variable exists but hasn't been assigned anything yet.

Null is different. You'd assign it explicitly to signal "no value" or "no object." It's intentional, not a default state.

## What are the storage options available in browsers?

[INTERNAL-LINK: LocalStorage and sessionStorage → Browser storage guide]

Cookies store small pieces of data primarily for HTTP requests, like authentication tokens and session information.

sessionStorage holds temporary data that disappears when you close the browser tab or window.

localStorage persists even after closing and reopening the browser. It handles larger datasets than cookies, making it better for long-term storage needs.

## How does the browser render a website?

The browser follows several steps to display your page.

It parses HTML first, then CSS and applies styles. Next comes layout calculation to position each element, followed by painting pixels and sorting them into layers.

The browser composes all layers together, considering z-index and opacity values. Then JavaScript runs, and finally async resources load.

## What are media queries?

Media queries let you apply different styles based on device characteristics. You can target width, height, orientation, or device type.

Through media queries, you build responsive designs that adapt to different screen sizes and capabilities. Mobile-first approach works best here.

# Intermediate

## What is the difference between em and rem units?

Both are relative units, but they reference different things.

"em" units scale relative to the parent element's font size. If the parent is 20px and you set 2em, you get 40px.

"rem" units are "root em" and scale relative to the root `<html>` element instead. This makes them more predictable in complex layouts.

## How do you create a flexbox layout?

[INTERNAL-LINK: Flexbox concepts → Complete flexbox guide]

Two steps. First, set `display: flexbox` on your container element. Then configure properties on the children.

Properties like `flex-direction`, `justify-content`, and `align-items` control how items flow and align. A simple `flex: 1` on children makes them grow equally.

## Can you explain CSS specificity and how it works?

When multiple styles target the same element, specificity determines which one wins.

Inline styles have highest priority. Then ID-based selectors. Then class-based selectors. Finally, type selectors have the lowest precedence.

Knowing this helps you debug unexpected styling and write maintainable CSS.

## How do you create a CSS grid layout?

[INTERNAL-LINK: CSS Grid → Grid layout tutorial]

Set `display: grid` on your container element. Then define the structure using `grid-template-rows` and `grid-template-columns`.

Place children inside and control their position with `grid-column` and `grid-row` properties. It's incredibly powerful for two-dimensional layouts.

## What are closures, and why would you use them?

A closure forms when an inner function retains access to its outer function's variables, even after that outer function finishes executing.

This connection between the inner function and its scope is the closure. You get private variables accessible only through the inner function, making it great for encapsulation.

[INTERNAL-LINK: JavaScript closures → Understanding closures]

## Can you explain what event delegation is in JavaScript?

Instead of adding listeners to individual elements, you attach one listener to a parent element.

When events bubble up through the DOM, the parent catches them. This way, you handle events for all children through one listener.

It's efficient and works great for dynamic content.

![event delegation](https://assets.roadmap.sh/guest/event-delegation-explained-5e2vt.png)

## What are promises, and how do they work?

Promises represent the eventual completion of an async operation. They handle success or failure without blocking other code.

You chain `.then()` for success and `.catch()` for errors, making async code more manageable than nested callbacks.

## How do you optimize website assets for better loading times?

Minify and compress your CSS and JavaScript files. For images, compress them during save or use tools like JPEGOptim, but watch quality.

Efficient code, optimized assets, and proper caching all contribute. The goal is getting content to users as fast as possible.

## What are service workers, and what are they used for?

Service workers run in the background, separate from your main page thread. They enable offline caching, push notifications, and background sync.

Think of them as a proxy between your app and the network, giving you control over caching strategies.

[INTERNAL-LINK: Service workers → PWA implementation]

## What is the same-origin policy in web development?

This security feature prevents websites from accessing data from other origins. It blocks scripts from making requests to different domains.

This protects users from malicious scripts trying to steal cookies, local storage, or content from other sites.

You can work around this with CORS, as long as the server specifies which domains it accepts requests from and the client sends proper headers.

# Advanced

## What are CSS variables, and when would you use them?

CSS variables let you store values and reuse them throughout your stylesheets. They're great for centralizing global values.

Design systems use them heavily. Instead of hardcoding "#222" everywhere, you define it once as a variable. Change it in one place, and it updates everywhere.

[INTERNAL-LINK: CSS variables → Variable usage guide]

## How would you implement critical CSS to improve perceived load time?

Extract the CSS needed for above-the-fold content and inline it directly in your `<head>`. This eliminates a render-blocking request.

The critical CSS loads immediately, and non-critical styles load later through your main CSS file. Users see content faster, which feels snappier.

## How does the event loop work in JavaScript?

[INTERNAL-LINK: Event loop → JavaScript concurrency model]

JavaScript runs on a single thread using a call stack where functions execute one at a time.

Async operations use browser Web APIs and get offloaded. When they're done, their callbacks go into a task queue.

The event loop constantly checks if the call stack is empty. When it is, it grabs the next task from the queue and runs it. This cycle keeps going.

![event loop javascript](https://assets.roadmap.sh/guest/javascript-event-loop-explained-d92hx.png)

## What are the different ways to handle asynchronous operations in JavaScript?

Callbacks let you pass a function to run once an async operation completes. They work but can nest badly.

Promises improved things by letting you chain `.then()` calls and handle errors more cleanly.

Async/await is syntactical sugar over promises but makes async code read synchronously, which is much easier to follow.

Event listeners handle callbacks triggered by user interactions or system events.

## How do you handle state management in single-page applications?

Without frameworks, it's challenging but doable. Global variables can centralize state but get unmanageable fast in larger apps.

The module pattern encapsulates state with a clear API. The pub/sub pattern decouples state changes through events, giving you more flexibility.

For serious apps, libraries like Redux handle this complexity elegantly.

[INTERNAL-LINK: State management → Frontend state patterns]

## How does virtual DOM work, and what are its advantages?

The virtual DOM is an in-memory copy of your actual DOM. When state changes, a new virtual DOM gets created.

The system diffs the new version against the previous one and calculates the minimum changes needed. Only the affected nodes update in the real DOM.

This approach reduces direct DOM manipulation, which is slow. It also provides an abstraction layer enabling cross-platform rendering. Plus, it keeps your UI in sync with internal state, reducing bugs.

## What is server-side rendering, and when might you use it?

SSR generates the complete HTML on the server before sending it to the browser. This contrasts with client-side rendering where the browser downloads a minimal HTML and builds everything with JavaScript.

Content-heavy sites like news outlets and blogs benefit greatly. SEO-focused applications see improved performance since search engines crawl rendered content easily.

Progressive web apps often start with SSR for fast initial loads, then hydrate on the client for interactive experiences.

[INTERNAL-LINK: SSR vs CSR → Server-side rendering guide]

## How do you analyze and improve the performance of a web application?

Monitor Core Web Vitals. First Contentful Paint tracks when the first content appears. Largest Contentful Paint measures the main content load time.

Time to Interactive shows when the page becomes usable. Total Blocking Time sums up moments when the main thread slows down. Cumulative Layout Shift measures unexpected visual shifts.

Tools like Lighthouse and Chrome DevTools help you track and improve these metrics.

## What is Content Security Policy, and how does it improve security?

CSP is a security header that prevents cross-site scripting and code injection attacks. You define which sources are allowed for scripts, styles, images, and other resources.

It gives you fine-grained control over what content can load from where. This helps meet security compliance requirements too.

[INTERNAL-LINK: Web security → Frontend security best practices]

## What is tree shaking, and how does it help performance?

Tree shaking removes unused code from your final bundle. Bundlers like Webpack and Vite analyze your imports and strip out anything not actually called.

Smaller bundles mean faster load times and less bandwidth used. You write modular code freely without worrying about bloating the final output.

# Frequently Asked Questions

## What topics should I focus on for a frontend interview?

Focus on HTML semantics and accessibility, CSS layout systems (flexbox and grid), JavaScript fundamentals including closures and the event loop, and how browsers render pages. Knowing one major framework like React or Vue deeply also helps.

## How do I prepare for JavaScript interview questions?

Build strong foundations in closures, prototypes, async patterns (promises, async/await), and event handling. Understand how the event loop works and be comfortable with array methods and functional programming concepts.

## What is the difference between var, let, and const?

`var` is function-scoped and hoisted. `let` is block-scoped and not hoisted in the same way. `const` is also block-scoped but cannot be reassigned after declaration. Use const by default, let when you need to reassign.

## How does CSS specificity affect my styles?

Specificity determines which CSS rule wins when multiple rules target the same element. Inline styles have highest priority, then IDs, then classes, then type selectors. Avoid using `!important` and keep specificity low for maintainability.

## What questions should I ask the interviewer?

Ask about the tech stack and why they chose it, the team structure.
