# Compiling C Programs with Make

For developers accustomed to modern package managers like npm or pip, compiling a C program from source can feel like black magic. But with the right tools, it's a completely manageable and rewarding process.

Let's demystify `make` and get your C programs compiled!

<Callout type="info" title="No Dependency Manager?">
  Unlike newer languages, C doesn't have a universal built-in dependency manager. You often have to hunt down and install dependencies manually via your OS package manager (`apt`, `brew`, etc.).
</Callout>

## The Compilation Pipeline

<Stepper>
  <Step title="Step 1: Install a C Compiler">
    You need `gcc` and `make`. On Ubuntu, run `sudo apt-get install build-essential`. On a Mac, install the Xcode command line tools.
  </Step>
  <Step title="Step 2: Install Dependencies">
    Install required libraries. For example, if a program needs `libqpdf`, run `sudo apt install -y libqpdf-dev` (Ubuntu) or `brew install qpdf` (Mac).
  </Step>
  <Step title="Step 3: Run ./configure">
    Many C projects include a `./configure` script. Run it! It checks your system and generates a customized `Makefile` based on your environment.
  </Step>
  <Step title="Step 4: Run make">
    Execute `make` in the terminal. Pro tip: Use `make -j8` to utilize 8 CPU cores and parallelize the build for massive speedups!
  </Step>
</Stepper>

### Troubleshooting Common Errors

<Callout type="danger" title="Library Not Found?">
  If you see `ld: library 'qpdf' not found`, your compiler doesn't know where the library is installed. You must pass the correct paths using environment variables!
</Callout>

To fix linker errors (especially common on macOS with Homebrew), manually pass the compiler (`CPPFLAGS`) and linker (`LDLIBS`) flags to `make`:

```bash
CPPFLAGS="-I/opt/homebrew/include" LDLIBS="-L/opt/homebrew/lib -liconv" make
```

Once compilation succeeds, you can usually install the binary system-wide by running `sudo make install`, or simply copy the generated binary to your `~/bin` folder!
