# Compiling C Programs with Make

If you're used to `npm install` or `pip install`, compiling a C program from source feels archaic. No dependency manager, no lockfile, no clear error messages — just a `Makefile` and your willingness to read linker output.

I've done this enough times now that the process is predictable. Here's the workflow that works.

## The Standard Dance: configure && make && make install

Most C projects follow this pattern:

```bash
./configure
make -j$(nproc)
sudo make install
```

- `./configure` checks your system for required libraries and generates a `Makefile` tailored to your environment
- `make` compiles everything
- `sudo make install` copies the binary to a system directory

Not all projects use autotools (`./configure`), but enough do that it's worth knowing. If there's no `configure` script, the project usually includes instructions in an `INSTALL` or `README` file.

## Installing Dependencies

The most common failure point: `./configure` exits with an error about a missing library.

```
checking for libqpdf... no
configure: error: Package requirements (libqpdf >= 8.0) were not met
```

On Ubuntu or Debian:

```bash
sudo apt install -y libqpdf-dev
```

On macOS with Homebrew:

```bash
brew install qpdf
```

The `-dev` suffix on Ubuntu packages matters — the runtime library (`libqpdf`) isn't enough; you need the headers in `libqpdf-dev` for compilation.

## The macOS Homebrew Problem

On macOS, libraries installed via Homebrew live in `/opt/homebrew/` (Apple Silicon) or `/usr/local/` (Intel). The compiler doesn't know to look there. If you see `ld: library not found` errors, pass the paths explicitly:

```bash
CPPFLAGS="-I/opt/homebrew/include" \
LDFLAGS="-L/opt/homebrew/lib" \
./configure
```

For projects that don't use `./configure`, you can set these inline with `make`:

```bash
CPPFLAGS="-I/opt/homebrew/include" LDLIBS="-L/opt/homebrew/lib -liconv" make
```

## Make It Fast

By default, make uses one CPU core. On a modern machine, that's wasting most of your hardware:

```bash
make -j$(nproc)    # Linux
make -j$(sysctl -n hw.ncpu)  # macOS
```

The `-j` flag sets parallel jobs. `$(nproc)` returns the number of CPU cores. This can cut compile time from minutes to seconds for larger projects.

## When Things Still Fail

**Symbol not found** — A library is too old or too new. Check the project's documented dependency versions.

**Permission denied** — The `./configure` script may not be executable. Run `chmod +x configure`.

**Custom install paths** — If you don't want to use `sudo make install` (or can't), use `./configure --prefix=$HOME/.local` to install to your user directory instead of `/usr/local/`.
