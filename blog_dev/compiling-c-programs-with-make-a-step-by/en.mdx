# Compiling C Programs with Make: A Step-by-Step Guide
=====================================================

As a non-C programmer, compiling C programs from source can be a daunting task. However, with the right tools and knowledge, it can be a manageable process. In this guide, we'll walk through the steps to compile a C program using `make`, including installing dependencies, running configure scripts, and troubleshooting common errors.

### Step 1: Install a C Compiler

Before we can compile a C program, we need to install a C compiler. On an Ubuntu system, we can install the `build-essential` package, which includes `gcc`, `g++`, and `make`:
```bash
sudo apt-get install build-essential
```
On a Mac, we can install the Xcode command line tools.

### Step 2: Install Dependencies

Unlike some newer programming languages, C doesn't have a built-in dependency manager. Therefore, we need to hunt down and install the dependencies manually. Thankfully, C programmers usually keep their dependencies minimal, and the dependencies are often available in the package manager.

For example, the `paperjam` program requires the `libqpdf-dev` and `libpaper-dev` packages. We can install them on a Debian-based system using:
```bash
sudo apt install -y libqpdf-dev libpaper-dev
```
Note that if we're using a Mac, we need to use Homebrew to install the dependencies. For example, we can install `qpdf` using:
```bash
brew install qpdf
```
### Step 3: Run `./configure` (if needed)

Some C programs come with a `Makefile`, while others come with a `./configure` script. The `./configure` script generates a `Makefile` based on the system configuration. For example, the `sqlite` program comes with a `./configure` script.

We can run the `./configure` script to generate a `Makefile`:
```bash
./configure
```
Note that the `./configure` script may fail if we're missing some dependencies.

### Step 4: Run `make`

Once we have a `Makefile`, we can run `make` to compile the program:
```bash
make
```
Note that we can also use `make -j8` to parallelize the build and make it faster.

### Troubleshooting Common Errors

When compiling a C program, we may encounter errors due to missing dependencies or incorrect compiler flags. For example, we may see an error message like:
```bash
ld: library 'qpdf' not found
```
To fix this error, we need to pass the correct compiler flags to `make`. We can use environment variables like `CPPFLAGS` and `LDLIBS` to pass flags to the compiler and linker.

For example, we can use the following command to pass flags to `make`:
```bash
CPPFLAGS="-I/opt/homebrew/include" LDLIBS="-L/opt/homebrew/lib -liconv" make
```
This command passes the `-I` flag to the compiler to specify the include directory and the `-L` and `-l` flags to the linker to specify the library directory and link in the `iconv` library.

### Installing the Binary

Once we've compiled the program, we can install it using the `make install` command. However, this command may not be available in all `Makefile`s. Alternatively, we can manually copy the binary to the installation directory:
```bash
cp qf ~/bin
```
### Contributing to Packaging Systems

Once we've figured out how to compile a C program, we can contribute to packaging systems like Homebrew. This allows us to easily install the program on future systems using a simple command like `brew install paperjam`.

### Conclusion

Compiling C programs using `make` can be a challenging task, but with the right knowledge and tools, it can be manageable. By understanding how to install dependencies, run `./configure` scripts, and troubleshoot common errors, we can successfully compile C programs and contribute to packaging systems.
