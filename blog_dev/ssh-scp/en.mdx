If you've spent any time working with servers, you've likely typed `ssh` into a terminal without thinking too hard about it. But when you get locked out at 2 AM or file transfers keep failing, knowing the mechanics of **SSH** (Secure Shell) and **SCP** (Secure Copy Protocol) becomes invaluable. 

<Callout type="info" title="What Even Is SSH?">
  SSH allows you to log into a remote computer and run commands securely. Before SSH, tools like Telnet transmitted everything—including passwords—in plain text. SSH fixed this by encrypting the entire session.
</Callout>

## Port 22 and Connectivity

By default, SSH connects over **port 22**. It's a "well-known port" reserved for system services. While you can't hide from determined attackers, moving SSH to a non-standard port (like `2222`) dramatically reduces automated brute-force attempts.

To connect to a custom port:
```bash
ssh -p 2222 user@yourserver.com
```

## Level Up Your SSH Workflow

<Stepper>
  <Step title="Stop Typing Passwords (Use SSH Keys)">
    Passwords are slow and insecure. Generate an Ed25519 key pair (it's faster and more secure than RSA):
    ```bash
    ssh-keygen -t ed25519 -C "your_email@example.com"
    ```
    Copy the public key to your server. From then on, you log in securely and instantly.
    ```bash
    ssh-copy-id user@yourserver.com
    ```
  </Step>

  <Step title="Configure Your SSH Profile">
    Stop memorizing IP addresses and ports. Create an SSH config file (`~/.ssh/config`) to create clean aliases for your servers.
    ```text
    Host myserver
        HostName 192.168.1.100
        User alice
        Port 2222
        IdentityFile ~/.ssh/id_ed25519
    ```
    Now, connecting is as simple as typing:
    ```bash
    ssh myserver
    ```
  </Step>

  <Step title="Transfer Files Securely with SCP">
    SCP uses SSH under the hood to transfer files between machines. The syntax is simply `scp source destination`.
    
    Copy to a server:
    ```bash
    scp myfile.txt myserver:/home/alice/
    ```
    Copy from a server (note the trailing `./`):
    ```bash
    scp myserver:/home/alice/myfile.txt ./
    ```
    <Callout type="warning" title="Capital P for SCP Ports">
      While SSH uses a lowercase `-p` for custom ports, SCP uses a capital `-P`. 
      `scp -P 2222 myfile.txt user@server:/path/`
    </Callout>
  </Step>
</Stepper>

## SCP vs. The Alternatives

SCP is fantastic for quick file transfers, but it isn't always the best tool for the job.

*   **`rsync`**: The ultimate tool for syncing directories. It only transfers changed files, making it drastically faster for large folders. (e.g., `rsync -avz local/ myserver:/remote/`)
*   **SFTP**: A robust SSH-based protocol that lets you interactively browse, rename, and delete files (used by GUI clients like Cyberduck).

<Callout type="danger" title="Crucial Security Fundamentals">
  Don't leave your server front door wide open! Edit `/etc/ssh/sshd_config` to harden your setup:
  1. **Disable Password Auth:** Set `PasswordAuthentication no`.
  2. **Disable Root Login:** Set `PermitRootLogin no`.
  3. **Use a Firewall:** Limit port 22 access using `ufw`.
  4. **Install Fail2Ban:** Automatically block IPs that fail too many login attempts.
</Callout>

## Conclusion

SSH and SCP are the fundamental plumbing of the modern web. By configuring SSH keys, writing a clean `~/.ssh/config`, and understanding basic server hardening, you'll move faster and sleep better knowing your infrastructure is secure.
