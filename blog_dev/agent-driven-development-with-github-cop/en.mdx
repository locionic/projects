# Agent-Driven Development with GitHub Copilot: A New Era in Software Collaboration

As a software engineer, have you ever found yourself stuck in a repetitive task, wishing you could automate it away? This is a common phenomenon, where we often build systems to remove toil and focus on more creative work, only to end up owning and maintaining those systems ourselves. But what if you could take this to the next level by leveraging AI-powered coding agents like GitHub Copilot?

In this article, we'll explore how to harness the power of coding agents to revolutionize your software development workflow, automate repetitive tasks, and unlock a faster development loop. We'll delve into the strategies and principles that make agent-driven development possible, and how to apply them to your own projects.

## The Impetus: Analyzing Coding Agent Performance

As an AI researcher, I was tasked with analyzing the performance of coding agents on standardized evaluation benchmarks. This involved poring through hundreds of thousands of lines of code, a daunting task that seemed impossible to do alone. But with the help of GitHub Copilot, I was able to surface patterns in the code and investigate them further, reducing the number of lines of code I had to read from hundreds of thousands to a few hundred.

However, I soon realized that this process was repetitive and could be automated. This sparked the idea of creating a tool that could automate this kind of intellectual work, and thus eval-agents was born.

## The Plan: Engineering and Science Teams Working Together

My guiding principle was to make eval-agents easy to share and use, and to make it easy to author new agents. I approached the design and implementation strategy with a few key goals in mind:

* Make coding agents the primary vehicle for contributions
* Make it easy to share and use agents
* Make it easy to author new agents

I leveraged the Copilot SDK to accelerate agent creation, which gave me access to existing tools and MCP servers, a way to register new tools and skills, and a whole bunch of other agentic goodness out of the box that I didn’t have to reinvent myself.

## Prompting Strategies: Conversational, Verbose, and Planning Modes

I found that agents work best when you're conversational, verbose, and when you leverage planning modes before agent modes. By putting some stream-of-consciousness musings about a problem into a prompt and working with Copilot in planning mode, I was able to get to the right solution faster.

Here's an example of a prompt I wrote to add more robust regression tests to the tool:
```
/plan I've recently observed Copilot happily updating tests to fit its new paradigms even though those tests shouldn't be updated. How can I create a reserved test space that Copilot can't touch or must reserve to protect against regressions?
```
This resulted in a back and forth that ultimately led to a series of guardrails akin to contract testing that can only be updated by humans.

## Architectural Strategies: Refactor Often, Update Docs Often, Clean Up Often

By refactoring names and file structures, documenting new features or patterns, and adding test cases for problems that I've uncovered as I go, I was able to make the codebase more readable and maintainable. This work makes it easy for Copilot to navigate the codebase and understand the patterns, just like it would for any other engineer.

## Iteration Strategies: Trust but Verify, Blame Process Not Agents

As agents and models have improved, I have moved from a "trust but verify" mindset to one that is more trusting than doubtful. This mirrors how the industry treats human teams: "blame process, not people." By implementing processes and guardrails to protect against mistakes, and when a mistake does happen, adding additional guardrails and processes, I was able to unlock an incredibly rapid iteration pipeline.

## Putting it All Together

Here's what all this means for your development loop when you've got your codebase set up for agent-driven development:

* Plan a new feature with Copilot using `/plan`.
* Iterate on the plan.
* Ensure that testing is included in the plan.
* Ensure that docs updates are included in the plan and done before code is implemented.
* Let Copilot implement the feature on `/autopilot`.
* Prompt Copilot to initiate a review loop with the Copilot Code Review agent.

By following these strategies and principles, you can unlock a faster development loop, automate repetitive tasks, and revolutionize your software development workflow.

## Takeaways

* The skills that make you a great engineer and a great teammate are the same skills that make you great at building with Copilot.
* Treat your Copilot like the newest member of your team.
* Go clean up that codebase, write that documentation you've been putting off, and start treating your Copilot like a valuable collaborator.

By embracing agent-driven development, you can automate yourself into the most interesting work, and unlock a new era in software collaboration.
