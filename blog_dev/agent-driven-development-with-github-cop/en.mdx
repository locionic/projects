# Agent-Driven Development with GitHub Copilot

Ever find yourself grinding through the same repetitive task, wishing you could just automate it away? We build systems to remove toil, only to end up maintaining those systems ourselves.

What if you could push that further with **AI-powered coding agents** like GitHub Copilot?

<Callout type="idea" title="The Goal">
  Automate intellectual, repetitive work so you can focus on creative problem-solving and architectural design!
</Callout>

## The Plan: Engineering & Science Teams Together

When you're designing tools for agent-driven development, three principles matter:

1. Make coding agents the primary vehicle for contributions.
2. Make it easy to share and use agents.
3. Make it easy to author new agents.

<Stepper>
  <Step title="Step 1: Leverage the Copilot SDK">
    Accelerate agent creation using the SDK. This grants access to existing tools, MCP servers, and out-of-the-box agentic goodness without reinventing the wheel.
  </Step>
  <Step title="Step 2: Master Prompting">
    Agents work best when you are conversational, verbose, and use planning modes (`/plan`) before jumping straight to autopilot.
  </Step>
  <Step title="Step 3: Refactor & Document Often">
    Clean up your codebase, rename files logically, and update documentation. Just like a human engineer, an AI agent needs a clean house to work efficiently.
  </Step>
</Stepper>

## Trust but Verify -> Blame Process, Not Agents

As models get better, we need to shift our mindset. Stop second-guessing the AI at every turn and treat it like a teammate: **blame the process, not the people.** If an agent makes a mistake, add better guardrails and tests.

<Callout type="info" title="The New Development Loop">
  1. Plan a new feature with `/plan`.
  2. Ensure testing and docs are in the plan.
  3. Let Copilot implement via `/autopilot`.
  4. Initiate a review loop with the Copilot Code Review agent.
</Callout>

### Takeaways

- The skills that make you a great teammate also make you great at working with Copilot.
- Treat your Copilot like the newest member of your team.
- Go clean up that codebase and write that documentation you've been putting off!
