An escape code is a sequence of characters that begins with the "escape" character (usually represented as `ESC`, `\x1b`, `\E`, `\033`, or `^[`). These codes are used by terminal emulators to communicate with programs running in the terminal. There are two types of escape codes:

*   **Input codes**: sent by the terminal emulator for keypresses or mouse movements that don't fit into Unicode.
*   **Output codes**: printed by programs to control the terminal, such as coloring text, moving the cursor, or clearing the screen.

Standards for ANSI Escape Codes
-------------------------------

### ECMA-48

The first standard for ANSI escape codes is ECMA-48, published in 1976. It defines general formats for escape codes, such as "CSI" codes (`ESC[` + something) and "OSC" codes (`ESC]` + something). ECMA-48 also specifies some common escape codes, like "move the cursor to the left" (`ESC[D`) and "turn text red" (`ESC[31m`). Although it provides a foundation for escape codes, many popular codes are not defined in ECMA-48.

### xterm Control Sequences

xterm, a widely used terminal emulator, has introduced many escape codes not defined in ECMA-48. These include enabling mouse reporting, bracketed paste, and OSC 52 for copying text to the system clipboard. While not a formal standard, the xterm control sequences document is influential and widely implemented.

### terminfo

In the 1980s, the terminfo database was created to manage the variations in escape code support across different terminals. The standard for terminfo is X/Open Curses, which defines the database format and a C library interface ("curses") for accessing the database. terminfo is still used today, although its usage has declined.

Should Programs Use terminfo?
-----------------------------

There are two main approaches to handling ANSI escape codes:

1.  **Use the terminfo database**: depend on the TERM environment variable to determine which escape codes to use.
2.  **Identify a "single common set" of escape codes**: hardcode a set of codes that work in most terminal emulators.

Some programs, like fish, use terminfo, while others, like kakoune and python-prompt-toolkit, use a hardcoded set of escape codes.

Is There a "Single Common Set" of Escape Codes?
------------------------------------------------

There is no clear agreement on a "single common set" of escape codes. It's likely a combination of:

*   The codes supported by the VT100 terminal
*   The codes defined in ECMA-48
*   The codes supported by xterm
*   Testing in popular terminal emulators

Reasons to Use terminfo
-----------------------

Some arguments in favor of using terminfo include:

*   Expectation of compatibility with the TERM environment variable
*   Variation in terminal emulator support for escape codes
*   Lack of a standard for the "single common set" of escape codes

Challenges and Opportunities
-----------------------------

Standardizing ANSI escape codes is a complex task. The web's history with HTML, CSS, and JavaScript shows that improvement is possible, but it requires effort and collaboration. A clearer standards landscape could enable terminal emulator developers to build new features and authors of terminal applications to adopt those features more confidently.

Conclusion
----------

ANSI escape codes are essential for terminal usability, but the standards surrounding them are fragmented. By exploring the existing standards and understanding the challenges, we can work towards a more standardized and reliable terminal ecosystem.
