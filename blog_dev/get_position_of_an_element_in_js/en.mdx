## The Problem

You try to get the vertical position of an element in a modern web app like ChatGPT:

```js
window.scrollY
```
...but it **always returns 0**! Even worse, `element.getBoundingClientRect().top + window.scrollY` completely fails.

<Callout type="warning" title="Why is it always 0?">
  In many modern SPAs (like ChatGPT), the page itself does not scroll! The `<body>` has a fixed height, and scrolling happens inside a **nested container**. Therefore, `window.scrollY` is completely useless.
</Callout>

---

## How to Fix It

<Stepper>
  <Step title="Step 1: Target the Element">
    Find the element you want to measure:
    ```js
    const el = document.querySelector("div.whitespace-pre-wrap");
    ```
  </Step>
  <Step title="Step 2: Find the True Scroll Container">
    Use a helper function to walk up the DOM and find the actual element that has an `overflowY` of `auto` or `scroll`:
    ```js
    function findScrollParent(element) {
      let parent = element.parentElement;
      while (parent) {
        const style = getComputedStyle(parent);
        if (/(auto|scroll)/.test(style.overflowY)) return parent;
        parent = parent.parentElement;
      }
      return document.documentElement;
    }
    const container = findScrollParent(el);
    ```
  </Step>
  <Step title="Step 3: Calculate the Real Y Position">
    Subtract the container's offset and add its current scroll position:
    ```js
    const y = el.getBoundingClientRect().top - container.getBoundingClientRect().top + container.scrollTop;
    console.log(y);
    ```
  </Step>
</Stepper>

---

### How to Verify the Value

<Callout type="idea" title="Test the Coordinate!">
  Draw a horizontal red line at your calculated Y to visually verify it perfectly aligns with your element:
  ```js
  const marker = document.createElement("div");
  marker.style.cssText = `position: absolute; left: 0; right: 0; background: red; height: 2px; top: ${y}px`;
  document.body.appendChild(marker);
  ```
</Callout>

### Key Takeaways

- `window.scrollY` only works when the `document` itself scrolls.
- Always compute positions relative to the **actual scroll container**.
- Use `getBoundingClientRect().top + container.scrollTop` for reliable results in complex SPAs, Browser Extensions, and UI Automation tools like Playwright!
