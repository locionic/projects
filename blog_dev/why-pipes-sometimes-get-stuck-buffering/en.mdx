# Why Pipes Sometimes Get "Stuck": Buffering

If you've ever chained Unix commands together and wondered why the output suddenly stops—or appears "stuck"—you've likely encountered **buffering**.

Buffering is a performance optimization technique where programs save up data (usually around 8KB) before writing it to a pipe or file. This reduces expensive system calls, making batch processing highly efficient.

<Callout type="danger" title="The Problem">
In a pipe, data should flow continuously. If a program buffers its output, it won't pass data down the chain immediately. The result? Your pipeline appears frozen.
</Callout>

### Common Culprits 

Many everyday Unix tools and languages buffer by default when writing to a pipe. Here is how to disable buffering for popular commands:

* **`grep`**: Use `--line-buffered`
* **`sed`**: Use `-u`
* **`awk`**: Use `fflush()`
* **`tcpdump`**: Use `-l`
* **`jq`**: Use `-u`
* **`tr`**: Use `-u`

<Callout type="warning" title="What about cut?">
The `cut` command has no built-in way to disable buffering!
</Callout>

### Programming Languages & Buffering

Your scripts might also be causing the jam. Here is how to fix it across different languages:

<Stepper>
  <Step title="Python">
    Use `python -u`, set `PYTHONUNBUFFERED=1`, use `sys.stdout.reconfigure(line_buffering=False)`, or `print(x, flush=True)`.
  </Step>
  <Step title="C">
    Use `setvbuf()` to explicitly disable buffering on standard streams.
  </Step>
  <Step title="Ruby & Perl">
    In Ruby, set `STDOUT.sync = true`. In Perl, set `$| = 1`.
  </Step>
</Stepper>

### Solutions to Keep the Data Flowing

<Callout type="idea" title="Pro Tips to Avoid Stuck Pipes">
1. **Use Specific Flags:** Always remember to pass the unbuffered flags (like `--line-buffered` for `grep`).
2. **Combine Commands:** Rewrite multiple `grep` operations into a single `awk` command.
3. **Use `stdbuf`:** This handy tool turns off libc's buffering for almost any command.
4. **Use `unbuffer`:** Trick a program into thinking it's writing to a TTY to force unbuffered output.
</Callout>

Understanding buffering empowers you to build robust, smooth-flowing pipelines. No more stuck terminal windows!
