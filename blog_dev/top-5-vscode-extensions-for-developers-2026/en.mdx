## Introduction

VS Code is the most popular code editor in the world — and a big reason for that is its incredible extension ecosystem. The right extensions can save you hours every week by automating repetitive tasks, catching errors early, and streamlining your workflow.

Here are the 5 VS Code extensions I use every single day in 2026.

---

## 1. GitHub Copilot

**What it does:** AI-powered code completion that suggests entire lines and functions as you type.

GitHub Copilot has become an essential tool for most developers. It understands context from your code and comments, and suggests relevant completions in real time.

**Best for:**
- Boilerplate code generation
- Writing repetitive patterns faster
- Exploring unfamiliar APIs

**How to use it:**
- Start typing and press `Tab` to accept suggestions
- Press `Alt+]` to cycle through alternative suggestions
- Write a comment describing what you want and let Copilot generate the code
```ts
// Create a function that validates an email address
function validateEmail(email: string): boolean {
  // Copilot suggests the regex automatically
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}
```

**Install:** Search `GitHub Copilot` in the VS Code Extensions marketplace.

---

## 2. ESLint

**What it does:** Catches JavaScript and TypeScript errors and enforces code style rules in real time.

ESLint highlights problems in your code as you write, before you even run it. It integrates with your project's ESLint config and shows warnings and errors directly in the editor.

**Best for:**
- Catching bugs early (undefined variables, unused imports)
- Enforcing consistent code style across a team
- Auto-fixing simple issues on save

**Setup tip** — add this to your VS Code settings to auto-fix on save:
```json
{
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}
```

**Install:** Search `ESLint` by Microsoft.

---

## 3. Prettier

**What it does:** Automatically formats your code on save — consistent indentation, quotes, semicolons, and more.

Prettier eliminates all debates about code formatting by enforcing one consistent style automatically. You never have to manually fix formatting again.

**Best for:**
- Consistent code formatting across your team
- Saving time on manual formatting
- Works with JS, TS, CSS, HTML, JSON, Markdown

**Setup tip** — add this to VS Code settings:
```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode"
}
```

**Install:** Search `Prettier - Code formatter` by Prettier.

---

## 4. GitLens

**What it does:** Supercharges VS Code's built-in Git features with powerful history, blame, and comparison tools.

GitLens shows you **who changed what and when** — right inside your editor. Hover over any line of code to see the last commit that changed it, the author, and the commit message.

**Best for:**
- Understanding why code was written a certain way
- Reviewing Git history without leaving VS Code
- Comparing branches and file revisions
- Finding when a bug was introduced

**Key features:**
- Inline blame annotations on every line
- File history and line history
- Interactive rebase editor
- Branch and commit comparison

**Install:** Search `GitLens` by GitKraken.

---

## 5. Playwright Test for VS Code

**What it does:** Run, debug, and record Playwright E2E tests directly from VS Code.

If you write Playwright tests (and you should!), this extension is a game changer. It adds a test explorer panel where you can run individual tests, see results inline, and even record new tests by clicking through your browser.

**Best for:**
- Running Playwright tests without leaving VS Code
- Debugging failing tests with breakpoints
- Recording new tests by interacting with your browser

**Key features:**
- Run single tests or entire test files with one click
- See pass/fail results inline in your test file
- Step-through debugging with breakpoints
- **Codegen** — record tests by clicking through your browser

**Install:** Search `Playwright Test for VS Code` by Microsoft.

## Bonus: My VS Code Settings

Here are my key VS Code settings that work great with these extensions:
```json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.tabSize": 2,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "explorer.confirmDelete": false,
  "typescript.updateImportsOnFileMove.enabled": "always"
}
```
