##エージェント本部のご紹介：どんなエージェントでも、どんな働き方でも

GitHub Universe 2025では、開発者のための次の進化を明らかにしました。オールインワンのワークフローにより、誰でも*任意の*エージェント、*どこでも*、Web、VS Code、またはコマンドラインからオーケストレーションできます。

##ミッションコントロール：いたるところにある指令センター

* GitHub、VS Code、モバイル、およびCLIにわたって統合されたダッシュボード。
*任意のデバイスから複数のエージェントを並行して割り当て、操縦、追跡します。
*エージェント生成コードの実行時間ポリシーのブランチコントロール。
* IDとポリシー機能を使用すると、他の開発者と同じようにエージェントを扱うことができます。

**主な機能**

|機能|提供するもの|
|--------|-------------------|
| **エージェントの選択** |事前に構築されたエージェントのフリートから選択します。|
| **並列ディスパッチ** |複数のエージェントを同時に実行します。|
| **進捗の追跡** |ステータスを一目で確認。|
| **ポリシーの適用** |エージェントごとにチェック、ポリシー、アクセスを設定します。|

##チームの自信とコントロール

###コード品質  
- ** GitHubコード品質**（パブリックプレビュー） ：メンテナンス性、信頼性、テストカバレッジのための組織全体の可視性とガバナンス。
- Copilotのセキュリティチェックには、メンテナンス性と信頼性への影響が含まれるようになりました。

###コードレビュー  
- Copilotは、独自のワークフローに最初のファーストラインレビューを追加し、開発者がコードを見る前に問題をキャッチします。

###メトリクス  
- ** Copilotメトリクスダッシュボード** (公開プレビュー):組織全体のインパクトと使用状況を確認できます。

###エンタープライズコントロール  
- ** AIコントロールプレーン**： AIアクセス、エージェントポリシー、監査ログ、およびメトリック収集のためのガバナンスレイヤー。
-利用可能なエージェント/モデルを指定し、組織全体で使用状況を追跡します。

## Kyle （ GitHub最高執行責任者）について

Kyleは、GitHubでカルチャー、アウトリーチ、オペレーション、コミュニケーションをリードしています。  
*エコシステムエンジニアリングチームの構築とスケーリング、主要な買収の監督（ Semmle、npm ）。  
* 3,000人以上のHubber従業員にAIの採用を促進します。  
*運用に実践をもたらし、GitHubの開発者ファーストの精神を維持する情熱的な開発者。  
*社外活動：ホームオートメーション、非営利団体、ゲーム、家族旅行。
