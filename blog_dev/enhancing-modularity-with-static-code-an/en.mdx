# Enhancing Modularity with Static Code Analysis Sensors

Birgitta Böckeler has expanded on the concept of static code analysis sensors, introducing three new sensors that concentrate on promoting better modularity in software design. These sensors build upon the existing computational sensors for dependency checks, which, although effective in enforcing rules, were limited in their scope.

## Limitations of Computational Sensors

While computational sensors for dependency checks were good at enforcing predefined rules, their effectiveness was restricted by the rules themselves. In other words, the rules had to be explicitly defined, which limited the sensor's ability to detect more complex modularity issues.

## Inferential Sensors for Coupling Data

In an effort to overcome these limitations, an inferential sensor was developed to review modularity by analyzing coupling data. However, the results were underwhelming, indicating that a different approach was needed.

## New Sensors for Enhanced Modularity

Birgitta Böckeler's work introduces three new static code analysis sensors that focus on enhancing modularity:

*   **Dependency Check Sensor**: This sensor builds upon the existing computational sensor, but with a more comprehensive approach to dependency checks.
*   **Coupling Data Sensor**: This sensor uses a more advanced analysis of coupling data to detect potential modularity issues.
*   **Modularity Review Sensor**: This sensor uses a combination of computational and inferential analysis to review modularity and provide recommendations for improvement.

These new sensors demonstrate the ongoing evolution of static code analysis and its potential to improve software design and modularity.
