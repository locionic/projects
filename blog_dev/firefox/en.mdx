# How to Harden Firefox on Linux (Beginner's Guide)

Firefox is already a better starting point than Chrome or Edge when it comes to privacy. It's open source, actively maintained, and Mozilla — for all its flaws — is at least not a company whose entire business model is built on selling your attention to advertisers.

But out of the box, Firefox still phones home more than it needs to. It collects telemetry, uses Google as the default search engine, and leaves a bunch of tracking protections either off or set to the weakest level. The good news is these are all easy to fix, and you don't need to be a security expert to do it.

This guide is for Linux users who want a meaningfully more private and secure browser without going down a three-hour rabbit hole. We'll work through it step by step, from the settings menu all the way to `about:config`.

---

## Before You Start: Back Up Your Profile

This is the one thing most guides skip, and then readers complain when something breaks and they can't undo it.

Your Firefox profile is where all your settings, bookmarks, and extensions live. On Linux it's tucked away at:

```
~/.mozilla/firefox/
```

Before changing anything, close Firefox and copy that folder somewhere safe:

```bash
cp -r ~/.mozilla/firefox/ ~/firefox-backup/
```

If something goes wrong or a website breaks and you can't figure out why, you can restore from this backup in about 30 seconds. It's worth the extra minute.

---

## Step 1: Turn Off Telemetry

The first thing to do is stop Firefox from sending usage data back to Mozilla. This includes interaction data, crash reports, and the ability for Mozilla to "install and run studies" on your browser — which sounds worse than it is, but still isn't something you need enabled.

Go to **Settings → Privacy & Security** and scroll down to **Firefox Data Collection and Use**. Uncheck everything:

- Allow Firefox to send technical and interaction data to Mozilla
- Allow Firefox to install and run studies
- Allow Firefox to send backlogged crash reports on your behalf

You can also disable this through `about:config` by setting `toolkit.telemetry.enabled` to `false`, but unchecking the boxes in Settings is enough for most people.

While you're here, also disable the AI integrations: go to **Settings → AI Controls** and turn off AI enhancements. This blocks Firefox from routing any browser data through third-party AI services.

---

## Step 2: Switch Your Search Engine

Firefox defaults to Google. Google's entire business is tracking what you search for. Swap it out.

Go to **Settings → Search → Default Search Engine** and pick one of these:

- **DuckDuckGo** — the most beginner-friendly option, good results, no search history tracking
- **Startpage** — returns Google results without letting Google see who you are
- **Brave Search** — independent index, solid results, no tracking

DuckDuckGo is the easiest choice if you're just getting started. You can always change it later.

---

## Step 3: Tighten Tracking Protection

Firefox has a feature called Enhanced Tracking Protection (ETP) that blocks known trackers, fingerprinters, and cross-site cookies. By default it's set to **Standard**, which is fine but not great.

Go to **Settings → Privacy & Security → Enhanced Tracking Protection** and switch it to **Custom**. Then set:

- **Cookies:** All cross-site cookies
- **Tracking content:** In all windows
- **Suspected fingerprinters:** In all windows
- **Suspected cryptominers:** checked

The warning about possible site breakage is real but overstated. The vast majority of sites work fine. If something breaks, you can add it to the exceptions list with one click — go to **Manage Exceptions** and add the domain.

---

## Step 4: Enable HTTPS-Only Mode

This one is simple and has no downside worth worrying about. HTTPS-Only Mode forces Firefox to connect using encrypted HTTPS connections wherever possible. If a site only supports plain HTTP, Firefox will warn you before loading it.

Go to **Settings → Privacy & Security** and scroll to the bottom. Under **HTTPS-Only Mode**, select **Enable HTTPS-Only Mode in all windows**.

That's it. You've just eliminated a whole class of attacks where someone on your network intercepts your traffic.

---

## Step 5: Enable DNS over HTTPS

By default, when you type a URL into Firefox, your DNS request — the lookup that translates a domain name into an IP address — is sent in plain text to your ISP's DNS server. Anyone on your network can see every domain you visit, even if the actual page content is encrypted.

DNS over HTTPS (DoH) fixes this by encrypting the DNS request itself.

Go to **Settings → Privacy & Security**, scroll to **DNS over HTTPS** and set it to **Max Protection**. Then choose a provider:

- **Cloudflare (1.1.1.1)** — fast, well-maintained, logs minimal data
- **Quad9 (9.9.9.9)** — also blocks known malicious domains, good for extra safety

Quad9 is a solid choice for beginners because it adds a layer of malware protection without requiring any extra setup.

---

## Step 6: about:config Tweaks

This is where newcomers often stop reading, but don't — it's easier than it looks. Type `about:config` in the address bar, click **Accept the Risk and Continue**, and you'll get a searchable list of every Firefox setting.

Use the search bar at the top to find each of these and set them to the values shown:

**Resist fingerprinting:**
```
privacy.resistFingerprinting = true
```
This makes your browser look more generic to websites trying to identify you by your browser's unique characteristics. Some sites may look slightly different — fonts and time zones can be affected.

**Disable WebGL (optional, stronger security):**
```
webgl.disabled = true
```
WebGL is used for 3D graphics in browsers but has a history of security vulnerabilities. Disable it if you don't use browser-based 3D apps or games. If a site breaks, you'll know why and can re-enable it.

**Enable Total Cookie Protection:**
```
privacy.partition.network_state = true
```
This isolates cookies per website so one site can't read cookies set by another. Firefox has this partially enabled by default but this setting strengthens it.

**Disable the Pocket integration:**
```
extensions.pocket.enabled = false
```
Pocket is Mozilla's read-later service baked into Firefox. If you don't use it, turn it off.

**Disable Firefox's built-in password manager prompt** (if you use a separate password manager like Bitwarden):
```
signon.rememberSignons = false
```

Don't go changing everything you see. These six are safe, meaningful changes with low risk of breaking things. Learn what each one does before going deeper.

## Step 7: Install the Right Extensions

Extensions are where you get the biggest bang for your effort. Install these three and you'll cover most of the remaining surface area:

**uBlock Origin** — the one extension everyone should have. It blocks ads, trackers, malware domains, and more. After installing it, go to its Dashboard → Filter Lists and enable:
- All built-in lists
- EasyList (Ads)
- EasyPrivacy + AdGuard URL Tracking Protection (Privacy)
- Online Malicious URL Blocklist (Security)

Don't install multiple ad blockers — they conflict and slow your browser down. uBlock Origin alone is better than any combination of two lesser blockers.

**Firefox Multi-Account Containers** — this is a Mozilla-made extension that lets you isolate different parts of your browsing into color-coded containers. Cookies in one container can't leak into another. Use it to keep your Google account in one container, your work stuff in another, and your personal browsing in a third. Trackers can't connect your activity across containers.

**Bitwarden** — not technically a hardening extension, but using a proper password manager means you can have unique, long passwords on every site. A browser with perfect privacy settings is still compromised if you're reusing passwords.

---

## Step 8: Consider a Separate Profile for Sensitive Browsing

One underrated tip: use Firefox profiles to separate your browsing contexts. Firefox lets you create multiple completely isolated profiles — each with its own cookies, extensions, history, and settings.

Create a new profile by typing `about:profiles` in the address bar and clicking **Create a New Profile**. Use this hardened profile for sensitive browsing (banking, healthcare, anything personal) and keep a more relaxed profile for streaming and casual browsing where you don't want fingerprinting resistance messing with video players.

Launch separate profiles simultaneously with:

```bash
firefox -P "ProfileName" --no-remote
```

It sounds like extra overhead, but once it's set up it becomes second nature.

## What You Don't Need to Do

Because beginner guides sometimes go too far:

**You don't need LibreWolf** unless you want someone else making these decisions for you. LibreWolf is a hardened Firefox fork with stricter defaults baked in — it's fine, but it's slower to receive security patches and you lose control over individual settings. Doing it yourself on mainline Firefox is better.

**You don't need to disable JavaScript** unless you're running a very high threat model. NoScript and similar extensions that block JS by default will break huge portions of the modern web and require constant whitelisting. uBlock Origin in default mode gives you most of the protection without the constant friction.

**You don't need 10 privacy extensions.** More extensions mean more attack surface, more potential conflicts, and a more unique browser fingerprint (ironic, but true). Keep it lean.

## Testing What You've Done

Want to see how your changes affected your browser's fingerprint? Run these tests after making the changes above:

- **[coveryourtracks.eff.org](https://coveryourtracks.eff.org)** — from the Electronic Frontier Foundation, shows how trackable your browser is
- **[browserleaks.com](https://browserleaks.com)** — detailed breakdown of what information your browser is leaking

Don't expect a perfect score. The goal isn't to be invisible — it's to not be the easiest person in the room to track. These changes put you well ahead of the average user.

---

## The Short Version

If you read nothing else, do these five things:

1. Turn off telemetry in Settings → Privacy & Security
2. Switch your search engine to DuckDuckGo or Startpage
3. Set Enhanced Tracking Protection to Custom with everything blocked
4. Enable HTTPS-Only Mode and DNS over HTTPS with Quad9
5. Install uBlock Origin

Everything else in this guide layers on top of that foundation. Start there, browse for a week, and come back for the `about:config` changes when you're ready.

A browser you actually use is better than a perfectly hardened one you've broken so badly you switched back to Chrome.
