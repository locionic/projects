# Post-Quantum Encryption in Cloudflare IPsec: A Major Milestone

While post-quantum cryptography has been gaining traction in the TLS world, the IPsec community has been slower to adopt. However, with the recent advances in quantum computing, it's become clear that post-quantum security is no longer a nicety, but a necessity. To address this, Cloudflare has made post-quantum encryption in Cloudflare IPsec generally available, using the new IETF draft for hybrid ML-KEM (FIPS 203).

## Cloudflare IPsec: A WAN Network-as-a-Service

Cloudflare IPsec is a WAN Network-as-a-Service that connects data centers, branch offices, and cloud VPCs to Cloudflare's global IP Anycast network. It provides simplified configuration, high availability, and the scale of Cloudflare's global network, all through encrypted IPsec tunnels.

## Post-Quantum Encryption in IPsec

Cloudflare IPsec now uses post-quantum encryption with hybrid ML-KEM (FIPS 203) to stop harvest-now-decrypt-later attacks. These attacks occur when an adversary harvests data today and then decrypts it later, after Q-Day, when powerful quantum computers can break classical public key cryptography. ML-KEM is a post-quantum cryptography algorithm based on mathematical assumptions that are not vulnerable to attacks by quantum computers. It's designed to be implemented in software across standard processors, providing post-quantum encryption of network traffic.

## Interoperable Implementation

Cloudflare has successfully tested interoperability with branch connectors from Fortinet and Cisco, using the new IETF draft for hybrid ML-KEM (FIPS 203). This means customers can start protecting their wide-area network (WAN) against harvest-now-decrypt-later attacks today, using hardware they already have.

## The Importance of Interoperability

Upgrading cryptography is hard and can take years. That's why Cloudflare hopes the IPsec community will continue to focus on the development of interoperable standards like draft-ietf-ipsecme-ikev2-mlkem. A full specification for hybrid ML-KEM in IPsec became available only in late 2025, roughly four years after support for hybrid ML-KEM landed in TLS.

## The Challenges of Quantum Key Distribution (QKD)

The IPsec community's interest in QKD has contributed to the delay in adopting post-quantum cryptography. However, QKD requires specialized hardware and a dedicated physical link between the two parties, making it unsuitable for Internet-scale deployment. Additionally, QKD does not provide authentication, so post-quantum cryptography is still necessary to stop active attackers.

## Towards an Interoperable Post-Quantum Internet

Cloudflare is committed to making a secure and post-quantum Internet accessible to everyone, without specialized hardware and at no extra cost to customers. Post-quantum Cloudflare IPsec is a major milestone on this path, and the company will continue to work towards full post-quantum security by 2029.
