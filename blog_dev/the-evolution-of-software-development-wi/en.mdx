# The Evolution of Software Development with LLMs

As Large Language Models (LLMs) become increasingly prevalent in code generation, the concept of cognitive debt has gained attention. Cognitive debt refers to the loss of understanding of a system's functionality, which can lead to difficulties in making changes or improvements. Margaret-Anne Storey proposes a framework for understanding system health, comprising three layers:

### Technical Debt

Technical debt lives in code and accumulates when implementation decisions compromise future changeability. It limits how systems can change.

### Cognitive Debt

Cognitive debt lives in people and accumulates when shared understanding of the system erodes faster than it is replenished. It limits how teams can reason about change.

### Intent Debt

Intent debt lives in artifacts and accumulates when the goals and constraints that should guide the system are poorly captured or maintained. It limits whether the system continues to reflect what we meant to build and how humans and AI agents can continue to evolve the system effectively.

## The Tri-System Theory of Cognition

A recent paper by Shaw and Nave adds LLMs to Kahneman's two-system model of thinking, introducing a third system:

### System 1 (Intuition)

Makes rapid decisions, often barely-consciously.

### System 2 (Deliberation)

Applies deliberate thinking to a problem.

### System 3 (LLMs)

Externally generated artificial reasoning, which can lead to cognitive surrender, characterized by uncritical reliance on externally generated artificial reasoning, bypassing System 2.

## The Importance of Verification

Ajey Gore argues that if coding agents make coding free, what becomes the expensive thing is verification. Verification is crucial in ensuring that the output of LLMs is correct and meets the desired goals. This encourages practices like Test-Driven Development (TDD) and highlights the need for more effort to make it easier for humans to comprehend larger ranges of tests.

## The Future of Source Code

The increasing use of LLMs raises questions about the future of source code. Some experts believe that existing languages, especially strictly typed languages like TypeScript and Rust, will be the best fit for LLMs. Others are experimenting with entirely new languages built with the LLM in mind.

## The Role of Humans in LLM-Driven Development

While LLMs can generate code efficiently, humans are still necessary to design verification systems, define quality, and handle ambiguous cases that LLMs can't resolve. This requires a reorganization of the development process, with a focus on verification and human judgment.

### Key Takeaways

* Cognitive debt and system health are critical concerns in LLM-driven development.
* Verification is essential in ensuring the correctness and quality of LLM-generated code.
* Humans play a crucial role in designing verification systems, defining quality, and handling ambiguous cases.
* The development process needs to be reorganized to prioritize verification and human judgment.
