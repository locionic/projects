# Write Your First Python Code

I've seen so many tutorials that make you read five paragraphs about what Python is before they show you a single line of code. It drives me crazy. Let's skip all of that.

Install Python, learn the four things that actually matter, and write something that runs. You can do this in under an hour.

<Callout type="info" title="Step 0: Install Python">
Download it from **python.org**. If you're on Windows, check the box that says **"Add Python to PATH"** during installation. I forgot this once and spent twenty minutes wondering why nothing worked. Verify everything by typing `python --version` in your terminal.
</Callout>

---

## Core Concepts to Know

Here's the thing. You don't need to understand object-oriented programming or decorators or generators to get started. You need exactly four things. Let me show you.

<Stepper>
  <Step title="Variables & Types">
    You don't have to declare types in Python. Just pick a name and set a value. It's that simple.

    ```python
    name = "Ava"       # string
    age = 19           # integer
    height = 1.72      # float
    is_ready = True    # boolean
    ```
  </Step>
  
  <Step title="Conditions (if/else)">
    What if the user is over 18 but not over 21? Conditions let your code make those decisions. Python uses **indentation** instead of curly brackets. This annoyed me at first, but now I prefer it. It forces clean code.

    ```python
    if age >= 18:
        print("You can vote!")
    else:
        print("Not yet.")
    ```
  </Step>
  
  <Step title="Lists & Loops">
    Ever had a list of things to do? Same idea here. Store multiple items, then loop through them. It's one of those patterns you'll use constantly.

    ```python
    colors = ["red", "green", "blue"]
    for color in colors:
        print(f"I love {color}")
    ```
  </Step>
  
  <Step title="Functions">
    Functions are just a way to wrap code you want to reuse. Type it once, call it anywhere. Your future self will thank you.

    ```python
    def greet(user_name):
        return f"Hi, {user_name}!"
    ```
  </Step>
</Stepper>

## Build a Mini Project

Theory is boring. Let's make something. An Odd/Even checker is the perfect first project. It uses all four concepts above and you can write it in two minutes.

```python
# 1. Ask for input
user_text = input("Enter a number: ")

# 2. Convert text to integer
number = int(user_text)

# 3. Check remainder using the modulo operator (%)
if number % 2 == 0:
    print("That number is even.")
else:
    print("That number is odd.")
```

Run it. Type a number. It works, right? Now here's the trick. Don't just move on.

<Callout type="warning" title="How to actually learn this">
Copy the code, then break it. Delete the colon. Mess up the indentation. Type `int(user_text` without the closing parenthesis. Read what Python tells you. Those error messages aren't your enemy. They're Python's way of saying "I found the exact thing that's wrong, here's where to look."

I learned more from breaking code than from writing it correctly the first time.
</Callout>

## What Next?

You've got the basics. Now what?

Pick something small and build it. A calculator that adds two numbers. A script that asks for your name and says hello. A temperature converter. The project doesn't matter. What matters is that you type it out, run into problems, and fix them yourself.

Every programmer I know started exactly here. One tiny project at a time, breaking things constantly. You're not behind. You're just getting started.
