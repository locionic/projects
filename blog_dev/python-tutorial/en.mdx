# Write Your First Python Code

Most tutorials spend five paragraphs telling you what Python is before showing you any code. Let's skip that. Install it, learn the four things you actually need to start, and write something that runs.

<Callout type="info" title="Step 0: Install Python">
Download it from **python.org**. If you're on Windows, check the box that says **"Add Python to PATH"** during installation (crucial!). Verify it works by typing `python --version` in your terminal.
</Callout>

---

## Core Concepts to Know

<Stepper>
  <Step title="Variables & Types">
    You don't need to declare types. Just pick a name and set a value.
    ```python
    name = "Ava"       # string
    age = 19           # integer
    height = 1.72      # float
    is_ready = True    # boolean
    ```
  </Step>
  
  <Step title="Conditions (if/else)">
    Branch your code based on conditions. Python uses **indentation** instead of curly brackets `{}`.
    ```python
    if age >= 18:
        print("You can vote!")
    else:
        print("Not yet.")
    ```
  </Step>
  
  <Step title="Lists & Loops">
    Store multiple items in a list, then loop through them with `for`.
    ```python
    colors = ["red", "green", "blue"]
    for color in colors:
        print(f"I love {color}")
    ```
  </Step>
  
  <Step title="Functions">
    Wrap reusable logic in a function so you're not repeating yourself.
    ```python
    def greet(user_name):
        return f"Hi, {user_name}!"
    ```
  </Step>
</Stepper>

## Build a Mini Project

Let's tie everything together with an Odd/Even checker:

```python
# 1. Ask for input
user_text = input("Enter a number: ")

# 2. Convert text to integer
number = int(user_text)

# 3. Check remainder using the modulo operator (%)
if number % 2 == 0:
    print("That number is even.")
else:
    print("That number is odd.")
```

<Callout type="warning" title="How to keep learning">
Don't just copy-paste. Break the code on purpose. Change the text, forget a colon, mess up the indentation, and read the error messages. Errors are just Python telling you exactly what it didn't understand.
</Callout>
