Mình nhớ từng ngồi nhìn một tutorial Python chạy nhanh quá mức, rồi tự nghĩ: “Hay đấy… nhưng mình bắt đầu viết code từ đâu?” Cảm giác lạc hướng và hơi ngại ngùng này cực kỳ phổ biến. Thế nên mình bỏ qua phần hiệu năng cho mấy thứ xa xôi, và chỉ tập trung giúp bạn viết Python để có thể tự chạy trên máy của mình.

## Python là gì (và không phải là gì)
Python là một ngôn ngữ lập trình. Nói đơn giản vậy thôi.

Nhưng bản chất hữu ích hơn là: Python thân thiện với người mới vì cú pháp dễ đọc và thông báo lỗi thường đủ “tử tế” để bạn có thể khôi phục và sửa lại.

Tuy vậy, Python không phải phép màu. Bạn vẫn phải học các nền tảng: biến (variables), kiểu dữ liệu (data types), luồng điều khiển (control flow) và hàm (functions). Tin vui là Python làm những bước đó đỡ đau khổ hơn rất nhiều so với nhiều ngôn ngữ khác.

## Cài Python (để máy tính bạn thật sự chạy được)
Nếu bạn dùng Windows, hãy tải Python từ python.org và tick vào ô kiểu như “Add Python to PATH” (thêm Python vào PATH). Nếu không, bạn sẽ gặp lỗi kinh điển: gõ `python` mà máy không thấy, rồi bạn sẽ ngồi cả buổi tối đổ lỗi cho… thứ không đúng.

Trên macOS, Python thường đã có sẵn, nhưng mình vẫn khuyên bạn cài bản mới nhất từ python.org. Còn trên Linux, bạn thường có thể dùng trình quản lý gói (package manager), nhưng nếu muốn ít “bất ngờ” hơn thì bản từ python.org vẫn an toàn hơn.

Sau khi cài xong, mở terminal và chạy:

```bash
python --version
```

Bạn sẽ thấy hiện ra một số phiên bản. Nếu có lỗi thì xử lý lỗi trước. Không có chuyện code gì trước khi mọi thứ chạy được.

## Chương trình thật đầu tiên của bạn
Mở Python REPL (terminal) hoặc tạo một file tên `hello.py`. Gõ:

```python
print("Hello, world!")
```

Chạy nó:

```bash
python hello.py
```

Nếu chạy được thì bạn đã làm được việc quan trọng nhất: biến “học” thành “chạy code.” Mọi thứ còn lại gần như là lặp lại, nhưng mỗi lần bạn sẽ đặt được câu hỏi tốt hơn một chút.

## Biến: ngừng nghĩ “type”, bắt đầu đặt tên cho đúng
Trong Python, bạn không khai báo biến kèm kiểu dữ liệu. Bạn chỉ cần gán giá trị:

```python
name = "Ava"
age = 19
height = 1.72
```

Nếu bạn in ra:

```python
print(name)
print(age)
print(height)
```

Hầu hết lỗi của người mới đến từ việc lưu sai “loại giá trị”, nên đừng chỉ học thuộc cú pháp. Hãy để ý biến thực sự đang giữ gì.

## Các kiểu dữ liệu cơ bản mà bạn dùng liên tục
Bạn sẽ gặp các kiểu này ở khắp nơi:

- `int` cho số nguyên: `10`, `-3`
- `float` cho số thập phân: `3.14`, `-0.5`
- `str` cho chuỗi/ký tự: `"hi"`
- `bool` cho đúng/sai: `True`, `False`

Demo nhanh:

```python
x = 5
y = 2.5
message = "Python"
is_ready = True
```

Và vâng, Python khá “khó tính” với dấu nháy khi viết chuỗi. `"Python"` là một chuỗi (string). Còn `Python` không có dấu nháy sẽ được hiểu là tên một biến—và nếu bạn chưa định nghĩa biến đó, Python sẽ phàn nàn.

## Toán học (và một quy tắc nhiều người quên)
Python làm phép tính như bạn mong đợi:

```python
total = 10 + 5
difference = 10 - 3
product = 4 * 2
quotient = 9 / 3
```

Có một điểm làm nhiều người vấp: `/` sẽ cho ra số kiểu float bất cứ khi nào có lý do để nó làm vậy. Nếu bạn muốn chia lấy phần nguyên, dùng `//`:

```python
print(7 // 2)  # 3
print(7 / 2)   # 3.5
```

Ngoài ra, `%` trả về phần dư. Nó xuất hiện liên tục trong các bài kiểu “số chẵn hay lẻ”.

## Điều kiện: ra quyết định bằng if
Cấu trúc cơ bản sẽ trông như thế này:

```python
age = 17

if age >= 18:
    print("You can vote.")
else:
    print("You can't vote yet.")
```

Lưu ý phần thụt dòng (indentation). Python coi khoảng trắng là một phần của ngôn ngữ. Không có dấu ngoặc nhọn, không có dấu chấm phẩy—chỉ có thụt dòng. Nếu các dòng của bạn không thẳng hàng, Python sẽ báo lỗi. Lúc đầu thấy khó chịu, rồi bạn sẽ “thấm” ra ngay.

## Vòng lặp: làm việc gì đó nhiều lần
Lúc mới học, bạn sẽ dùng `for` nhiều hơn.

### Ví dụ vòng lặp for
```python
for i in range(5):
    print(i)
```

Đoạn này sẽ in ra `0` đến `4`. `range(5)` nghĩa là “bắt đầu từ 0, dừng trước 5.”

### Ví dụ vòng lặp while
```python
count = 3
while count > 0:
    print(count)
    count -= 1
```

Cẩn thận với `while`. Nếu điều kiện dừng không bao giờ thay đổi, vòng lặp có thể chạy mãi mãi. Nếu rơi vào trường hợp đó thì dừng bằng Ctrl+C ngay trong terminal.

## Danh sách (lists): “siêu năng lực” của người mới
Một list chứa nhiều giá trị:

```python
colors = ["red", "green", "blue"]
print(colors[0])  # "red"
```

List là thứ bạn dùng để chứa “một đống thứ.” Bạn có thể thêm và cập nhật phần tử:

```python
colors.append("yellow")
colors[1] = "lime"
```

Duyệt qua từng phần tử:

```python
for color in colors:
    print(color)
```

## Hàm: đừng tự lặp lại
Nếu bạn thấy mình đang copy-paste code, khả năng cao bạn cần một hàm.

```python
def greet(name):
    return f"Hi, {name}!"

message = greet("Sam")
print(message)
```

Hàm giúp bạn tổ chức code tốt hơn và dùng lại dễ dàng để kiểm thử. Người mới hay bỏ qua ở giai đoạn đầu, rồi sau đó nhận ra chương trình của mình nhanh chóng trở nên lộn xộn.

## Một mini dự án luyện tập nhỏ mà thật sự thấy có ích
Làm gì nhỏ thôi nhé: bộ kiểm tra số.

```python
number = int(input("Enter a number: "))

if number % 2 == 0:
    print("That number is even.")
else:
    print("That number is odd.")
```

Có hai kỹ năng quan trọng:
- `input()` lấy chuỗi do người dùng nhập
- `int(...)` chuyển chuỗi đó thành số để bạn có thể tính toán

Chạy thử nó. Đổi input. Cố ý “đập” nó một chút cho lỗi xảy ra. Đó là cách học nhanh hơn.

## Học Python thế nào để không mất động lực
Quan điểm thật của mình là: học kiểu tua nhanh tutorial không “dính” được lâu như người ta kỳ vọng. Bạn có cảm giác mình làm được, nhưng bạn nhớ ít hơn bạn nghĩ.

Chọn một mục tiêu nhỏ, viết code, và chấp nhận rằng lỗi là một phần của quá trình. Khi Python ném lỗi cho bạn, hãy đọc kỹ. Nó không hề “drama”—nó đang nói chính xác là nó không hiểu phần nào.
