The data sprawl led to several issues:

*   **Too many disparate systems**: Each system had its own credentials, query language, and retention policy, making it difficult for analysts to access and query the data.
*   **Sampled data**: The analytics pipeline downsampled data to handle high volumes, but this made it unsuitable for certain use cases, such as billing.
*   **External dependencies**: Some internal reporting relied on external vendors, introducing additional costs and dependencies.
*   **Tribal knowledge**: Data infrastructure was treated as a back-office function, leading to a lack of understanding and expertise among employees.

## The Solution
----------------

To address these challenges, Cloudflare built two in-house tools:

*   **Town Lake**: A unified data analytics platform that provides a single SQL interface to all Cloudflare data.
*   **Skipper**: An AI data agent that enables employees to ask questions in plain English and receive accurate, auditable answers.

### Town Lake

Town Lake is a data lakehouse architecture that consists of:

*   **Query engine**: Apache Trino, which allows for SQL queries to join data from multiple sources without materializing intermediate results.
*   **R2 Data Catalog**: A managed Apache Iceberg service that stores cold and warm data.
*   **DataHub**: A metadata catalog that provides information about tables, columns, owners, and lineage.
*   **Lifeguard**: An access control service that stores access rules and dynamically pulls user and group membership.
*   **Skimmer**: A PII detection scanner that continuously samples rows from every column and uses Workers AI to classify sensitive data.

### Skipper

Skipper is a conversational AI agent that:

*   **Finds the right tables**: Uses DataHub search to identify relevant tables and their schemas.
*   **Writes SQL**: Generates SQL queries based on the user's question.
*   **Submits queries**: Executes queries on Trino and polls for results.
*   **Provides answers**: Displays results in a table or chart.
*   **Carries context**: Refines queries based on user feedback and adjusts the query accordingly.

### Layers of Context

To ensure accurate results, Skipper uses multiple layers of grounded context:

*   **Schema and usage metadata**: DataHub provides information about tables, columns, and usage patterns.
*   **Human annotations**: Team owners provide descriptions and annotations for tables and columns.

By using these layers of context, Skipper can provide accurate and relevant answers to user queries.

## Conclusion
----------

Cloudflare's unified data platform, Town Lake, and AI data agent, Skipper, have empowered employees to access and query data in a secure and governed manner. By providing a single SQL interface to all Cloudflare data and using AI to generate accurate and relevant answers, Town Lake and Skipper have improved the overall data experience at Cloudflare.
