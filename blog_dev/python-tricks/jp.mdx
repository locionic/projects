# 本当に使えるPythonの小技集

「頭いいでしょ」とアピールするためだけの難解なコードは忘れましょう。ここで紹介するのは、コードをきれいで読みやすく、バグの少ない状態に保つために、私が毎日使っている実用的なテクニックです。

<Callout type="idea" title="なぜこれらを使うのか？">
これらの小技は見せびらかすためではなく、冗長なコードを減らし、インデックスのズレ（Off-by-one）エラーを防ぎ、コードを読むときの脳の負担を減らすためにあります。
</Callout>

---

## 毎日使えるベストテクニック

<Stepper>
  <Step title="1. 変数をスマートに入れ替える">
    `temp` みたいな一時変数はもう不要です。
    ```python
    a, b = b, a
    ```
  </Step>
  
  <Step title="2. enumerate() でインデックス管理を楽に">
    `for i in range(len(items))` なんて書くのはやめましょう。
    ```python
    for i, item in enumerate(items):
        print(i, item)
    ```
  </Step>
  
  <Step title="3. zip() で2つのリストを同時に回す">
    2つのリストをスッキリと一緒にループ処理できます。
    ```python
    for name, score in zip(names, scores):
        print(name, score)
    ```
  </Step>
  
  <Step title="4. dict.get() で KeyError を回避">
    キーが存在しない場合のデフォルト値を指定して、クラッシュを防ぎます。
    ```python
    count = counts.get(key, 0)
    ```
  </Step>
</Stepper>

---

## コードをさらにレベルアップ

<Callout type="info" title="5. 高速な検索には Set を使う">
要素が存在するかどうかを確認したい場合 (`if action in allowed:`)、リストではなく Set (`{"read", "write"}`) を使いましょう。Setは要素のチェックが爆速です！
</Callout>

<Callout type="info" title="6. リスト内包表記 (Comprehensions)">
ごちゃごちゃした `for` ループを読みやすい1行にまとめられます（ただしネストのしすぎには注意）。
```python
squares = [x * x for x in nums if x >= 0]
```
</Callout>

<Callout type="warning" title="7. 使わない値は _ (アンダースコア) で捨てる">
関数が複数の値を返してきても、1つしか使わない場合は `_` を使いましょう。「この値は意図的に無視しているよ」と他の開発者に明確に伝えられます。
```python
_, result = fetch_data()
```
</Callout>
