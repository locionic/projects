前は「Pythonの小技」って、だいたいは“文法の小アレンジ”みたいなものだと思ってました。気の利いた書き方を覚えたらちょっと嬉しい、みたいな。

でも、コードがぐちゃぐちゃになり始めたときに、そういう小技に頼るようになったんです。すると不思議と「かわいい」から「役に立つ」に変わりました。行数は減るし、バグも減るし、考える負担も軽くなる。

ここに、実際に何度も戻って使ってる10個をまとめます。「見て見て賢いでしょ」みたいな、いわゆる見せかけのやつはなし。次にループを書いたり、辞書を組み立てたり、イレギュラーに頭を抱えたりしたときに、そのまま使える実用ネタだけです。

---

## 1) 一時変数なしで変数を入れ替える
```python
a, b = b, a
```

短いのに読みやすい。しかも、最初から用意する必要のなかった余計な変数を避けられます。

---

## 2) インデックスを自作しないために `enumerate()` を使う
`for i in range(len(xs))` みたいなことをやっているなら、不要な定型文を自分で背負ってます。

```python
for i, x in enumerate(xs):
    print(i, x)
```

これなら、欲しいときだけインデックスを取れます。

---

## 3) 2つのシーケンスを一緒に回すなら `zip()` を使う
```python
for name, score in zip(names, scores):
    print(name, score)
```

短いほうに合わせて止まります。データが多少不揃いなときも、だいたいこれがちょうどいいです。

---

## 4) if/elseの散らかりを避けて `dict.get()` を使う
```python
count = counts.get(key, 0)
counts[key] = count + 1
```

存在しないキーが分岐の温床になって、こっちが想定してない分岐ロジックを増やす——そういう流れを止められます。

---

## 5) 集計や似たパターンには `defaultdict` を使う
```python
from collections import defaultdict

counts = defaultdict(int)
counts[key] += 1
```

「なんで前からこれ使ってなかったんだろう」となるタイプの変更です。

---

## 6) 複数の例外をまとめてキャッチする
```python
try:
    x = int(s)
except (TypeError, ValueError):
    x = 0
```

毎回“とりあえず全部”キャッチするより、具体的な例外を指定するほうがいいです。

---

## 7) `in` と set を組み合わせて高速なメンバーシップチェック
```python
allowed = {"read", "write", "delete"}
if action in allowed:
    ...
```

メンバーシップチェックを何度もやるなら、リストよりsetのほうが有利です。

---

## 8) リスト内包表記でループを読みやすく保つ
平方の例：
```python
squares = [x * x for x in nums if x >= 0]
```

辞書のマッピング：
```python
by_id = {user["id"]: user for user in users}
```

責任を持って使う前提だけど、内包表記はコードをキュッと締めてくれるのに、難読化にはなりにくいです。

---

## 9) 手書きループの代わりに `any()` / `all()` を使う
```python
if any(x is None for x in values):
    ...
if all(x > 0 for x in scores):
    ...
```

ループの“手順”を書くより、意図をそのまま表現できます。

---

## 10) 値を本当に気にしないなら `_` を使う
```python
for _, value in items:
    print(value)
```

アンパックするときも同じ：
```python
_, result = some_tuple_returning_fn()
```

「この変数は解決に関係ないんだよ」というのをはっきり示せます。
