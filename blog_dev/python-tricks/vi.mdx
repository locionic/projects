Trước đây mình nghĩ mấy “mẹo” Python chủ yếu là kiểu linh hoạt về cú pháp — cho đến khi mình bắt đầu dùng chúng khi code bị bừa bộn. Lúc đó, chúng không còn “ngầu cho vui” nữa mà trở nên hữu ích: ít dòng hơn, ít bug hơn, và giảm hẳn gánh nặng phải suy nghĩ trong đầu.

Dưới đây là 10 thứ mình thực sự quay lại dùng thường xuyên. Không phải kiểu giả bộ “nhìn xem mình thông minh thế nào” đâu. Chỉ là các cách làm thực dụng bạn có thể áp dụng ngay lần tới khi viết vòng lặp, dựng một dictionary, hay phải vật lộn với các trường hợp biên.

---

## 1) Đổi chỗ biến mà không cần biến tạm
```python
a, b = b, a
```

Ngắn gọn, dễ đọc, và quan trọng là không phải tạo thêm một biến phụ mà bạn thật ra chẳng cần.

---

## 2) Dùng `enumerate()` để khỏi tự chế indexing
Nếu bạn đang làm `for i in range(len(xs))` thì bạn đang tự ký hợp đồng với một mớ boilerplate chẳng cần thiết.

```python
for i, x in enumerate(xs):
    print(i, x)
```

Giờ bạn có index chỉ khi bạn thật sự muốn nó.

---

## 3) Dùng `zip()` để duyệt cùng lúc hai dãy
```python
for name, score in zip(names, scores):
    print(name, score)
```

Nó sẽ dừng khi tới dãy ngắn hơn — mà thường thì chính xác thứ bạn cần khi dữ liệu không đều.

---

## 4) Dùng `dict.get()` thay vì nhét if/else cho rối
```python
count = counts.get(key, 0)
counts[key] = count + 1
```

Những key bị thiếu không còn biến thành bài toán rẽ nhánh mà bạn không hề muốn.

---

## 5) Dùng `defaultdict` cho đếm số và các pattern tương tự
```python
from collections import defaultdict

counts = defaultdict(int)
counts[key] += 1
```

Đây là một kiểu thay đổi khiến bạn tự hỏi: “Sao hồi trước mình không làm thế nhỉ?”

---

## 6) Bắt nhiều exception ở cùng một chỗ
```python
try:
    x = int(s)
except (TypeError, ValueError):
    x = 0
```

Exception cụ thể bao giờ cũng tốt hơn kiểu bắt bừa mọi thứ.

---

## 7) Dùng `in` với set để kiểm tra membership nhanh
```python
allowed = {"read", "write", "delete"}
if action in allowed:
    ...
```

Nếu bạn kiểm tra membership nhiều lần, set sẽ nhanh hơn list.

---

## 8) Dùng list/dict comprehensions để vòng lặp gọn hơn
Bình phương:
```python
squares = [x * x for x in nums if x >= 0]
```

Ánh xạ theo id:
```python
by_id = {user["id"]: user for user in users}
```

Dùng đúng cách thì comprehensions giúp code “chặt” hơn mà không hề khó đọc.

---

## 9) Dùng `any()` / `all()` thay vì tự viết vòng lặp thủ công
```python
if any(x is None for x in values):
    ...
if all(x > 0 for x in scores):
    ...
```

Bạn đang diễn đạt đúng ý định thay vì phải viết ra cơ chế vòng lặp.

---

## 10) Dùng `_` khi bạn thật sự không quan tâm giá trị đó
```python
for _, value in items:
    print(value)
```

Và khi unpack:
```python
_, result = some_tuple_returning_fn()
```

Nó báo rõ ràng: “biến này không phải một phần của lời giải.”
