I used to think Python “tricks” were mostly syntax flex—until I started reaching for them when code got messy. That’s when they stopped being cute and started being useful: fewer lines, fewer bugs, and less mental overhead.

Here are 10 I actually keep coming back to. Not the fake “look how clever I am” kind. Just practical stuff you can use the next time you’re writing a loop, building a dictionary, or wrestling with edge cases.

---

## 1) Swap variables without a temp variable
```python
a, b = b, a
```

It’s short, readable, and it avoids that extra variable you didn’t need in the first place.

---

## 2) Use `enumerate()` so you don’t reinvent indexing
If you’re doing `for i in range(len(xs))`, you’re signing up for needless boilerplate.

```python
for i, x in enumerate(xs):
    print(i, x)
```

Now you get the index only when you actually want it.

---

## 3) Use `zip()` to iterate two sequences together
```python
for name, score in zip(names, scores):
    print(name, score)
```

It stops at the shorter sequence, which is usually exactly what you want when data is uneven.

---

## 4) Use `dict.get()` instead of if/else clutter
```python
count = counts.get(key, 0)
counts[key] = count + 1
```

Missing keys stop turning into branching logic you didn’t ask for.

---

## 5) Use `defaultdict` for counting and similar patterns
```python
from collections import defaultdict

counts = defaultdict(int)
counts[key] += 1
```

This is one of those “why didn’t I do this earlier?” changes.

---

## 6) Catch multiple exceptions in one place
```python
try:
    x = int(s)
except (TypeError, ValueError):
    x = 0
```

Specific exceptions beat blanket catching every time.

---

## 7) Use `in` with sets for fast membership checks
```python
allowed = {"read", "write", "delete"}
if action in allowed:
    ...
```

If you’re checking membership a lot, a set beats a list.

---

## 8) Use comprehensions to keep loops readable
Squares:
```python
squares = [x * x for x in nums if x >= 0]
```

Dictionary mapping:
```python
by_id = {user["id"]: user for user in users}
```

Used responsibly, comprehensions make code feel tighter without being harder to read.

---

## 9) Use `any()` / `all()` instead of manual loops
```python
if any(x is None for x in values):
    ...
if all(x > 0 for x in scores):
    ...
```

You’re expressing intent directly instead of spelling out the loop mechanics.

---

## 10) Use `_` when you truly don’t care about a value
```python
for _, value in items:
    print(value)
```

And when unpacking:
```python
_, result = some_tuple_returning_fn()
```

It signals clearly: “this variable isn’t part of the solution.”
