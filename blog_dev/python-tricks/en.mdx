# 10 Practical Python Tricks

Forget the clever "golfed" code that no one can read. These are the practical Python features I reach for every day to keep my codebase clean, readable, and bug-free.

<Callout type="idea" title="Why use these?">
These tricks aren't just for showing off—they actively reduce boilerplate, prevent off-by-one errors, and decrease your cognitive load while reading code.
</Callout>

---

## The Best Everyday Tricks

<Stepper>
  <Step title="1. Swap variables cleanly">
    No need for a temporary `temp` variable.
    ```python
    a, b = b, a
    ```
  </Step>
  
  <Step title="2. Ditch manual indexing with enumerate()">
    Stop doing `for i in range(len(items))`. 
    ```python
    for i, item in enumerate(items):
        print(i, item)
    ```
  </Step>
  
  <Step title="3. Iterate pairs with zip()">
    Loop through two lists at the same time cleanly.
    ```python
    for name, score in zip(names, scores):
        print(name, score)
    ```
  </Step>
  
  <Step title="4. Avoid KeyError with dict.get()">
    Provide a default value if the key doesn't exist, preventing crashes.
    ```python
    count = counts.get(key, 0)
    ```
  </Step>
</Stepper>

## Level Up Your Code

<Callout type="info" title="5. Use Sets for Fast Lookups">
If you need to check if an item exists (`if action in allowed:`), use a set (`{"read", "write"}`) instead of a list. Sets are highly optimized for membership checks!
</Callout>

<Callout type="info" title="6. List Comprehensions">
Turn messy `for` loops into readable one-liners (but don't nest them too deeply).
```python
squares = [x * x for x in nums if x >= 0]
```
</Callout>

<Callout type="warning" title="7. Ignore values with _ (Underscore)">
If a function returns multiple values but you only need one, use `_` to signal to other developers that the value is intentionally ignored.
```python
_, result = fetch_data()
```
</Callout>
