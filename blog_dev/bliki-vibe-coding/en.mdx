The term "vibe coding" was coined by Andrej Karpathy in February 2025. Karpathy described vibe coding as a process where the user "fully gives in to the vibes" and "forgets that the code even exists." This approach is useful for rapid development of disposable software or prototypes, but it also poses significant risks.

## Key Characteristics of Vibe Coding
-----------------------------------

The key characteristic of vibe coding is that the user does not review or understand the generated code. This approach is different from agentic programming, where the user reviews and understands the generated code.

### Advantages of Vibe Coding
-----------------------------

Vibe coding has several advantages, including:

* **Rapid development**: Vibe coding enables rapid development of software applications without requiring extensive programming knowledge.
* **Ease of use**: Vibe coding is accessible to non-programmers, who can build applications without needing to write code.

### Limitations of Vibe Coding
-----------------------------

Vibe coding also has several limitations, including:

* **Maintainability**: Vibe-coded software can be difficult to maintain, as the generated code may be of low quality and hard to understand.
* **Correctness**: Vibe-coded software may contain errors or incorrect behavior, which can be difficult to detect.
* **Security**: Vibe-coded software can pose significant security risks, as the generated code may expose sensitive information or credentials.

## Risks Associated with Vibe Coding
-------------------------------------

Vibe coding poses several risks, including:

* **Security risks**: Vibe-coded software can expose sensitive information or credentials, making it vulnerable to attacks.
* **Low-quality code**: Vibe-coded software may contain low-quality code, which can be difficult to maintain and extend.
* **Non-determinism**: Vibe-coded software may exhibit non-deterministic behavior, making it difficult to predict its behavior.

## Best Practices for Vibe Coding
---------------------------------

To mitigate the risks associated with vibe coding, it is essential to follow best practices, including:

* **Use vibe coding for disposable software**: Vibe coding is best suited for disposable software that is only used by its author or a close group of collaborators.
* **Be aware of the risks**: Users should be aware of the risks associated with vibe coding and take steps to mitigate them.
* **Use LLMs with caution**: Users should use LLMs with caution and be aware of their limitations.
