Katalon Studio is a powerhouse automation testing tool built on top of **Selenium** and **Appium**. Whether you're a beginner or a seasoned pro, it has you covered!

In this guide, we'll blast through setting up Katalon on Linux, squashing a notorious JVM bug, and writing a slick script to automate a Bootstrap date picker.

<Callout type="info" title="What to Expect">
  We'll be automating a real-world scenario: searching for a product and picking a target date dynamically using XPath!
</Callout>

---

## 1. Setting Up Katalon Studio on Linux

<Stepper>
  <Step title="Download the Archive">
    First, grab the latest version of Katalon Studio Enterprise for Linux using `wget`:
    
    ```bash
    wget https://download.katalon.com/10.3.2/Katalon_Studio_Enterprise_Linux_64-10.3.2.tar.gz
    tar -xvzf Katalon_Studio_Enterprise_Linux_64-10.3.2.tar.gz
    cd Katalon_Studio_Enterprise_Linux_64-10.3.2
    ```
  </Step>
  <Step title="Launch It!">
    Run the executable script:
    
    ```bash
    ./katalon
    ```
  </Step>
</Stepper>

<Callout type="danger" title="Did it crash immediately?">
  If you see `Error: Could not create the Java Virtual Machine.`, **your Java version is too old.** Katalon 10.x requires Java 17+. Simply install Java 17 and point your `katalon.ini`'s `-vm` argument to the new Java path!
</Callout>

## 2. Automating the Date Picker

Let's automate the **HP LP3065** page from the TutorialsNinja demo site to dynamically select `04/Jan/2021`.

### Object Repository Setup

Ditch the hardcoded locators! Set up your objects in Katalon with these smart **Dynamic XPath** locators:

| Object Name | XPath Locator |
| ---- | ------- |
| `input_Search` | `//input[@name='search']` |
| `picker_Year` | `//span[text()='${year}']` |
| `picker_Month` | `//span[text()='${month}']` |
| `picker_Day` | `//td[@class='day' and text()='${day}']` |

<Callout type="idea" title="Why Dynamic XPath?">
  The `${year}` syntax allows us to pass variables from our script directly into the locator at runtime!
</Callout>

### The Groovy Script

Here's how we bring it all together:

```groovy
import com.kms.katalon.core.webui.keyword.WebUiBuiltInKeywords as WebUI

WebUI.openBrowser('http://tutorialsninja.com/demo/')
WebUI.setText(findTestObject('Page_Home/input_Search'), 'HP LP3065')
WebUI.click(findTestObject('Page_Home/button_Search'))

// Pick the date dynamically!
def targetYear = '2021'
def targetMonth = 'Jan'
def targetDay = '4'

WebUI.click(findTestObject('Page_Product/picker_Year', [('year') : targetYear]))
WebUI.click(findTestObject('Page_Product/picker_Month', [('month') : targetMonth]))
WebUI.click(findTestObject('Page_Product/picker_Day', [('day') : targetDay]))
```

<Callout type="warning" title="Out-of-Range Years?">
  If your target year isn't visible, use a `while` loop with `FailureHandling.OPTIONAL` to keep clicking the navigation arrows until it appears!
</Callout>
