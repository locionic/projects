Katalon Studio isn't the tool everyone talks about in 2025. But if you're doing enterprise QA automation, it saves you from stitching together five different libraries. It wraps Selenium and Appium, gives you a UI, built-in keywords, test management, and reporting out of the box.

This guide covers getting it running on Linux, working around a JVM version issue, and automating a Bootstrap date picker with dynamic XPath.

<Callout type="info" title="What to Expect">
  We'll search for a product and pick a target date using XPath variables passed at runtime.
</Callout>

---

## 1. Setting Up Katalon Studio on Linux

<Stepper>
  <Step title="Download the Archive">
    Download the latest Katalon Studio Enterprise for Linux using `wget`:

    ```bash
    wget https://download.katalon.com/10.3.2/Katalon_Studio_Enterprise_Linux_64-10.3.2.tar.gz
    tar -xvzf Katalon_Studio_Enterprise_Linux_64-10.3.2.tar.gz
    cd Katalon_Studio_Enterprise_Linux_64-10.3.2
    ```
  </Step>
  <Step title="Launch It">
    Run the executable script:

    ```bash
    ./katalon
    ```
  </Step>
</Stepper>

<Callout type="danger" title="Did it crash immediately?">
  If you see `Error: Could not create the Java Virtual Machine.`, **your Java version is too old.** Katalon 10.x requires Java 17+. Install Java 17 and point your `katalon.ini`'s `-vm` argument to the new Java path.
</Callout>

I hit this exact error the first time. Spent twenty minutes thinking my download was corrupt. Turns out my system had Java 11. Katalon 10.x needs Java 17 minimum. Fixed it by installing OpenJDK 17 and updating the `-vm` line in `katalon.ini` to point to the new Java binary.

## 2. Automating the Date Picker

Let's automate the **HP LP3065** page from the TutorialsNinja demo site and select `04/Jan/2021`.

### Object Repository Setup

Skip the hardcoded locators. Set up your objects in Katalon with dynamic XPath locators instead:

| Object Name | XPath Locator |
| ---- | ------- |
| `input_Search` | `//input[@name='search']` |
| `picker_Year` | `//span[text()='${year}']` |
| `picker_Month` | `//span[text()='${month}']` |
| `picker_Day` | `//td[@class='day' and text()='${day}']` |

<Callout type="idea" title="Why Dynamic XPath?">
  The `${year}` syntax lets you pass variables from your script directly into the locator at runtime.
</Callout>

### The Groovy Script

Here's the full script I use:

```groovy
import com.kms.katalon.core.webui.keyword.WebUiBuiltInKeywords as WebUI

WebUI.openBrowser('http://tutorialsninja.com/demo/')
WebUI.setText(findTestObject('Page_Home/input_Search'), 'HP LP3065')
WebUI.click(findTestObject('Page_Home/button_Search'))

// Pick the date dynamically
def targetYear = '2021'
def targetMonth = 'Jan'
def targetDay = '4'

WebUI.click(findTestObject('Page_Product/picker_Year', [('year') : targetYear]))
WebUI.click(findTestObject('Page_Product/picker_Month', [('month') : targetMonth]))
WebUI.click(findTestObject('Page_Product/picker_Day', [('day') : targetDay]))
```

<Callout type="warning" title="Out-of-Range Years?">
  If your target year isn't visible, loop with a `while` and `FailureHandling.OPTIONAL` to click the navigation arrows until it appears.
</Callout>

The key insight: `${year}`, `${month}`, `${day}` in the XPath become runtime parameters. You pass them as a map when calling `findTestObject`. No string concatenation. No brittle selectors.

Want to pick a different date? Change three variables at the top. That's it.

## 3. Handling Edge Cases

The script above works for visible dates. But what if the year you need is two years back?

```groovy
def targetYear = '2019'

while (true) {
    def yearElement = WebUI.findTestObject('Page_Product/picker_Year', [('year') : targetYear], FailureHandling.OPTIONAL)
    if (yearElement != null) {
        WebUI.click(yearElement)
        break
    }
    WebUI.click(findTestObject('Page_Product/btn_PreviousYear'))
}
```

This loops backward until the year appears. Same pattern works for months.
