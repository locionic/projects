# Katalon, Playwright, Java — Interview Practice

<Callout type="idea" title="Chuẩn bị sẵn sàng!">
Nắm vững các khái niệm cốt lõi của Katalon, Playwright và Java sẽ giúp bạn tự tin vượt qua các buổi phỏng vấn QA Automation.
</Callout>

## Katalon Studio

**Q1. Katalon Studio là gì, khác biệt thế nào với Selenium?**

<Callout type="info" title="Nền tảng toàn diện">
Katalon là nền tảng tự động hóa kiểm thử hoàn chỉnh dựa trên Selenium & Appium.
Khác với Selenium (chỉ là thư viện), Katalon cung cấp sẵn IDE, keywords, quản lý test case, report và tích hợp CI/CD.
</Callout>

**Q2. Xử lý test data trong Katalon Studio như thế nào?**

Sử dụng Data-Driven Testing (DDT):
- Lưu trữ qua Excel, CSV, Database và bind vào test case.
- Tận dụng **Data Files** và **Global Variables** để tái sử dụng dữ liệu hiệu quả.

---

## Playwright

**Q3. Ưu điểm của Playwright so với Selenium hoặc Puppeteer?**

<Callout type="warning" title="Không cần cài Driver">
Playwright đi kèm sẵn các trình duyệt (Chromium, Firefox, WebKit) - chỉ dùng một API duy nhất.
</Callout>

- **Auto-wait:** Tự động chờ các element.
- **Web hiện đại:** Xử lý mượt mà SPA và shadow DOM.
- **Tốc độ:** Hỗ trợ nhiều context để chạy test song song, nhanh hơn Puppeteer.

**Q4. Xử lý multiple browser contexts và chạy song song?**

<Stepper>
  <Step title="Browser Contexts">Tạo nhiều context độc lập (có cookie, storage riêng) trong cùng một browser instance.</Step>
  <Step title="Parallel Execution">Tận dụng Playwright Test runner chia test ra nhiều workers (CPU cores) để giảm thời gian chạy.</Step>
</Stepper>

## Java (for Test Automation)

**Q5. Các khái niệm OOP áp dụng vào test automation như thế nào?**

<Stepper>
  <Step title="Đóng gói (Encapsulation)">Gom nhóm dữ liệu và hàm vào class (vd: Page Object Model).</Step>
  <Step title="Kế thừa (Inheritance)">Tái sử dụng code dùng chung từ class cha (vd: BaseTest cho setup/teardown).</Step>
  <Step title="Đa hình (Polymorphism)">Ghi đè (override) các method tùy theo ngữ cảnh thực thi.</Step>
  <Step title="Trừu tượng (Abstraction)">Ẩn chi tiết phức tạp, chỉ bộc lộ tính năng cần thiết (vd: abstract class cho browser).</Step>
</Stepper>

**Q6. Xử lý ngoại lệ (Exceptions) trong script test?**

Dùng `try-catch-finally`.
<Callout type="info">
Trong automation, bắt ngoại lệ giúp ta ghi log lỗi, chụp màn hình tự động, đóng trình duyệt gọn gàng hoặc retry lại test step bị lỗi!
</Callout>
