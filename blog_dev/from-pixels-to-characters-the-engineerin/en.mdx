# From Pixels to Characters: The Engineering Behind GitHub Copilot CLI's Animated ASCII Banner
====================================================================

Most people think ASCII art is simple, but when the GitHub Copilot CLI team asked for a small entrance banner for the new command-line experience, they discovered the opposite: An ASCII animation in a real-world terminal is one of the most constrained UI engineering problems you can take on.

### The Challenge of Terminals

Terminals behave differently, have few shared standards, and offer almost no consistent accessibility guidelines. Different terminals interpret ANSI color codes differently, screen readers treat fast-changing characters as noise, layout engines vary, buffers flicker, and some users override global colors for accessibility.

### By the Numbers

* 3 seconds of animation
* ~20 frames
* ~6,000 lines of TypeScript
* Dozens of terminal + theme combinations tested

### The Technical Story

#### What's New in GitHub Copilot CLI

GitHub Copilot CLI brings agentic workflows directly into your terminal, letting you plan projects, modify files, run commands, use custom agents, and delegate tasks to the cloud, all without leaving the CLI.

#### Why Animated ASCII is a Hard Engineering Problem

Terminals don't have a canvas, and every "frame" has to be manually repainted using cursor movements and redraw commands. ANSI escape codes are inconsistent, and terminal color is its own engineering challenge.

### The Engineering Process

#### Part 1: A Request that Didn’t Fit Any Workflow

Cameron Foxly, a brand designer at GitHub, was asked to create a banner for the Copilot CLI. He wanted something more fun than a static ASCII intro, so he decided to create an animated 3D Copilot mascot flying in to reveal the CLI logo.

#### Part 2: Building an ASCII Animation Editor from Scratch

Cameron opened an empty repository in VS Code and began asking GitHub Copilot for help scaffolding an animation MVP that could read text files as frames, render them sequentially, control timing, clear the screen without flicker, and add a primitive "UI".

#### Part 3: ANSI Color Theory and the Real-World Limitations

The Copilot brand palette is vibrant and high-contrast, which is great for web but exceptionally challenging for terminals. Cameron needed a way to paint characters with ANSI color roles while previewing how they look in different terminals.

#### Part 4: Exporting to Ink (React for the Terminal)

Cameron asked Copilot to help generate an Ink component that would accept frames, render them line-by-line, animate them with state updates, and integrate cleanly into the CLI codebase.

#### Part 5: Terminal Animation Isn’t Solved Technology

Andy Feller, a long-time GitHub engineer, partnered with Cameron to bring the animation into the Copilot CLI codebase. Unlike browsers, terminal animation isn't a solved technology, and the team had to build a custom solution.

### Conclusion

The GitHub Copilot CLI's animated ASCII banner is a technically impressive project that required a custom design toolchain and a tight pairing between a designer and an engineer. The project highlights the challenges of working with terminals and the importance of accessibility and consistency in UI engineering.
