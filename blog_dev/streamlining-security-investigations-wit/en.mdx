# Streamlining Security Investigations with AI Agents

Slack's Security Engineering team is responsible for safeguarding the company's core infrastructure and services. The team's security event ingestion pipeline processes billions of events daily from diverse data sources. Reviewing alerts generated by the security detection system is a primary responsibility during on-call shifts. This post explores how the team utilizes AI agents to optimize working efficiency and bolster Slack's security defenses.

## The Development Process

### The Prototype

In May 2025, the team developed a rudimentary prototype of the service, which consisted of a 300-word prompt with five sections:

* **Orientation**: Defined the role of the security analyst
* **Manifest**: Specified available data sources
* **Methodology**: Outlined investigation steps
* **Formatting**: Requested a markdown report
* **Classification**: Provided response classification options

The team implemented a simple "stdio" mode MCP server to safely expose a subset of data sources and repurposed a coding agent CLI as an execution environment. However, the prototype's performance was highly variable, and the team needed to refine the investigation process to achieve consistent results.

### The Solution

To address the limitations of the prototype, the team broke down the complex investigation process into a sequence of model invocations, each with a single, well-defined purpose and output structure. These tasks were chained together by the application, and each task had a structured output format. This approach provided more precise control at each step of the investigation process.

## Investigation Loop

The team designed an investigation loop consisting of three defined persona categories:

* **Director Agent**: The Investigation Director, responsible for progressing the investigation from start to finish
* **Expert Agent**: A domain expert, responsible for producing findings from their data sources in response to the Director's questions
* **Critic Agent**: A "meta-expert," responsible for assessing and quantifying the quality of findings made by domain experts

The investigation loop involves the following steps:

1. The Director poses a question and domain experts respond, generating findings.
2. The Critic reviews findings for quality and assembles a timeline using the most credible.
3. The Director uses the high-quality findings and timeline to determine how to progress the investigation.

## Knowledge Pyramid

The team created a "knowledge pyramid" to strategically use low, medium, and high-cost models for the expert, critic, and director functions, respectively. The pyramid consists of three levels:

1. **Domain Experts**: Generate investigation findings by interrogating complex data sources
2. **Critic**: Reviews findings and assembles an investigation timeline
3. **Director**: Uses the condensed timeline to progress the investigation

## Investigation Flow

The investigation process is broken into several phases, allowing the team to vary the structure of the investigation loop as the investigation proceeds. The current phases include:

1. **Discovery**: The first phase, where the goal is to examine every available data source
2. **Director Decision**: A "meta-phase" where the Director decides whether to advance to the next investigation phase or continue in the current one
3. **Trace**: The Director transitions the investigation to this phase when sufficient information has been gathered to focus on a specific expert
4. **Conclude**: The Director transitions the investigation to this phase when sufficient information has been gathered to produce the final report

## Service Architecture

The team created a service architecture to support the investigation loop, consisting of:

1. **Hub**: Provides the service API and an interface to persistent storage
2. **Worker**: Investigation workers pick up queued investigation tasks from the API
3. **Dashboard**: Used by staff to interact with the service, observe investigations in real-time, and manage model invocations

## Example Report

The team included an edited investigation report demonstrating the potential of the agents to exhibit novel emergent behavior. In this case, the original alert was raised for a specific command sequence, which the agents analyzed and discovered a separate credential exposure elsewhere in the process ancestry.

The investigation report highlights the ability of the agents to:

* Identify critical security concerns
* Notice contradictions between expert and critic assessments
* Pivot the investigation to focus on a new issue

The report demonstrates the value of using AI agents to streamline security investigations and enhance the efficiency and effectiveness of the Security Engineering team.
