# Những Thư Viện Python Thực Sự Đáng Dùng

Đừng nhắm mắt chọn bừa một thư viện chỉ vì nó nghe có vẻ "phổ biến". Dưới đây là danh sách những công cụ mình thực sự tin tưởng cho năm 2026—đã được màng lọc qua thực tế triển khai production.

<Callout type="idea" title="Tiêu Chí Chọn Lọc">
"Chạy được" khác xa với "chạy ổn định khi đưa lên production". Hãy chọn những thư viện giúp bạn bảo vệ sức khỏe tinh thần khi dự án phình to.
</Callout>

---

## Web & APIs

<Stepper>
  <Step title="FastAPI (Lựa chọn mặc định)">
    Siêu phẩm hiện đại để làm API. Cực nhanh, validation dữ liệu tuyệt vời nhờ Pydantic, và tự động tạo document (Swagger) siêu xịn.
  </Step>
  
  <Step title="Django (Kẻ vác ngà voi)">
    Nếu bạn cần admin panel, ORM, auth, và migrations ngay từ đầu? Triết lý "batteries included" của Django sẽ cứu bạn hàng trăm giờ code cho các web app phức tạp.
  </Step>
</Stepper>

---

## Dữ liệu & Machine Learning

<Callout type="info" title="Xử Lý Dữ Liệu">
- **pandas:** Vẫn là vị vua không ngai trong việc nhào nặn và làm sạch dữ liệu CSV/Excel hằng ngày.
- **polars:** Kẻ thay thế hiện đại, tốc độ bàn thờ. Cực kỳ hữu ích khi pandas bắt đầu báo lỗi hết RAM.
</Callout>

<Callout type="info" title="Machine Learning & AI">
- **scikit-learn:** Xương sống của ML cổ điển (regression, clustering, random forests).
- **PyTorch:** Ông trùm vô đối trong thế giới Deep Learning và nghiên cứu.
- **Hugging Face Transformers:** Điểm bắt đầu mặc định nếu bạn muốn chạy, fine-tune hay vọc vạch các mô hình LLM xịn xò hiện nay.
</Callout>

## Tooling & Kiểm Thử

<Stepper>
  <Step title="Testing: pytest">
    Quên `unittest` đi. `pytest` là ông hoàng testing của Python nhờ hệ thống fixture và parametrization cực mạnh.
  </Step>
  
  <Step title="Linting: ruff">
    Kẻ thay thế hoàn hảo cho Flake8, Black, isort... Chạy nhanh như chớp và dẹp tan mọi cuộc cãi vã về format code trong team.
  </Step>
  
  <Step title="Quản lý package: Poetry">
    Nói lời tạm biệt với `requirements.txt` hay bị vỡ. Poetry mang lại cách quản lý dependency hiện đại và file lock an toàn.
  </Step>
</Stepper>

<Callout type="warning" title="Lưu Ý Về Async">
Đừng lạm dụng async (`aiohttp`, `httpx`) trừ khi bạn thật sự cần xử lý đồng thời lượng lớn request. Với 90% các script gọi API thông thường, thư viện `requests` cổ điển vẫn là lựa chọn dễ đọc và an tâm nhất.
</Callout>
