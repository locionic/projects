# The Python Libraries That Actually Deliver

Don't just pick whatever library sounds popular. I've derailed too many weekend projects by choosing the wrong tools. Here are the tools I'm betting on for 2026—filtered for actual production readiness.

<Callout type="idea" title="The Rule">
"It works" is not the same as "It works well when you ship." Choose libraries that scale with your sanity.
</Callout>

---

## Web & APIs

<Stepper>
  <Step title="FastAPI (The Default)">
    The modern go-to for APIs. Extremely fast, incredible type validation (thanks Pydantic), and auto-generates Swagger docs. 
  </Step>
  
  <Step title="Django (The Heavy Lifter)">
    Need an admin panel, ORM, auth, and migrations out of the box? Django’s "batteries included" philosophy saves hundreds of hours for full web apps.
  </Step>
</Stepper>

---

## Data & Machine Learning

<Callout type="info" title="Data Manipulation">
- **pandas:** Still the undefeated king for general CSV/Excel data wrangling.
- **polars:** The modern, blazingly fast alternative to pandas. Use this when pandas runs out of memory.
</Callout>

<Callout type="info" title="Machine Learning & AI">
- **scikit-learn:** The absolute backbone for classic ML (regression, clustering, random forests).
- **PyTorch:** The undisputed leader for Deep Learning and research.
- **Hugging Face Transformers:** The default starting point if you want to run, fine-tune, or experiment with modern LLMs.
</Callout>

---

## Testing & Tooling

<Stepper>
  <Step title="Testing: pytest">
    Don't even look at `unittest`. `pytest` is the king of Python testing thanks to fixtures and parametrization.
  </Step>
  
  <Step title="Linting: ruff">
    Replaces Flake8, Black, isort, and more. It runs incredibly fast and stops arguments about formatting instantly.
  </Step>
  
  <Step title="Package Management: Poetry">
    Say goodbye to broken `requirements.txt`. Poetry provides sane dependency management and lock files.
  </Step>
</Stepper>

<Callout type="warning" title="A Note on Async">
Don't use async (`aiohttp`, `httpx`) unless you actually need high concurrency. For 90% of basic scripts, good old `requests` is still the best and most readable choice.
</Callout>
