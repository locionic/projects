# モダンなPython開発環境の構築

従来の `pip install` だけでは、バージョン競合や「自分のPCでは動くのに」という問題が頻発します。
現代のPython開発には **バージョン管理・環境分離・依存関係のロック** が不可欠です。

<Callout type="info" title="現代のPython開発に必要なツール">
- **pyenv:** Pythonのバージョン（3.10, 3.12など）を管理。
- **Poetry / uv:** 依存関係と仮想環境の管理を統合。
- **pyproject.toml:** プロジェクトの設定を標準化。
</Callout>

---

## 環境構築のステップ

<Stepper>
  <Step title="Step 1: pyenvでPythonバージョンを管理する">
    システムPythonを汚さず、プロジェクトごとにバージョンを切り替えます。
    ```bash
    # インストールとバージョン切り替え
    pyenv install 3.12
    pyenv local 3.12  # プロジェクト専用に設定
    ```
  </Step>
  
  <Step title="Step 2: 仮想環境 (venv) を作る">
    グローバル環境の汚染を防ぐため、プロジェクト専用の分離された環境を作ります。
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```
  </Step>
  
  <Step title="Step 3: パッケージ管理をPoetryに移行する">
    `requirements.txt` の代わりに `pyproject.toml` とロックファイルで依存関係を厳密に管理します。
    ```bash
    # プロジェクト作成とパッケージの追加
    poetry new myapp
    poetry add requests
    ```
  </Step>
</Stepper>

---

## どのツールを選ぶべきか？

<Callout type="idea" title="ツールの選び方">
- **venv:** Python標準。シンプルで確実。
- **Pipenv:** 過去に流行したが、今は少し重い。
- **Poetry:** 現在のデファクトスタンダード。依存関係解決が優秀。
- **uv:** 超高速。これからの新しいトレンド。
</Callout>

<Callout type="warning" title="注意">
システム全体に `pip install` するのは避けましょう。必ず仮想環境を作り、プロジェクトごとに依存関係を分離してください！
</Callout>
