# pyenv、Poetry、uv、およびpyproject.tomlを使ったモダンなPython開発環境構築

最近のプロジェクトでPythonのバージョンを切り替えることが頻繁に必要になり、システムのPythonが複数のパッケージで混乱していることに気づきました。プロジェクトの依存関係を追跡するのも難しく、より強固なシステムが必要だと感じました。その結果、`pyenv`、`Poetry`、`uv`、および`pyproject.toml`を発見し、これらのツールがPythonの開発ワークフローを大幅に改善してくれました。

伝統的な`pip install requests`アプローチは、現代のPythonプロジェクトには不十分です。今日のプロジェクトには以下が求められます：
- 複数のPythonバージョン
- 環境の分離
- 依存関係のロック
- 再現可能なビルド
- 標準化された設定
- 高速なパッケージ管理

さまざまなツールやワークフローを試した結果、一貫して結果を出すセットアップを見つけました。

# モダンなPythonツールの重要性

適切なツールがないと、Pythonプロジェクトはすぐにフラストレーションの源になります。よくある問題には以下があります：
- 依存関係の競合が解決に数時間かかる
- グローバルパッケージの汚染で他のプロジェクトが壊れる
- 環境の不一致で「私の環境では動く」問題が発生する
- デプロイメントが壊れてダウンタイムが発生する
- 新しいチームメンバーのオンボーディングが難しい

最近の2つのプロジェクトを考えてみましょう。プロジェクトAはPython 3.10とDjango 4が必要で、プロジェクトBはPython 3.12と最新のFastAPIが必要でした。これらをシステムのPythonだけで管理しようとすると、大変なことになります。モダンなツールはこれらの問題を解決し、分離、再現可能性、そしてより良い依存関係管理を提供します。

# pyenvを使ったPythonバージョン管理

`pyenv`は複数のPythonバージョンを管理するのを大いに簡単にしてくれます。異なるPythonバージョンを簡単にインストールして切り替えることができます。これにより、システムパッケージを管理する必要や、効果の低い仮想環境に頼る必要がありません。

## pyenvのインストール

UbuntuやLinuxでは、`pyenv`のインストールは簡単です。通常、以下のコマンドから始めます：

```bash
curl https://pyenv.run | bash
```

その後、環境を設定するために`.bashrc`に以下の行を追加します：

```bash
export PATH="$HOME/.pyenv/bin:$PATH"
eval "$(pyenv init -)"
```

シェルをリロードするために`source ~/.bashrc`を実行すると、`pyenv`が設定されます。

## 複数のPythonバージョンのインストール

`pyenv`を使って複数のPythonバージョンをインストールするのは簡単です。通常、プロジェクトで必要なバージョンをインストールします：

```bash
pyenv install 3.10
pyenv install 3.11
pyenv install 3.12
```

インストールされているバージョンを確認するには、以下のコマンドを実行します：

```bash
pyenv versions
```

出力は以下のようになります：

```text
3.10
3.11
3.12
```

## グローバルPythonバージョンの設定

グローバルPythonバージョンを設定するには、以下のコマンドを使用します：

```bash
pyenv global 3.12
```

これにより、Python 3.12がすべてのプロジェクトに対するデフォルトバージョンとなります。

## プロジェクトごとのローカルPythonバージョンの設定

プロジェクト固有のPythonバージョンを設定するには、プロジェクトディレクトリに移動し、以下のコマンドを実行します：

```bash
pyenv local 3.11
```

これにより、プロジェクトディレクトリに`.python-version`ファイルが作成され、このプロジェクトでPython 3.11を使用するように`pyenv`に指示します。これで、プロジェクトディレクトリに入るたびに`pyenv`が自動的に正しいPythonバージョンに切り替わります。

# 仮想環境

仮想環境は、プロジェクトごとに依存関係を分離するために不可欠です。仮想環境がないと、グローバルにパッケージをインストールすることで競合や壊れたプロジェクトが発生することがあります。例えば、あるプロジェクトのためにパッケージをインストールする際に、他のプロジェクトに影響を与えることがあります。仮想環境は、各プロジェクトに対して分離された環境を作成することでこの問題を解決します。

## venvの使用

`venv`はPythonに組み込まれており、仮想環境を作成するための良いスタート地点です。仮想環境を作成するには、以下のコマンドを実行します：

```bash
python -m venv .venv
```

環境をアクティブにするには、以下のコマンドを使用します：

```bash
source .venv/bin/activate
```

Windowsでは、コマンドが少し異なります：

```powershell
.venv\Scripts\activate
```

環境がアクティブになると、グローバルな競合を心配せずにパッケージをインストールできます。例えば：

```bash
pip install requests fastapi
```

依存関係をフリーズして`requirements.txt`ファイルを作成するには、以下のコマンドを使用します：

```bash
pip freeze > requirements.txt
```

これにより、インストールされたパッケージとそのバージョンが記載されたファイルが作成され、後で環境を再現するのが容易になります。

# virtualenv vs venv

`virtualenv`は`venv`よりも古いですが、高速な代替手段です。より多くの機能を提供し、しばしば高速ですが、`venv`はPythonに組み込まれており、ほとんどのユースケースに十分です。私は両方を使用してきましたが、`virtualenv`は高速ですが、`venv`は便利です。

`virtualenv`をインストールするには、以下のコマンドを使用します：

```bash
pip install virtualenv
```

`virtualenv`を使って仮想環境を作成するには、以下のコマンドを実行します：

```bash
virtualenv .venv
```

以下に簡単な比較を示します：

| 特徴       | venv       | virtualenv |
|---------------|------------|------------|
| Pythonに組み込まれている | はい        | いいえ         |
| スピード         | 中程度     | 高速       |
| 追加機能| 最小限    | 多い       |

# 伝統的なpipワークフロー

古いPythonのワークフローでは、パッケージを手動でインストールし、依存関係をフリーズすることが一般的でした。例えば：

```bash
pip install flask
pip freeze > requirements.txt
```

このワークフローにはいくつかの問題があります：
- ロック解決がないため、環境が一貫しない
- 環境を再現するのが難しく、プロジェクトを共有する際に問題が発生する
- 手動管理がエラーを起こしやすく、時間がかかる

これらの問題により、`Pipenv`や`Poetry`のような新しいツールが開発されました。

# Pipenv

`Pipenv`は、`pip`と`virtualenv`を一つのワークフローにまとめ、依存関係と仮想環境の管理を簡単にしてくれます。`Pipenv`をインストールするには、以下のコマンドを使用します：

```bash
pip install pipenv
```

新しいプロジェクトを作成し、依存関係をインストールするには、以下のコマンドを実行します：

```bash
pipenv install requests
```

これにより、`Pipfile`と`Pipfile.lock`が作成され、依存関係とそのバージョンがロックされます。仮想環境をアクティブにするには、以下のコマンドを使用します：

```bash
pipenv shell
```

`Pipenv`には以下のような利点があります：
- `pip`と`virtualenv`を組み合わせた簡単なワークフロー
- ロックファイルで再現可能な環境を保証
- 自動仮想環境で手動セットアップを減らす

ただし、以下のような欠点もあります：
- 他のツールに比べてパフォーマンスが低い
- 人気が低く、多くの開発者が`Poetry`や`uv`に移行している

# Poetry

`Poetry`は、最も人気のある現代的なPythonプロジェクトマネージャーの一つです。依存関係、パッケージング、公開、仮想環境の管理を一つのツールで行います。`Poetry`をインストールするには、以下のコマンドを使用します：

```bash
curl -sSL https://install.python-poetry.org | python3 -
```
