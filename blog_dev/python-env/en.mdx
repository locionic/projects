# Modern Python Development Environment Setup

Using just `pip install` often leads to version conflicts and "works on my machine" problems. Modern Python development requires **environment isolation, version locking, and build management**.

<Callout type="info" title="Standard Modern Toolkit">
- **pyenv:** Manage and switch between Python versions (3.10, 3.12...).
- **Poetry / uv:** Professional package management with auto virtual environments.
- **pyproject.toml:** Standard config file for all Python projects (PEP 621).
</Callout>

---

## Complete Setup Steps

<Stepper>
  <Step title="Step 1: Manage Python Versions with pyenv">
    Install and activate the right Python version per project.
    ```bash
    # Install and set local version for project
    pyenv install 3.12
    pyenv local 3.12
    ```
  </Step>
  
  <Step title="Step 2: Create Virtual Environment (venv)">
    Isolate your project's packages from system and other projects.
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```
  </Step>
  
  <Step title="Step 3: Manage Dependencies with Poetry">
    Replace legacy `requirements.txt` with `pyproject.toml` and lock files for absolute safety.
    ```bash
    # Initialize project and add packages
    poetry new myapp
    poetry add fastapi
    ```
  </Step>
</Stepper>
