If you spend enough time in the terminal, you'll notice a strange harmony. Despite the lack of an absolute authority, most terminal programs behave in remarkably similar ways. 

This consistency is driven by POSIX standards and historical conventions. Below are the unwritten rules of terminal programs—descriptive patterns that make command-line interfaces intuitive and predictable.

<Callout type="info" title="Descriptive, Not Prescriptive">
  These rules are based on real-world observation. While modern tools like the CLI Guidelines attempt to standardize behavior, these rules represent how things actually work in practice.
</Callout>

## The Unwritten Rules

<Stepper>
  <Step title="Rule 1: Quit on Ctrl-C (Noninteractive)">
    If a program is noninteractive (like `ping` or `curl`), pressing `Ctrl-C` should kill it immediately via a `SIGINT` signal. Interactive programs (like shells or `vim`) often catch this signal to cancel the current prompt rather than exiting.
  </Step>

  <Step title="Rule 2: 'q' Quits TUIs">
    Text-based User Interfaces (TUIs) like `less`, `top`, or `htop` almost universally use the `q` key to safely exit the application.
  </Step>

  <Step title="Rule 3: Ctrl-D Exits REPLs">
    When inside a Read-Eval-Print Loop (REPL) like Python, Node, or IRB, pressing `Ctrl-D` on an empty line sends an `EOF` (End of File) character, signaling the program to shut down gracefully.
  </Step>
</Stepper>

### Color and Display Guidelines

<Callout type="warning" title="Rule 4: Stick to 16 Colors">
  Terminal programs rarely use more than 16 colors. Relying on true color or 256 colors can break user-defined color themes and accessibility settings. Keep it simple.
</Callout>

<Callout type="idea" title="Rule 6: Disable Colors in Pipes">
  Always detect when stdout is piped (`!isatty(stdout)`). If writing to a pipe, disable color escape sequences. Otherwise, users will see corrupted characters like `\033[31m` in their log files!
</Callout>

### Keybindings & Input Conventions

**Rule 5: Vaguely Support Readline Keybindings**
Users expect standard bash/readline shortcuts to work everywhere:
*   `Ctrl-E` jumps to the end of the line.
*   `Ctrl-A` jumps to the beginning of the line.
*   `Ctrl-W` deletes the last word (matching the default "cooked mode" behavior of the OS).

**Rule 7: `-` Means Stdin/Stdout**
When a CLI tool expects a file path, passing a single hyphen (`-`) conventionally tells the program to read from Standard Input or write to Standard Output.

## Conclusion

These unwritten rules take time to absorb because they are learned through muscle memory rather than reading a manual. By understanding and implementing these conventions, developers can create tools that instantly feel right at home in any developer's workflow.
