# The Problem

On a Ubuntu system with an NVIDIA GPU (GTX 1650 SUPER), everything looked normal:

- Boot time: ~20 seconds
- Xorg session (GNOME desktop)
- NVIDIA driver installed correctly

But there was a major issue:

> After entering the login password, the system would freeze for 2–3 minutes before the desktop appeared.

---

# Initial Investigation

System boot analysis:

```bash
systemd-analyze
```

Showed:
- Boot time was normal (~20 seconds)
- Kernel and userspace were fast

User session analysis:

```bash
systemd-analyze --user blame
```

Also showed nothing abnormal.

So the delay was NOT caused by system services.

---

# The Real Clue

Kernel logs showed repeated NVIDIA DRM errors:

```
[nvidia-drm] Failed to grab modeset ownership
```

and also:

```
nvidia-gpu ... i2c timeout error
```

These indicated a GPU display initialization issue.

---

# Root Cause

The issue was caused by:

> NVIDIA DRM/KMS modesetting conflict during Xorg startup.

What was happening:

- `nvidia-drm` tried to take control of display output
- Another process already held DRM ownership
- Driver kept retrying for ~2–3 minutes
- Eventually timed out and released control
- GNOME session finally loaded

---

# What Was NOT the Problem

- Not slow boot
- Not GNOME extensions
- Not Snap packages
- Not disk performance
- Not CPU or RAM issues

---

# The Fix

Disable NVIDIA DRM modesetting.

Edit GRUB config:

```bash
sudo nano /etc/default/grub
```

Update this line:

```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvidia-drm.modeset=0"
```

Then apply changes:

```bash
sudo update-grub
sudo reboot
```

---

# Why This Fix Works

Normally NVIDIA uses DRM/KMS:

```
Kernel → nvidia-drm → Xorg → Desktop
```

But the issue caused a deadlock:

```
nvidia-drm → waiting for GPU ownership → timeout → desktop loads
```

Disabling it switches to:

```
Kernel → NVIDIA driver → Xorg → Desktop
```

This avoids the conflict completely.

---

# Result

After applying the fix:

- Login delay disappeared
- Desktop appears immediately
- No more modeset ownership errors
- System remains stable under Xorg

---

# Important Note About Resolution (1920x750)

Custom resolution still works normally:

- NVIDIA Settings GUI
- xrandr commands
- Xorg configuration

---

# Conclusion

The issue was not Ubuntu performance or hardware failure.

It was a NVIDIA DRM/KMS modeset ownership conflict during graphical startup.

---

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvidia-drm.modeset=0"
