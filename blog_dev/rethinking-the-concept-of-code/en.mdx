# Rethinking the Concept of Code

As humans increasingly rely on agents to write code, it's essential to examine the nature of code itself. Will we still have source code in the future? To address this question, we need to understand the fundamental purposes of code.

## Code's Dual Purpose

Unmesh Joshi identifies two interconnected yet distinct purposes of code:

1. **Instructions to a machine**: Code provides a set of instructions that a machine can execute to perform a specific task.
2. **Conceptual model of the problem domain**: Code represents a conceptual model of the problem domain, allowing developers to think about and solve complex problems.

These two purposes are intertwined, as the instructions to the machine are often a reflection of the conceptual model of the problem domain.

## Building a Vocabulary to Talk to Machines

To effectively communicate with machines, developers need to build a vocabulary of programming languages, frameworks, and tools. This vocabulary enables them to express their thoughts and ideas in a way that the machine can understand.

## Programming Languages as Thinking Tools

Programming languages are not just a means of instructing machines; they are also thinking tools that help developers model and solve complex problems. The choice of programming language can influence the way developers think about a problem and the solutions they develop.

## Implications for the Future

As we work with Large Language Models (LLMs) and other agents that can write code, it's essential to consider the implications for the future of software development. Will we still need source code, or will LLMs become the primary means of creating software?

### Possible Futures

There are several possible futures for code:

* **Augmented coding**: LLMs augment human coding, helping with tasks such as code completion, debugging, and testing.
* **Automated coding**: LLMs automate coding, generating entire programs or systems with minimal human input.
* **New forms of coding**: LLMs enable new forms of coding, such as visual programming or natural language programming.

Ultimately, the future of code will depend on how we choose to work with LLMs and other agents. By understanding the fundamental purposes of code, we can better navigate this future and ensure that code remains a powerful tool for solving complex problems.

### Key Takeaways

* Code has two distinct but intertwined purposes: instructions to a machine and a conceptual model of the problem domain.
* Programming languages are thinking tools that help developers model and solve complex problems.
* The future of code will depend on how we choose to work with LLMs and other agents.
* Possible futures for code include augmented coding, automated coding, and new forms of coding.
