# Claude Managed Agents on Cloudflare

Cloudflare and Anthropic announced they're integrating Claude Managed Agents with Cloudflare's sandbox environment. If you've been following the agent space, this is one of the more practical announcements I've seen — it pairs a capable model with infrastructure that actually handles the deployment and security piece.

Running agent code in production has been a pain point for a while. You either hack something together with a VPS and hope for the best, or you pay enterprise SaaS prices before you've validated the idea. This sits somewhere in between.

<Callout type="idea" title="The Concept">
  Write your agent logic on Claude's platform. The actual execution — sandboxed, observed, connected — happens on Cloudflare's edge. You get SSH access, metrics, audit trails, and the agent doesn't touch your internal network unless you explicitly allow it.
</Callout>

## What They're Providing

Three things stand out to me:

- **Sandboxes** — full Linux microVMs that start fast and don't share state. If an agent goes rogue, it's contained.
- **Browser Run** — a programmable browser the agent can drive. Useful for any workflow that involves hitting a web UI.
- **Workers** — lightweight code execution for tasks that don't need a full VM.

## How It Works in Practice

<Stepper>
  <Step title="Agent Loop on Claude">
    Your agent runs on Claude's infrastructure. When it needs to execute code, hit a web page, or access a tool, Cloudflare handles the execution environment.
  </Step>
  <Step title="Connect Without Exposing">
    Agents connect to your internal services through Cloudflare Mesh. That means you don't open ports or manage VPNs. The agent gets a secure tunnel, nothing else.
  </Step>
  <Step title="Full Observability">
    Every sandbox session is recorded. You get metrics, browser session replays, and SSH access if you need to debug something in real time.
  </Step>
</Stepper>

<Callout type="info" title="Security Model">
  Outbound traffic goes through configurable proxies. You can block specific endpoints, log all traffic, or inject credentials at the proxy layer — the agent never handles raw secrets.
</Callout>

I haven't built anything serious with this yet, but the combination addresses the two biggest hurdles I've hit with agents: "where do I run this safely" and "how do I know what it's doing." The sandbox + observability combination is what makes this interesting, not the fact that another AI model is available somewhere.
