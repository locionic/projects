## The Problem
-------------

Mess With DNS, a DNS service, was experiencing periodic out-of-memory (OOM) errors due to its IP address lookup database consuming too much memory. The goal was to reduce memory usage while maintaining performance.

## Memory Usage Breakdown
-------------------------

The system had approximately 465MB of RAM, allocated as follows:

*   PowerDNS: 100MB
*   Mess With DNS: 200MB
*   Hallpass: 40MB
*   Free memory: 110MB

The IP address lookup database used around 117MB of memory, which was too high.

## Attempt 1: Using SQLite
---------------------------

The first approach was to store the IP address database in an SQLite database on disk, reducing memory usage. However, this introduced several issues:

*   **IPv6 address storage:** SQLite doesn't support big integers, so IPv6 addresses were stored as text.
*   **Performance:** The SQLite solution was 500x slower than the original binary search implementation.

## Attempt 2: Using a Trie
---------------------------

The second approach was to use a trie data structure to store the IP address database. However, this also had issues:

*   **Memory usage:** The trie implementation used significantly more memory (800MB) than the original array-based solution.
*   **Performance:** The trie implementation was slower than the original binary search implementation.

## Memory Profiling Notes
-------------------------

Some notes on memory profiling using the `runtime` package and `pprof` tool:

*   Use `runtime.GC()` to force garbage collection and get accurate memory usage numbers.
*   Use `pprof` to analyze heap profiles and identify memory usage patterns.
*   Pass `--alloc-space` or `--inuse-space` to `pprof` to analyze allocation patterns or in-use memory, respectively.

## Attempt 3: Optimizing the Array
----------------------------------

The third approach was to optimize the original array-based implementation:

*   **Deduplicate ASN information:** Store ASN information in a separate array and use indices to reference it, reducing memory usage by 50MB.
*   **Use `netip.Addr` instead of `net.IP`:** Switch to the more memory-efficient `netip.Addr` type, reducing memory usage by an additional 20MB.

The optimized array-based implementation achieved a significant reduction in memory usage while maintaining performance.

## Conclusion
----------

By iteratively refining the IP address lookup database implementation, we were able to reduce memory usage from 117MB to 46MB, a 60% reduction. This was achieved by deduplicating ASN information and using a more memory-efficient IP address type. The optimized implementation maintains performance while reducing memory usage, making it a successful solution for Mess With DNS.
