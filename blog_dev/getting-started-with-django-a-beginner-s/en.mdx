# Getting Started with Django: A Beginner's Notes

As a fan of learning old but reliable technologies, I recently started using Django, a popular Python web framework. In this post, I'll share my initial impressions and notes on using Django for a small project.

## Less Magic than Rails

Compared to Rails, Django feels more explicit and easier to navigate. With Django, I can see exactly where things are configured, making it simpler to pick up where I left off after a long break. My small Django project consists of five main files: `urls.py`, `models.py`, `views.py`, `admin.py`, and `tests.py`, each with clear and explicit references to other components.

## A Built-in Admin Interface

Django's built-in admin interface is a significant advantage. With minimal code, I can create a customizable admin interface to manage my data. For example, I can define an admin class to display specific fields, search on certain fields, and order the data:
```python
@admin.register(Zine)
class ZineAdmin(admin.ModelAdmin):
    list_display = ["name", "publication_date", "free", "slug", "image_preview"]
    search_fields = ["name", "slug"]
    readonly_fields = ["image_preview"]
    ordering = ["-publication_date"]
```
## Fun with ORMs

Django's ORM (Object-Relational Mapping) system has won me over. I can write queries using a more Pythonic syntax, which is easier to read and write. For instance, I can use `__` to represent a JOIN:
```python
Zine.objects.exclude(product__order__email_hash=email_hash)
```
This query involves multiple tables, but Django takes care of the underlying SQL.

## Automatic Migrations

The ORM also provides automatic migrations, which is a huge time-saver. When I make changes to my models, Django generates a migration script that I can apply to my database. This feature has been a game-changer for me, especially since I'm still experimenting with my data model.

## Excellent Documentation

Django's documentation is top-notch. I've enjoyed reading the official docs, which provide clear explanations and examples. The documentation culture is intentional, as highlighted in Jacob Kaplan-Moss's talk on Django's documentation culture.

## Using SQLite

After a bad experience with Postgres, I decided to use SQLite for my small project. It's been a great choice, and I appreciate the ease of backing up my database using a single file.

## Batteries-Included Approach

Django's batteries-included approach is a significant advantage. I can easily implement features like CSRF protection, Content-Security-Policy, and email sending using built-in components. For example, I can configure email sending in my settings file:
```python
EMAIL_BACKEND = "django.core.mail.backends.filebased.EmailBackend"
EMAIL_FILE_PATH = BASE_DIR / "emails"
```
## Settings File Still Feels Overwhelming

One area where I still feel overwhelmed is the settings file. With so many global variables to configure, I worry about typos and mistakes. However, I'm getting used to it, and the benefits of using Django far outweigh the initial learning curve.

Overall, I'm excited to continue learning and using Django for my projects. If you have any favorite Django features or tips, please share them with me!
