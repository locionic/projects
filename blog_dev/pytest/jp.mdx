# 実践的 pytest チュートリアル

「テストは通っているのに、コードは壊れている」——そんな恐ろしい事態を防ぐための、真に実践的な `pytest` の使い方をまとめました。

<Callout type="idea" title="大原則">
壊れたコードではテストが「落ちる」ことを確認してください。わざと失敗させたことがないテストは、信頼してはいけません。
</Callout>

---

## 1. pytest の基本ワークフロー

<Stepper>
  <Step title="インストールと実行">
    `pip install pytest` で入れます。プロジェクトのルートで `pytest` と打てば、`test_*.py` というファイルを自動で探して実行してくれます。
  </Step>
  
  <Step title="シンプルなテストを書く">
    特別な書き方は不要です。`test_` で始まる関数を作り、普通の `assert` を使うだけ。
    ```python
    def test_addition():
        assert 1 + 1 == 2
    ```
  </Step>
  
  <Step title="特定のテストだけを実行する">
    修正中のテストだけを回して、フィードバックを高速化しましょう。
    `pytest test_math.py::test_addition`
  </Step>
</Stepper>

---

## 2. pytest の強力な機能たち

<Callout type="info" title="Fixtures: セットアップの共有">
DB接続などの「準備」をコピペせずに済みます。テスト関数に引数として渡すだけで使えます。
```python
@pytest.fixture
def empty_cart():
    return ShoppingCart()

def test_add_item(empty_cart):
    empty_cart.add("apple")
    assert len(empty_cart) == 1
```
</Callout>

<Callout type="info" title="Parametrize: 複数パターンのテスト">
似たようなテスト関数を量産するのはやめましょう。入力と期待値をリストで渡すだけで、複数パターンのテストが可能です。
```python
@pytest.mark.parametrize("a,b,expected", [(1,2,3), (-1,1,0)])
def test_add(a, b, expected):
    assert add(a, b) == expected
```
</Callout>

<Callout type="warning" title="例外（エラー）のテスト">
自前で `try/except` を書かないでください。確実にエラーが起きることをテストするには `pytest.raises` を使います。
```python
def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        divide(1, 0)
```
</Callout>
