# Automate Your Annoying Click Tasks with Python

If you’ve clicked the same sequence of buttons twice, it’s a candidate for automation. Python excels at eliminating the boring parts of your day.

<Callout type="idea" title="The Golden Rule">
Don't build massive systems. Build tiny scripts to do one small job well.
</Callout>

---

## How to Automate Safely

<Stepper>
  <Step title="1. Pick a Hated Task">
    Identify something you do manually every day: renaming files, scraping a page, resizing images, or organizing downloads.
  </Step>
  
  <Step title="2. Use the Standard Library First">
    Don't download heavy packages yet. `pathlib` (files), `shutil` (copying), `csv` (data), and `datetime` can handle 90% of your chores.
    ```python
    from pathlib import Path
    for p in Path("downloads").glob("*.csv"):
        print(f"Would rename {p.name}") # Always test with print first!
    ```
  </Step>
  
  <Step title="3. Treat Data with Respect">
    Never overwrite original data. Load the CSV, clean it (use `pandas` if it's complex), and write to a **new** output file.
  </Step>
  
  <Step title="4. Schedule and Log It">
    Use **cron** (Linux/Mac) or **Task Scheduler** (Windows) to run it automatically. Add `logging` so you know it worked when you weren't watching.
  </Step>
</Stepper>

---

## Common Traps to Avoid

<Callout type="warning" title="Don't assume perfect inputs">
Files will arrive with weird names. APIs will time out. Always add simple validation checks and handle errors gracefully instead of letting the script crash silently.
</Callout>

<Callout type="info" title="Where to Start?">
Try writing a script that monitors your "Downloads" folder and automatically moves images to a "Pictures" folder and PDFs to "Documents". Get it working, schedule it, and enjoy the saved time!
</Callout>
