## Introduction

End-to-End (E2E) testing verifies that your entire application works correctly from the user's perspective — clicking buttons, filling forms, and navigating pages. 

**Playwright** is Microsoft's open-source framework that has become the gold standard for modern web testing.

<Callout type="idea" title="Why Playwright?">
It's incredibly fast, supports cross-browser testing (Chromium, Firefox, WebKit), auto-waits for elements to appear, and natively supports TypeScript with modern `async/await` syntax!
</Callout>

## 2. Writing Your First Test

Open `tests/example.spec.ts` and replace the content:

```ts
import { test, expect } from '@playwright/test';

test('homepage loads correctly', async ({ page }) => {
  // Navigate
  await page.goto('https://www.locionic.com');

  // Verify Title
  await expect(page).toHaveTitle(/Portfolio/);

  // Verify Heading
  await expect(page.getByRole('heading', { name: /Portfolio & Blog/i })).toBeVisible();
});
```

Run it using `npx playwright test`. You should see a beautiful green checkmark! ✅

## 3. Simulating User Flows

A real E2E test checks complex flows, like logging in:

```ts
test('user can log in successfully', async ({ page }) => {
  await page.goto('/login');

  // Fill forms using accessible labels
  await page.getByLabel('Email').fill('user@example.com');
  await page.getByLabel('Password').fill('correctpassword');

  // Click submit
  await page.getByRole('button', { name: 'Log in' }).click();

  // Verify success redirect
  await expect(page).toHaveURL(/dashboard/);
});
```

<Callout type="warning" title="Selector Best Practices">
Avoid fragile CSS selectors like `.btn-primary-v2`. Instead, use user-centric locators like `page.getByRole('button', { name: 'Submit' })`. If a user can't see the element, your tests shouldn't either!
</Callout>
