# Modifying Your PATH Like a Pro

The `PATH` environment variable tells your operating system where to look for executable files. If you've ever installed a tool but got a "command not found" error, you probably need to add its folder to your `PATH`!

Here is how you do it across any shell.

<Stepper>
  <Step title="Step 1: Identify Your Shell">
    Run `ps -p $$ -o pid,comm=` to see if you are using `bash`, `zsh`, or `fish`.
  </Step>
  <Step title="Step 2: Open the Config File">
    Find your shell's configuration file:
    - **Bash:** `~/.bashrc` (or `~/.bash_profile`)
    - **Zsh:** `~/.zshrc`
    - **Fish:** `~/.config/fish/config.fish`
  </Step>
  <Step title="Step 3: Edit and Apply">
    Open the file in a text editor and add this line at the bottom:
    ```bash
    export PATH=$PATH:/path/to/new/directory
    ```
    Save it, then restart your terminal or run `source ~/.zshrc` (replace with your config file name) to apply the changes!
  </Step>
</Stepper>

---

## Common Gotchas & Troubleshooting

<Callout type="warning" title="Duplicate PATH Entries">
  If you simply run `export PATH=...` directly in your terminal multiple times, your PATH variable will grow with duplicates! Always put it in your config file and let the shell handle it on startup.
</Callout>

<Callout type="info" title="Wrong Program Running?">
  The shell searches directories from left to right. If two directories have a program with the same name, the one listed *first* in your PATH is the one that executes. Use `which <command>` to see exactly which version is running!
</Callout>
