# How to Add a Directory to Your PATH

You installed a tool, ran it, and got `command not found`. You know the binary exists somewhere on your system — `/home/user/.local/bin/my-tool` — but your shell can't find it. This happens because your shell only looks in specific directories when you type a command.

The fix is adding that directory to your `PATH` environment variable. Here's what that actually means and how to do it.

## How PATH Works

When you type `my-tool` in your terminal, your shell checks every directory listed in the `PATH` variable, in order, until it finds an executable with that name. `PATH` looks like this:

```
/home/user/.local/bin:/usr/local/bin:/usr/bin:/bin
```

Colons separate the directories. If your tool lives in `/home/user/.local/bin` and that directory isn't in the list, the shell won't find it.

You can check your current PATH with:

```bash
echo $PATH
```

And see where a specific command resolves from with:

```bash
which <command>
```

## Adding a Directory Permanently

The exact file to edit depends on your shell, but the concept is the same everywhere.

### Bash

Edit `~/.bashrc` (or `~/.bash_profile` on macOS):

```bash
export PATH="$PATH:/home/user/.local/bin"
```

Then reload with `source ~/.bashrc` or open a new terminal.

### Zsh

Edit `~/.zshrc`:

```bash
export PATH="$PATH:/home/user/.local/bin"
```

Reload with `source ~/.zshrc`.

### Fish

Fish uses a different syntax. Edit `~/.config/fish/config.fish`:

```fish
fish_add_path /home/user/.local/bin
```

Fish's `fish_add_path` is smarter than the bash version — it automatically avoids duplicates and adds the path to the front.

<Callout type="warning" title="Don't Use export in a Terminal Session">
If you run `export PATH="/some/path:$PATH"` directly in your terminal, it only lasts for that session. Every new terminal will have the old PATH. Always put it in your shell config file.
</Callout>

## Common Pitfalls

**Wrong program running** — If two directories have a program with the same name, the one listed first in your PATH wins. Check with `which <command>` to see which one is actually being used.

**Permissions** — The directory must be readable and the file must have execute permissions. Run `chmod +x /path/to/file` if needed.

**Quoting** — Always quote the export: `export PATH="$PATH:/new/path"`. Without quotes, paths with spaces break silently.

**Duplicate entries** — Running `export PATH` in multiple config files creates duplicates. Fish's `fish_add_path` handles this. For bash/zsh, check with `echo $PATH | tr ':' '\n' | sort | uniq -d` to find duplicates.
