# Alternatives to Tailwind's Color Palette
When I decided to switch from Tailwind to vanilla CSS for new projects, I missed one feature in particular: the color palette. Tailwind's pre-defined colors made it easy to choose a suitable shade for my designs. However, I wanted to explore other options beyond Tailwind's default palette.

### Favorite Color Palettes

After asking for recommendations on Mastodon, I discovered some excellent alternatives:

* **Uchū**: A carefully crafted color palette with a CSS file and FAQ available.
* **Flexoki**: A simple and elegant CSS color palette.
* **Reasonable Colors**: A palette focused on accessibility, providing a CSS file for easy implementation.

### More Color Palettes and Design Systems

In addition to my favorites, I also found the following resources:

* **Web Awesome**: A collection of web design resources, including color palettes.
* **Radix**: A design system with a built-in color palette.
* **Material Design Colors**: A set of colors from Google's Material Design system.
* **Colorscheme Generators**: Tools for generating color palettes, including Harmonizer, Tints.dev, Coolors, and Colorpalette.pro.

While I've found these generators challenging to use in the past, they might be helpful for designers looking for a more customized color palette.

### Additional Color Tools

For those interested in exploring color theory and accessibility, I recommend the following tools:

* **ColorHexa**: A website providing information on color blindness and color theory.
* **OKLCH**: A color model that can be used to dynamically generate colors with CSS. The article "Generative Colors with CSS" offers an example of how to use OKLCH in CSS.

These resources should provide a good starting point for anyone looking to improve their color game in web development.
