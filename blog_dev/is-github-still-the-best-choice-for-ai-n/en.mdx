# Is GitHub Still the Best Choice for AI-Native Development?

As AI agents generate more code and pull requests, GitHub's reliability has become a major concern. With a massive increase in infrastructure load, GitHub's availability has dropped to unacceptable levels. In this article, we'll explore the reasons behind GitHub's struggles and examine alternative solutions.

## GitHub's Reliability Issues

GitHub's reliability has been a topic of discussion among developers, with many experiencing frequent outages and degradations. According to the "Missing GitHub Status Page," a third-party website that tracks GitHub's availability, the platform has been down for around 2.5 hours daily, which is approximately 10% of the time.

The growth of AI agents like Claude Code has contributed significantly to GitHub's infrastructure load. Claude Code's website, which tracks its contributions across GitHub, shows a 6x increase in load over the past three months.

## Analysis of GitHub's Outages

GitHub's CTO, Vladimir Fedorov, addressed the availability issues in a blog post, highlighting three major incidents:

*   February 2: Security policies unintentionally blocked access to virtual machine metadata.
*   February 9: A database cluster got overloaded.
*   March 5: Writes failed on a Redis cluster.

Software engineer Lori Hochstein analyzed these outages and noted that:

*   Saturation: The database cluster incident was caused by higher-than-expected usage, which is harder to scale up than stateless services.
*   Failover + telemetry gap: The February 2 incident was a combination of an infrastructure issue in one region failing over to a healthy region, and a telemetry gap that made things worse.
*   Failover + configuration issue: The March 5 incident was similar, with a failover causing a configuration issue that blocked writes on a Redis cluster.

## Startup Offers Alternative Solution

Pierre Computer, a startup founded by Jacob Thornton, claims to have built an "AI-native" solution for AI agents pushing code, which scales far beyond what GitHub can do. Pierre's product, Code.storage, supports features like:

*   Sustained peak of > 15,000 repos per minute for 3 hours.
*   > 9M repos created in the last 30 days.

These numbers are impressive, but it's essential to note that they are self-reported, and the product is still in closed beta.

## Has GitHub Lost Focus and Purpose?

GitHub's reliability issues are acute enough that teams may start considering alternative solutions. But how did the largest Git host in the world neglect its customers and fail to prepare its infrastructure for the increase in code commits and pull requests?

Mitchell Hashimoto, founder of Ghostty, suggests that GitHub needs to:

1.  Establish a North Star plan around being critical infrastructure for agentic code lifecycles.
2.  Fire everyone working on Copilot and shut it down.
3.  Buy Pierre and launch agentic repo hosting as the first agentic product.
4.  Re-evaluate all product lines and initiatives against the new North Star.

## Conclusion

GitHub's reliability issues and lack of focus on AI-native development are causing concerns among developers. While startups like Pierre Computer are building scalable solutions for AI agents, GitHub needs to reassess its priorities and vision. With no CEO and internal politics, GitHub teams may need to take a step back and re-evaluate their strategy to remain relevant in the AI-driven development landscape.
