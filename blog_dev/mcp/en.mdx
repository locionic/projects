# Building Your First MCP Server from Scratch

Model Context Protocol (MCP) is the universal bridge between AI models and external tools. Think of it like a USB-C port for AI: one standard interface that connects Claude, ChatGPT, or Cursor to your local files and databases.

<Callout type="idea" title="Beginner Friendly!">
Building an MCP server is incredibly simple. No HTTP routing, no authentication, no frontend. Just write Python functions and decorate them. Let's build a **Local Notes Server**!
</Callout>

---

## The Architecture

<Callout type="info" title="How it works">
- **MCP Host:** The app running the AI (e.g., Claude Desktop).
- **MCP Client:** The component inside the host talking to servers.
- **MCP Server:** Your code that exposes real tools the AI can use.
</Callout>

---

## Building the Notes Server

<Stepper>
  <Step title="Step 1: Setup the Environment">
    Initialize a Python project with `uv` and install the `mcp` SDK.
    ```bash
    mkdir notes-mcp-server && cd notes-mcp-server
    uv init
    uv add mcp
    ```
  </Step>
  <Step title="Step 2: Write the Code">
    Create a `server.py` file. Use `@mcp.tool()` to expose functions to the AI.
    ```python
    from mcp.server.fastmcp import FastMCP
    mcp = FastMCP("Notes Server")

    @mcp.tool()
    def add_note(title: str, content: str) -> str:
        """Save a new note."""
        # ... logic to save file ...
        return f"Note '{title}' saved!"
    
    if __name__ == "__main__":
        mcp.run()
    ```
  </Step>
  <Step title="Step 3: Connect to Claude Desktop">
    Add your server to the `claude_desktop_config.json` file.
    ```json
    {
      "mcpServers": {
        "notes": {
          "command": "uv",
          "args": ["--directory", "/path/to/project", "run", "server.py"]
        }
      }
    }
    ```
  </Step>
  <Step title="Step 4: Test it Out">
    Restart Claude Desktop, click the plug 🔌 icon, and ask Claude:
    *"Add a note called 'ideas' with the content: learn MCP today."*
  </Step>
</Stepper>

---

<Callout type="warning" title="Crucial Tip for STDIO Servers">
Never write to `stdout` (e.g., using a plain `print()` statement). It corrupts the JSON-RPC messages and breaks the server. Always debug using `sys.stderr` or Python's `logging`.
</Callout>
