# Building Your First MCP Server from Scratch

I built my first MCP server because I was tired of copying terminal output into Claude manually. I wanted Claude to just *read the file itself*. Turns out that's about 30 lines of Python.

Model Context Protocol (MCP) is the protocol Claude Desktop uses to let AI models call local tools. If you've used Claude Code's file editing or bash tools, that's MCP under the hood. You can expose your own tools the same way.

<Callout type="info" title="What You'll Build">
A Local Notes server — Claude can save, list, and read notes from your filesystem. No database, no HTTP server, no auth boilerplate.
</Callout>

## How It Fits Together

Three pieces:

- **MCP Host** — the app running the AI (Claude Desktop, in this case).
- **MCP Client** — the protocol handler inside the host that discovers and calls your tools.
- **MCP Server** — your Python script that advertises tools and runs them on request.

The server communicates over STDIO using JSON-RPC. You don't need to think about the wire format because the SDK handles it.

## Building the Notes Server

Install the SDK and set up a Python project:

```bash
mkdir notes-mcp-server && cd notes-mcp-server
uv init
uv add mcp
```

Create a `server.py`:

```python
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Notes Server")

@mcp.tool()
def add_note(title: str, content: str) -> str:
    """Save a new note to the local filesystem."""
    filename = f"{title.replace(' ', '_')}.md"
    with open(filename, "w") as f:
        f.write(content)
    return f"Saved '{title}' as {filename}"

@mcp.tool()
def list_notes() -> str:
    """List all saved notes."""
    import glob
    notes = glob.glob("*.md")
    if not notes:
        return "No notes found."
    return "\n".join(notes)

@mcp.tool()
def read_note(title: str) -> str:
    """Read a note by filename."""
    filename = f"{title.replace(' ', '_')}.md"
    try:
        with open(filename) as f:
            return f.read()
    except FileNotFoundError:
        return f"No note found for '{title}'"

if __name__ == "__main__":
    mcp.run()
```

That's it. Three tools, about 25 lines of logic.

## Wiring It to Claude Desktop

Edit your `claude_desktop_config.json` (in `~/Library/Application Support/Claude/` on macOS, `%APPDATA%\Claude\` on Windows):

```json
{
  "mcpServers": {
    "notes": {
      "command": "uv",
      "args": ["--directory", "/full/path/to/notes-mcp-server", "run", "server.py"]
    }
  }
}
```

Restart Claude Desktop. You should see a plug icon — click it and your tools show up.

<Callout type="warning" title="STDIO Rule">
Your MCP server communicates over STDIO. Never write debug output to `print()` — that corrupts the JSON-RPC messages. Use `sys.stderr` or Python's `logging` module instead.
</Callout>

## What I'd Build Next

The notes server is a template. I've since built:

- A **bookmark manager** that saves, tags, and searches URLs via Claude
- A **project tracker** that reads/writes TODO files in a shared directory
- A **deploy tool** that runs a deploy script and reports back the output

Each one followed the same pattern: write a Python function, add `@mcp.tool()`, done. The SDK handles discovery, argument parsing, error reporting, and the JSON-RPC wire format.

If you want to see what other servers look like, Anthropic publishes reference servers on GitHub under `modelcontextprotocol/servers`. The filesystem one is a good next step.
