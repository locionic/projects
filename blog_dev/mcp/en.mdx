# Building Your First MCP Server from Scratch

If you read the Claude free plan guide and got curious about MCP, this post is the natural next step. We talked about *using* MCP servers — connecting the filesystem server, reading your documents, letting Claude browse your local files. This time, we're building one ourselves.

Don't worry if you've never written a server in your life. An MCP server is genuinely one of the more beginner-friendly things you can build. There's no HTTP routing to figure out, no authentication to implement, no frontend. You define some functions, decorate them, and Claude can call them. That's mostly it.

By the end of this post, you'll have a working MCP server that gives Claude the ability to read your local notes, add new ones, and search through them. It's a real, useful tool — not a toy "Hello World" that you throw away after the tutorial.

---

## What Is an MCP Server, Really?

Before writing any code, let's be clear about what's actually happening.

MCP (Model Context Protocol) is an open standard created by Anthropic that gives AI models a consistent, structured way to use external tools. Think of it like a USB-C port — one standardized interface that works across every device. MCP standardizes how context is passed between AI models and systems, simplifying coordination across a large language model and external data sources and tools.

The architecture has three moving parts:

- **MCP Host** — the app that runs Claude (Claude Desktop, Cursor, VS Code)
- **MCP Client** — the component inside the host that talks to servers
- **MCP Server** — your code, exposing tools Claude can call

MCP servers are lightweight servers that expose specific functionalities, connecting to local or remote data sources. This separation of concerns makes MCP servers highly modular and maintainable.

When you ask Claude "summarize my notes from last week," Claude doesn't magically know where your notes are. But if you've connected an MCP server that exposes a `read_notes` tool, Claude can call that tool, get the results back, and then actually answer your question with real data.

Unlike traditional integrations, MCP includes automatic tool discovery — the AI client can list all available tools — along with structured parameter schemas so the AI knows exactly what arguments to pass. You build the server once and every compatible AI assistant can use it without custom integration code.

As of 2026, that includes Claude, ChatGPT, Gemini, Cursor, and VS Code's Copilot. Build it once, use it everywhere.

## What We're Building

A **local notes MCP server** with three tools:

- `add_note(title, content)` — saves a new note as a text file
- `read_note(title)` — reads an existing note
- `list_notes()` — lists all your notes
- `search_notes(query)` — searches note contents for a keyword

Simple, useful, and covers all the patterns you'll use in more complex servers.

## Prerequisites

- Python 3.10 or higher (`python3 --version` to check)
- Claude Desktop installed and logged in
- `uv` installed — a fast Python package manager (install with `curl -LsSf https://astral.sh/uv/install.sh | sh`)
- About 20 minutes

---

## Step 1: Set Up the Project

Create a directory and initialize a virtual environment with `uv`:

```bash
mkdir notes-mcp-server
cd notes-mcp-server
uv init
uv add mcp
```

This creates a Python environment with the official MCP SDK installed. Create a folder where your notes will live:

```bash
mkdir notes
```

Now create the main server file:

```bash
touch server.py
```

---

## Step 2: Write the Server

Open `server.py` and add the following:

```python
import os
import sys
from pathlib import Path
from mcp.server.fastmcp import FastMCP

# Initialize the MCP server with a name
mcp = FastMCP("Notes Server")

# Where notes are stored
NOTES_DIR = Path(__file__).parent / "notes"
NOTES_DIR.mkdir(exist_ok=True)


@mcp.tool()
def list_notes() -> str:
    """List all available notes."""
    files = list(NOTES_DIR.glob("*.txt"))
    if not files:
        return "No notes found."
    return "\n".join(f.stem for f in sorted(files))


@mcp.tool()
def read_note(title: str) -> str:
    """Read the contents of a note by its title."""
    note_path = NOTES_DIR / f"{title}.txt"
    if not note_path.exists():
        return f"Note '{title}' not found."
    return note_path.read_text()


@mcp.tool()
def add_note(title: str, content: str) -> str:
    """Save a new note with the given title and content."""
    note_path = NOTES_DIR / f"{title}.txt"
    note_path.write_text(content)
    return f"Note '{title}' saved successfully."


@mcp.tool()
def search_notes(query: str) -> str:
    """Search all notes for a keyword and return matching titles."""
    results = []
    for note_file in NOTES_DIR.glob("*.txt"):
        content = note_file.read_text()
        if query.lower() in content.lower():
            results.append(note_file.stem)

    if not results:
        return f"No notes found containing '{query}'."
    return f"Notes containing '{query}':\n" + "\n".join(results)


if __name__ == "__main__":
    mcp.run()
```

That's the entire server. Let's break down what's happening:

- `FastMCP("Notes Server")` — creates the server with a name Claude will see
- `@mcp.tool()` — this decorator is the magic. Any function decorated with it becomes a tool Claude can call
- The docstrings matter — Claude reads them to understand what the tool does and when to use it. Write them like you're explaining to a smart colleague, not a machine

One critical rule: for STDIO-based servers, never write to stdout. Writing to stdout will corrupt the JSON-RPC messages and break your server. If you need to debug, always use `print(..., file=sys.stderr)` or Python's `logging` module.

## Step 3: Connect It to Claude Desktop

Now we need to tell Claude Desktop this server exists. Find your config file:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Open it and add your server:

```json
{
  "mcpServers": {
    "notes": {
      "command": "uv",
      "args": [
        "--directory",
        "/absolute/path/to/notes-mcp-server",
        "run",
        "server.py"
      ]
    }
  }
}
```

Replace `/absolute/path/to/notes-mcp-server` with the actual full path to your project folder. On macOS/Linux, run `pwd` inside the folder to get it. On Windows, use `cd` in Command Prompt.

> **Note:** Use the full path to `uv` if Claude Desktop can't find it. Run `which uv` (macOS/Linux) or `where uv` (Windows) to get it.

Save the file and **restart Claude Desktop completely** — quit and reopen, don't just close the window.

## Step 4: Test It

Once Claude Desktop restarts, you should see a small plug 🔌 icon in the chat input area, indicating active MCP tools. Click it to confirm your "Notes Server" tools are listed.

Now try these prompts:

```
Add a note called "grocery-list" with the content: eggs, milk, bread, olive oil
```

```
List all my notes
```

```
Read my grocery-list note
```

```
Search my notes for the word "milk"
```

Claude will call your tools, get real results back from your filesystem, and respond with actual data. If you check the `notes/` folder in your project, you'll find the text files Claude created.

That's a working MCP server. You just gave Claude a real capability that didn't exist before.

## Step 5: Debugging When Things Go Wrong

It won't always work first try. Here's where to look:

**Check the Claude Desktop logs:**

- macOS: `~/Library/Logs/Claude/`
- Windows: `%APPDATA%\Claude\logs\`

These logs show exactly what Claude Desktop tried to do with your server and any errors that came back.

**Test the server directly** before connecting to Claude. Run it from your terminal:

```bash
uv run server.py
```

If it starts without error, the server itself is fine and the problem is likely in the config file path or the `uv` executable location.

**Common issues:**
- Relative path in the config instead of absolute path
- Forgot to restart Claude Desktop after editing the config
- `uv` not found — use the full path from `which uv`
- Python version below 3.10

## Going Further: What Else Can You Build?

The notes server is simple by design, but the pattern scales to almost anything:

**A bookmark manager** — `add_bookmark(url, tags)`, `search_bookmarks(tag)`, `list_bookmarks()`. Claude becomes your personal search engine across your saved links.

**A journal tool** — `write_entry(content)` auto-stamps with today's date, `read_week()` returns the last 7 days. Ask Claude to summarize your week from your own words.

**A project tracker** — `create_task(name, priority)`, `complete_task(name)`, `list_open_tasks()`. A minimal task manager where Claude can update your todo list directly in conversation.

**API wrappers** — anything with a REST API can become an MCP tool. Weather, GitHub issues, your company's internal API. As of early 2026, OpenAI added MCP support to ChatGPT and the OpenAI API, meaning your MCP server works with ChatGPT, Claude, Gemini, and other compatible clients without modification. The protocol is client-agnostic by design.

The more tools you build, the more Claude starts to feel less like a chatbot and more like an assistant that actually knows your world.
