React isn't hard — but it's easy to misunderstand. This guide goes from real questions to real answers, focusing on the *why*, not just the *how*.

## What is React?

React is a JavaScript library for building user interfaces using a **component** model. 

<Callout type="idea" title="Component Model">
Think of components like LEGO blocks. You build small, reusable pieces and snap them together to create complex UIs!
</Callout>

```jsx
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}
```

---

## State Management with `useState`

`useState` stores data in the **browser's RAM**.

<Callout type="warning" title="Memory Address Comparison">
React compares state **by memory address**. Mutating an object directly doesn't change its address, so React won't re-render! Always copy objects/arrays.
</Callout>

<Stepper>
  <Step title="The Wrong Way">
    Calling `setCount(count + 1)` multiple times in a row uses a stale snapshot.
    ```jsx
    setCount(count + 1); // ❌
    ```
  </Step>
  <Step title="The Right Way">
    Use the callback form to ensure you're working with the latest value!
    ```jsx
    setCount(prev => prev + 1); // ✅
    ```
  </Step>
</Stepper>

## Side Effects with `useEffect`

Side effects are things like fetching data or setting timers. 

<Callout type="danger" title="The Cleanup Function">
The `return () => ...` is NOT sequential code. You are handing a function to React to execute later (when dependencies change or the component unmounts).
</Callout>

```jsx
useEffect(() => {
  const timer = setTimeout(() => console.log('Ping!'), 1000);
  return () => clearTimeout(timer); // 🧹 Cleanup
}, []);
```

## Avoiding Prop Drilling with `useContext`

Tired of passing props down 5 levels? Context is your friend. 

<Callout type="info" title="Watch out for Re-renders!">
Every component consuming the context re-renders when *any* value inside it changes. For complex global state, consider Zustand instead!
</Callout>

## Next.js App Router: The Modern Way

Next.js flips the script: by default, everything is a **Server Component**.

<Stepper>
  <Step title="Server Components">
    Run on the server. Great for initial data fetching and SEO. No `useState` or `useEffect`.
  </Step>
  <Step title="Client Components">
    Add `"use client"` at the top. Used for interactivity, event listeners, and hooks.
  </Step>
</Stepper>

<Callout type="idea" title="The Decision Tree">
- Local UI state? `useState`
- App-wide state? `Zustand`
- Data fetching & caching? `TanStack Query`
- SEO & static data? `Next.js Server Components`
</Callout>
