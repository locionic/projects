React không khó — nhưng dễ hiểu sai. Bài viết này đi từ câu hỏi thực tế đến câu trả lời thực tế, giải thích *tại sao* thay vì chỉ nói *cách làm*.

## React là gì?

React là thư viện JavaScript do Meta phát triển, dùng để xây dựng giao diện người dùng theo mô hình **component** — chia giao diện thành các mảnh nhỏ, tái sử dụng được.

```jsx
function ChaoMung({ ten }) {
  return <h1>Xin chào, {ten}!</h1>;
}
```

## useState — Lưu trạng thái

`useState` lưu data trong **RAM của trình duyệt**, trong vùng React tự quản lý. Mỗi component có một danh sách slot riêng — React dùng thứ tự gọi hook để biết slot nào thuộc về cái gì.

```jsx
const [dem, setDem] = useState(0);
```

### Tại sao phải dùng callback `prev => prev + 1`?

Vì `setState` không cập nhật ngay — React xử lý theo từng "đợt". Nếu gọi nhiều lần liên tiếp:

```jsx
// ❌ Gọi 3 lần → chỉ cộng được 1 vì mỗi lần đều dùng snapshot cũ
setDem(dem + 1);
setDem(dem + 1);
setDem(dem + 1);

// ✅ Gọi 3 lần → cộng đúng 3 vì prev là giá trị mới nhất thực tế
setDem(prev => prev + 1);
setDem(prev => prev + 1);
setDem(prev => prev + 1);
```

> Nếu chỉ gọi 1 lần, `setDem(dem + 1)` và `setDem(prev => prev + 1)` cho kết quả như nhau.

### Tại sao phải copy object/array?

React so sánh state **theo địa chỉ ô nhớ**, không phải nội dung bên trong. Sửa trực tiếp không tạo địa chỉ mới → React không biết có thay đổi → không re-render.

```jsx
// ❌ Địa chỉ không đổi → không re-render
nguoi.tuoi = 21;
setNguoi(nguoi);

// ✅ Tạo object MỚI với địa chỉ mới → React thấy thay đổi → re-render
setNguoi({ ...nguoi, tuoi: 21 });
```

### useState lưu ở đâu?

Trong RAM — mất khi reload, mất khi chuyển trang (nếu component bị unmount), mất khi đóng tab. Nếu cần lưu lâu hơn, dùng:

| Nơi lưu | Qua Back | Reload | Share link |
|---|---|---|---|
| `useState` trong component | ❌ | ❌ | ❌ |
| URL params | ✅ | ✅ | ✅ |
| `localStorage` | ✅ | ✅ | ❌ |

## useEffect — Chạy code ngoài luồng render

Side effects là những việc ngoài việc render UI: gọi API, đặt timer, lắng nghe sự kiện...

```jsx
useEffect(() => {
  // code chạy ở đây

  return () => {
    // cleanup — React GIỮ hàm này, không chạy ngay
    // Chỉ chạy khi: dependency thay đổi, hoặc component unmount
  };
}, [dependencies]);
```

### `return () => ...` không phải code chạy tuần tự

Đây là điểm hay bị hiểu nhầm nhất. `return` ở đây **trao hàm cho React**, không gọi ngay. React tự quyết định khi nào gọi nó.

```jsx
useEffect(() => {
  const timer = setTimeout(() => {
    setDebouncedValue(value); // ← callback của setTimeout, React không quan tâm
  }, 300);

  return () => clearTimeout(timer); // ← đưa hàm cho React cất vào túi
}, [value]);
// Cleanup chạy khi value thay đổi — không phải khi setTimeout xong
```

### 3 dạng phổ biến

```jsx
useEffect(() => { ... }, []);        // 1 lần khi mount
useEffect(() => { ... }, [x]);       // mỗi khi x thay đổi
useEffect(() => { ... });            // mỗi lần render (ít dùng)
```

## useRef — Lưu giá trị không gây re-render

Hai công dụng chính:

```jsx
// 1. Truy cập DOM trực tiếp
const inputRef = useRef(null);
inputRef.current.focus();

// 2. Lưu giá trị nội bộ (timer ID, đếm render...)
const timerRef = useRef(null);
timerRef.current = setInterval(...);
clearInterval(timerRef.current);
```

| | `useState` | `useRef` |
|---|---|---|
| Gây re-render | ✅ | ❌ |
| Dùng khi | Data hiển thị ra UI | Data nội bộ, không hiển thị |

## useContext — Tránh prop drilling

Khi phải truyền props qua nhiều tầng component không dùng đến nó:

```jsx
// Tạo context
const UserContext = createContext(null);

// Bọc app
<UserContext.Provider value={{ user, setUser }}>
  <App />
</UserContext.Provider>

// Lấy data ở bất kỳ đâu — không cần truyền props
const { user } = useContext(UserContext);
```

**Điểm yếu:** Mọi component dùng context đều re-render khi bất kỳ giá trị nào trong context thay đổi. Giải pháp: tách context nhỏ, hoặc dùng Zustand.

## useMemo và useCallback — Tối ưu performance

```jsx
// Cache kết quả tính toán nặng
const ketQua = useMemo(() => {
  return danhSach.filter(x => x.ten.includes(query));
}, [danhSach, query]);

// Cache hàm truyền xuống component con (dùng kèm React.memo)
const handleClick = useCallback(() => {
  setDem(prev => prev + 1);
}, []);
```

> **Đừng lạm dụng.** useMemo và useCallback bản thân cũng tốn bộ nhớ để cache. Chỉ dùng khi có vấn đề performance thực sự.

## Custom Hook — Tái sử dụng logic

Một hàm bắt đầu bằng `use`, bên trong dùng các hook có sẵn:

```jsx
// hooks/useFetch.js
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(data => { setData(data); setLoading(false); })
      .catch(err => { setError(err.message); setLoading(false); });
  }, [url]);

  return { data, loading, error };
}
```

```jsx
// hooks/useDebounce.js
function useDebounce(value, delay = 300) {
  const [debounced, setDebounced] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer); // hủy timer cũ khi value thay đổi
  }, [value, delay]);

  return debounced;
}
```

## Zustand — State Management đơn giản

Giải quyết điểm yếu của `useContext` — chỉ re-render component lấy đúng phần data thay đổi.

```jsx
// store/useCounterStore.js
import { create } from 'zustand';

const useCounterStore = create((set) => ({
  dem: 0,
  tang: () => set(state => ({ dem: state.dem + 1 })),
  reset: () => set({ dem: 0 }),
}));
```

```jsx
// Dùng ở bất kỳ đâu, không cần Provider
const dem = useCounterStore(s => s.dem);       // chỉ re-render khi dem đổi
const tang = useCounterStore(s => s.tang);     // không bao giờ re-render
```

## TanStack Query — Fetch data chuyên nghiệp

Thay vì viết lại loading/error state mỗi lần, TanStack Query xử lý tất cả:

```jsx
const { data, isLoading, error, isFetching } = useQuery({
  queryKey: ['posts', page],      // cache theo key — thay đổi → tự fetch lại
  queryFn: () => fetch(`/api/posts?page=${page}`).then(r => r.json()),
});
```

Tự động: cache kết quả, retry khi lỗi, refetch khi focus tab.

```jsx
// Thay đổi data (POST/PUT/DELETE)
const mutation = useMutation({
  mutationFn: (data) => fetch('/api/posts', { method: 'POST', body: JSON.stringify(data) }),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ['posts'] }),
});
```

| | `isLoading` | `isFetching` |
|---|---|---|
| Lần đầu chưa có data | ✅ | ✅ |
| Refetch (đã có data cũ) | ❌ | ✅ |

## Next.js App Router

Mặc định mọi component đều là **Server Component** — chạy trên server, fetch data thẳng, không cần `useEffect`.

```jsx
// Server Component — không cần "use client"
export default async function BlogPage({ searchParams }) {
  const page = Number(searchParams.page) || 1;
  const posts = await fetch(`/api/posts?page=${page}`).then(r => r.json());
  return <PostList posts={posts} />;
}
```

Muốn dùng `useState`, `useEffect`, `onClick` → bắt buộc thêm `"use client"`:

```jsx
"use client";
import { useState } from 'react';

export default function SearchBar() {
  const [query, setQuery] = useState('');
  return <input onChange={e => setQuery(e.target.value)} />;
}
```

### Pattern chuẩn cho search + pagination

```
Client Component (SearchBar, Pagination)
  → nhận input → đẩy lên URL (?q=nextjs&page=2)

Server Component (page.jsx)
  → đọc URL → fetch data → render kết quả
```

Next.js chỉ re-render **phần thay đổi**, không reload toàn trang — trải nghiệm mượt như SPA nhưng data luôn fresh từ server.

## Khi nào dùng cái gì?

| Tình huống | Giải pháp |
|---|---|
| Data thay đổi, cần hiện ra UI | `useState` |
| Fetch API, timer, event | `useEffect` |
| Truy cập DOM, lưu giá trị nội bộ | `useRef` |
| Chia sẻ state, tránh prop drilling | `useContext` hoặc Zustand |
| Tính toán nặng, cache hàm | `useMemo`, `useCallback` |
| Tái sử dụng logic | Custom Hook |
| State toàn app (user, cart) | Zustand |
| Fetch data có cache, refetch | TanStack Query |
| Fetch data tĩnh, SEO | Next.js Server Component |
| Search, pagination giữ qua reload | URL params |
