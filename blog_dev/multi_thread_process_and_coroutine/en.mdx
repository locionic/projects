# Python Concurrency: Threads, Processes, and Coroutines

<Callout type="info" title="The Core Issue: The GIL">
The **Global Interpreter Lock (GIL)** allows only one thread to run at any given moment to protect Python's memory. Even with 8 cores, Python defaults to using just one! Here are 3 ways to bypass this limit.
</Callout>

---

## 1. Threading

<Callout type="idea" title="Best for I/O Tasks">
When a thread waits for I/O (like an API call or DB query), it releases the GIL. That's when other threads can jump in and run!
</Callout>

<Stepper>
  <Step title="How it works">Creates multiple threads in a single process. They share memory and the same GIL.</Step>
  <Step title="When to use">Perfect for network requests, disk operations, or waiting on databases.</Step>
  <Step title="When to avoid">Don't use for heavy math or CPU-bound tasks (the GIL will block true parallelism).</Step>
</Stepper>

---

## 2. Multiprocessing

<Callout type="warning" title="True Parallelism">
Multiprocessing creates **multiple independent processes**, each with its own Python interpreter, memory, and GIL. This achieves true parallelism across CPU cores!
</Callout>

<Stepper>
  <Step title="How it works">Spawns independent processes that run simultaneously on different cores.</Step>
  <Step title="When to use">Heavy computation, image processing, machine learning, and data crunching.</Step>
  <Step title="The tradeoff">Slower startup time and much higher memory usage (no shared memory).</Step>
</Stepper>

---

## 3. Coroutines (asyncio)

<Callout type="idea" title="The Modern Way">
Coroutines are incredibly lightweight. Instead of the OS switching threads, your code explicitly says: *"I'm waiting, go run something else"* using `await`.
</Callout>

<Stepper>
  <Step title="How it works">A single thread uses an event loop to jump between tasks whenever one yields control.</Step>
  <Step title="When to use">High-concurrency I/O like WebSockets or modern web frameworks (FastAPI).</Step>
  <Step title="The advantage">Creating 10,000 coroutines is easy and efficient, unlike 10,000 threads which would crash the system.</Step>
</Stepper>

---

## Summary: Which One Should You Choose?

<Callout type="info" title="Quick Decision Tree">
- **Thousands of concurrent I/O connections?** → Coroutine (`asyncio`)
- **A few dozen simple I/O tasks?** → Threading
- **Continuous heavy computation?** → Multiprocessing
</Callout>

If you're unsure, `concurrent.futures` is a great unified API for both threading and multiprocessing!
