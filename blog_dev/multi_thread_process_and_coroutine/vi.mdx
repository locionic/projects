# Tối Ưu Hóa Python: Threading, Multiprocessing và Coroutines

<Callout type="info" title="Rào cản lớn nhất: GIL">
**Global Interpreter Lock (GIL)** chỉ cho phép một thread chạy tại một thời điểm để bảo vệ bộ nhớ. Dù máy có 8 lõi, Python mặc định chỉ xài 1! Dưới đây là 3 cách vượt rào.
</Callout>

---

## 1. Threading

<Callout type="idea" title="Hoàn hảo cho I/O">
Mỗi khi thread phải chờ I/O (gọi API, đọc file), nó sẽ tự nhả GIL để thread khác chạy chen vào.
</Callout>

<Stepper>
  <Step title="Cách hoạt động">Chạy nhiều thread trong cùng một process, chia sẻ bộ nhớ và chung GIL.</Step>
  <Step title="Khi nào nên dùng">Gửi network request, đọc/ghi disk, truy vấn DB.</Step>
  <Step title="Tránh dùng khi">Các tác vụ tính toán nặng (CPU-bound) vì GIL sẽ cản trở tốc độ.</Step>
</Stepper>

---

## 2. Multiprocessing

<Callout type="warning" title="Chạy Song Song Đích Thực">
Tạo ra các process hoàn toàn độc lập! Mỗi process có bộ nhớ riêng, GIL riêng và chạy trên một CPU core riêng biệt.
</Callout>

<Stepper>
  <Step title="Cách hoạt động">Nhân bản thành nhiều process chia ra các lõi CPU xử lý cùng lúc.</Step>
  <Step title="Khi nào nên dùng">Render video, xử lý AI, nén dữ liệu, tính toán toán học nặng.</Step>
  <Step title="Đánh đổi">Khởi động tốn thời gian, ngốn RAM nhiều hơn do không dùng chung bộ nhớ.</Step>
</Stepper>

---

## 3. Coroutines (asyncio)

<Callout type="idea" title="Giải pháp thời thượng">
Coroutine siêu nhẹ. Thay vì để hệ điều hành quyết định, code của bạn sẽ tự lên tiếng: *"Tui đang phải chờ, chạy task khác đi"* bằng lệnh `await`.
</Callout>

<Stepper>
  <Step title="Cách hoạt động">Chỉ dùng duy nhất 1 thread và 1 event loop để xoay vòng các task.</Step>
  <Step title="Khi nào nên dùng">Làm WebSockets, xây dựng API backend siêu tốc (như FastAPI).</Step>
  <Step title="Lợi thế vượt trội">Có thể chạy 10,000 coroutines cực mượt, trong khi 10,000 threads sẽ làm treo máy.</Step>
</Stepper>

## Tóm lại: Chọn Cái Nào?

<Callout type="info" title="Công thức chọn siêu tốc">
- **Chờ mạng/DB với số lượng kết nối khổng lồ?** → Coroutine (`asyncio`)
- **Chờ vài chục task I/O đơn giản?** → Threading
- **Tính toán cày ải CPU liên tục?** → Multiprocessing
</Callout>
