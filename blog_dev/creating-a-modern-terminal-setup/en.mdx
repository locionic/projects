# Creating a Modern Terminal Setup

I've been tweaking my terminal for years. There were periods where I spent more time configuring my prompt than actually writing code — the classic trap. But after enough iterations, I landed on a setup that's fast, useful, and doesn't require me to remember arcane config syntax every time I touch a new machine.

Here's what actually matters.

<Callout type="idea" title="Why Bother?">
  A good terminal setup saves you a few seconds dozens of times a day. That adds up fast. It also catches mistakes — a prompt that shows your Git branch and Python env prevents "wrong branch, wrong venv" headaches before they happen.
</Callout>

## The Foundation

**24-bit color.** If your terminal still looks like a 1980s terminal, fix this first. Most modern emulators support it out of the box, and the difference is noticeable the moment you open a colored diff or a lint output.

**Infinite scrollback.** Nothing worse than a complex command scrolling off-screen right when you need to reference it. Set your scrollback to unlimited or a very large number.

**Autocomplete.** Fish shell does this natively. With Zsh you need a plugin. Either way, once you get used to tab completing git branches and file paths, going back to a bare shell feels broken.

<Stepper>
  <Step title="Pick an Emulator">
    **iTerm2** on macOS, **Kitty** or **GNOME Terminal** on Linux. GPU acceleration matters more than you'd think — it keeps the terminal responsive when you're running something that spews logs at high speed.
  </Step>
  <Step title="Swap Your Shell">
    Default bash is fine but limited. **Zsh** with a framework or **Fish** out of the box gives you autocomplete, theming, and plugin management without fighting .bashrc conventions from 1999.
  </Step>
  <Step title="Set Up a Prompt">
    **Starship** is my pick. It works across shells, shows Git status, Python version, Node version, and it's fast. No perceptible delay on render.
  </Step>
</Stepper>

## Where People Get Stuck

The dotfile rabbit hole is real. You'll find a cool prompt theme, then a plugin that depends on another plugin, and suddenly you're reading a 200-line .zshrc trying to figure out why `git status` takes two seconds.

**My rule:** if a terminal customization takes more than 10 minutes to configure, it better save me 10 minutes a week or it gets cut. By that standard, most prompt themes and half the popular Zsh plugins don't make the cut.

### What Actually Survived

1. A minimal Starship prompt showing Git branch, Python version, and exit code.
2. `zsh-autosuggestions` — grays out recently used commands as you type. Saves more keystrokes than anything else I've installed.
3. `zsh-syntax-highlighting` — red text for invalid commands before you press enter. Catches typos constantly.
4. A handful of aliases that I actually use. Not the 50-line alias dump from a blog post I copied once and never edited.

Your setup should reflect how you actually work, not how some blog post said you should work. Start with the three things above, use them for a week, then decide what's missing.
