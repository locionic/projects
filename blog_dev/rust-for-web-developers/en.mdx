# Why I'm Learning Rust as a Web Developer

I spent six months avoiding Rust. "It's for kernel developers," I told myself. "Too hard for web stuff." Then I needed a CLI tool that was faster than Python, and I got curious.

Two months later, I'm writing Rust regularly. Not because I switched careers. Because knowing Rust has made me a better JavaScript and Python developer.

## What Rust Actually Teaches

The borrow checker—the thing everyone warns you about—isn't punishment. It's a teacher.

When you write JavaScript, you don't think about who owns a variable, when it gets freed, or if two parts of your code might mutate it simultaneously. Rust forces you to think about this. JavaScript hides memory management. Rust makes it explicit.

<Callout type="info">
You don't need to write production Rust. Learning the concepts will change how you write JavaScript, Python, Go—anything with mutable state.
</Callout>

## The Mental Model Shift

In JS, you write:
```javascript
let data = fetchData();
process(data);
doSomethingElse(data);
```

In Rust, you think about ownership:
```rust
let data = fetch_data();
process(&data);
do_something_else(&data);
```

Once this clicks, you start writing JavaScript differently. You realize that object you're mutating in place? Could be a const with a transformed copy instead.

## Where Rust Shows Up in Web Dev

**WebAssembly**: Rust compiles to WASM. Fast performance-critical web code.

**CLI Tools**: Build single-binary tools with Clap. No runtime dependencies.

**Edge Functions**: Cloudflare Workers support Rust. Better performance, lower cold starts.
