Building a personal portfolio is one of the most rewarding side projects a developer can take on. It's a playground for experimenting, sharing knowledge, and establishing your digital presence.

Here's how I built **locionic.com** — focusing on a blazing-fast, lightweight setup.

## The Tech Stack

I wanted a modern feel without the overhead of a database or traditional CMS.

<Stepper>
  <Step title="Next.js 14 (App Router)">
    Next.js provides the perfect balance of static generation and dynamic React Server Components. It ships less JavaScript to the client, keeping load times near-instant.
  </Step>
  <Step title="Tailwind CSS">
    Utility-first styling allows for rapid UI development and beautiful dark modes without leaving my TSX files.
  </Step>
  <Step title="Contentlayer">
    My secret weapon. It transforms local `.mdx` files into type-safe JSON data that my Next.js components can consume directly.
  </Step>
</Stepper>

---

## The Magic of Contentlayer

Writing a blog post is as simple as creating a new markdown file. 

<Callout type="info" title="Type-Safe Frontmatter">
  Contentlayer validates the `.mdx` frontmatter schema. If I forget a required field like `date` or `title`, the build fails immediately with a clear error! No runtime surprises.
</Callout>

Example Frontmatter:
```mdx
---
title: How I Built My Portfolio with Next.js
date: '2026-03-14'
tags: ['nextjs', 'portfolio']
category: Tech
---
```

---

## Monorepo Architecture

As the project expanded, I wanted a separate, single-page portfolio layout alongside my blog without duplicating code.

<Callout type="warning" title="Don't Duplicate UI!">
  I extracted common components (Headers, Footers, Buttons) into a shared local package `@locionic/shared-ui`. This allows both the blog and portfolio to consume the exact same design system seamlessly!
</Callout>

### Conclusion

Building this portfolio was a fantastic learning experience. The stack is modern, incredibly fast, and free to host on Vercel. 

My advice? **Just start.** A simple one-page site is infinitely better than a perfect site that doesn't exist yet!
