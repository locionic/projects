Building a personal portfolio is one of those projects that's both fun and useful. You get to experiment, share what you know, and carve out a spot on the web that's yours.

Here's how I built **locionic.com** — I wanted something fast and lightweight, and this is what I landed on.

## The Tech Stack

I didn't want a database or a traditional CMS. Just a modern setup that stays out of my way.

<Stepper>
  <Step title="Next.js 14 (App Router)">
    Next.js strikes a good balance between static generation and dynamic React Server Components. Less JavaScript shipped to the browser, faster load times.
  </Step>
  <Step title="Tailwind CSS">
    Utility-first CSS makes it quick to build UI and toggle dark mode without jumping out of my TSX files.
  </Step>
  <Step title="Contentlayer">
    Contentlayer takes `.mdx` files and turns them into type-safe JSON that my Next.js components can consume directly. It catches missing frontmatter fields at build time.
  </Step>
</Stepper>

---

## Contentlayer in Practice

Writing a blog post is creating a markdown file with frontmatter. That's it. Contentlayer validates the schema — if I forget `date` or `title`, the build fails with a clear error instead of silently using undefined at runtime.

<Callout type="info" title="Type-Safe Frontmatter">
  Contentlayer checks the `.mdx` frontmatter schema at build time. Forget a required field like `date` or `title` and the build fails with a clear error — no runtime surprises.
</Callout>

Example Frontmatter:
```mdx
---
title: How I Built My Portfolio with Next.js
date: '2026-03-14'
tags: ['nextjs', 'portfolio']
category: Tech
---
```

## Monorepo Architecture

As the project grew, I wanted a separate portfolio layout alongside the blog without duplicating code.

<Callout type="warning" title="Don't Duplicate UI!">
  I pulled shared components (Headers, Footers, Buttons) into a local package called `@locionic/shared-ui`. Now both the blog and portfolio use the exact same design system.
</Callout>

### Conclusion

Building this portfolio taught me a lot. The stack is modern, fast, and free to host on Vercel.

My advice? **Just start.** A simple one-page site beats a perfect one that never ships.
