## Introduction

Building a personal portfolio is one of the most rewarding side projects a developer can take on. It's a place to showcase your work, share your knowledge, and establish your online presence.

In this post, I'll walk you through how I built **locionic.com** — the site you're reading right now — including the tech stack, key decisions, and lessons learned along the way.

---

## The Goals

Before starting, I defined what I wanted:

- A **fast, lightweight** site with great performance
- A **blog** where I can write about coding and automation
- A **portfolio** to showcase my projects
- **Easy to update** — writing posts in Markdown, not a CMS
- **Free to host** — no monthly fees

---

## The Tech Stack

| Tool | Purpose |
|------|---------|
| Next.js 14 | React framework with App Router |
| Tailwind CSS | Utility-first styling |
| Contentlayer | MDX blog post management |
| Vercel | Hosting and deployment |
| TypeScript | Type safety |

---

## Why Next.js?

Next.js was the obvious choice because:

- **App Router** with React Server Components for fast performance
- **Static generation** for blog posts — blazing fast page loads
- **Built-in image optimization** with `next/image`
- **Great developer experience** with fast refresh and TypeScript support
- **Vercel integration** — push to GitHub, auto-deploy in seconds

---

## Why Contentlayer?

Managing blog posts with a database felt overkill. **Contentlayer** lets me write posts as `.mdx` files in my repository. It:

- Converts MDX files into type-safe JavaScript objects
- Validates frontmatter (title, date, tags, etc.)
- Works perfectly with Next.js App Router

Writing a new blog post is as simple as creating a new `.mdx` file and pushing to GitHub.

## Key Features

### MDX Blog Posts

Each blog post is an MDX file with frontmatter:
```mdx
---
title: My Blog Post
date: '2026-01-01'
tags: ['nextjs', 'testing']
draft: false
summary: A short description.
---

## Content here...
```

Contentlayer automatically picks up new posts, validates the schema, and makes them available as typed objects in my components.

### Dark Mode

Tailwind CSS makes dark mode trivial:
```tsx
<div className="bg-white dark:bg-gray-900 text-black dark:text-white">
  Content
</div>
```

### Syntax Highlighting

Code blocks in blog posts are automatically highlighted using **rehype-pretty-code** — great for technical posts.

## Deployment with Vercel

Deployment is the easiest part:

1. Push code to GitHub
2. Vercel detects the push
3. Builds and deploys in ~30 seconds
4. Live at `locionic.com`

Zero configuration needed. Vercel handles SSL, CDN, and performance optimizations automatically.

---

## Lessons Learned

**Start simple** — I was tempted to add many features upfront. Starting with just a home page, blog, and portfolio kept things manageable.

**Write content early** — the site looks empty without posts. Writing content early motivated me to keep improving the design.

**Performance matters** — running Lighthouse audits regularly kept me honest about performance. Next.js makes it easy to score 90+ with default settings.

**Your portfolio is never done** — treat it as a living project. Add features, write posts, and improve it over time.

---

## Conclusion

Building this portfolio with Next.js, Tailwind CSS, Contentlayer, and Vercel was a genuinely enjoyable experience. The stack is modern, fast, and free to run.

If you're thinking about building your own portfolio, my advice is: **start today**. A simple site with one page and one blog post is infinitely better than a perfect site that doesn't exist yet.
