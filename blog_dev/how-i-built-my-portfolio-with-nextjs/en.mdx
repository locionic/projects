Building a personal portfolio is one of those projects that's both fun and useful. You get to experiment, share what you know, and carve out a spot on the web that's actually yours.

Here's how I built **locionic.com**. I wanted something fast and lightweight. This is what I landed on.

## The Tech Stack

I didn't want a database. Didn't want a traditional CMS either. Just a modern setup that stays out of my way.

<Stepper>
  <Step title="Next.js 14 (App Router)">
    Next.js strikes a good balance between static generation and dynamic React Server Components. Less JavaScript shipped to the browser. Faster load times.
  </Step>
  <Step title="Tailwind CSS">
    Utility-first CSS makes it quick to build UI and toggle dark mode without jumping out of my TSX files.
  </Step>
  <Step title="Contentlayer">
    Contentlayer takes `.mdx` files and turns them into type-safe JSON that my Next.js components consume directly. It catches missing frontmatter fields at build time.
  </Step>
</Stepper>

---

## Why Contentlayer Changed Everything

Writing a blog post is creating a markdown file with frontmatter. That's it.

Contentlayer validates the schema. If I forget `date` or `title`, the build fails with a clear error instead of silently using undefined at runtime. No more "wait, why is this page blank?" moments at 2 AM.

<Callout type="info" title="Type-Safe Frontmatter">
  Contentlayer checks the `.mdx` frontmatter schema at build time. Forget a required field like `date` or `title` and the build fails with a clear error — no runtime surprises.
</Callout>

Example frontmatter:
```mdx
---
title: How I Built My Portfolio with Next.js
date: '2026-03-14'
tags: ['nextjs', 'portfolio']
category: Tech
---
```

## The Monorepo Decision

As the project grew, I wanted a separate portfolio layout alongside the blog without duplicating code.

<Callout type="warning" title="Don't Duplicate UI!">
  I pulled shared components (Headers, Footers, Buttons) into a local package called `@locionic/shared-ui`. Now both the blog and portfolio use the exact same design system.
</Callout>

This was the single best architectural decision I made. One change to a button component updates everywhere. No more "which repo has the latest version?" headaches.

## Hosting on Vercel: Free and Done

I deploy to Vercel. Zero config. Zero cost. The Edge Network handles caching automatically. My Lighthouse scores stay in the 90s without me thinking about it.

Is it perfect? No. But it's *done*. And done beats perfect every time.

## What I'd Do Differently

If I started over today, I'd probably reach for Astro instead of Next.js for the static parts. Less JavaScript. Simpler mental model. But Next.js 14's App Router is solid and I'm productive in it. No regrets.

The stack works. It's modern, fast, and free to host.

My advice? **Just start.** A simple one-page site beats a perfect one that never ships.
