Mythos Preview is a significant step forward in the field of LLMs. Its ability to construct exploit chains and generate proofs is unparalleled. Unlike previous models, Mythos Preview can take multiple vulnerabilities and reason about how to combine them into a working exploit. This capability is a game-changer in the field of vulnerability research.

Key Features of Mythos Preview
-----------------------------

### Exploit Chain Construction

Mythos Preview can construct exploit chains by combining multiple vulnerabilities. This feature is crucial in identifying potential attack vectors that may not be immediately apparent.

### Proof Generation

Mythos Preview can generate proofs for identified vulnerabilities. This feature allows us to verify the existence of a vulnerability and understand its potential impact.

The Signal-to-Noise Problem
---------------------------

One of the significant challenges in vulnerability research is distinguishing between real and false positives. Mythos Preview has made significant strides in reducing noise, but there is still room for improvement.

Why Generic Coding Agents Don't Work
--------------------------------------

We initially attempted to use generic coding agents to discover vulnerabilities. However, this approach proved ineffective due to the lack of context and throughput.

Building a Harness for Mythos Preview
--------------------------------------

To effectively utilize Mythos Preview, we built a harness that manages the overall execution of the model. This harness consists of multiple stages, each designed to address specific challenges in vulnerability research.

### Recon Stage

The Recon stage involves an agent reading the repository from top to bottom, generating an architecture document, and producing an initial queue of tasks for the next stage.

### Hunt Stage

The Hunt stage involves assigning tasks to agents, each responsible for a specific attack class paired with a scope hint.

### Adversarial Review Stage

The Adversarial Review stage involves adding a second agent to review the findings of the initial agent, reducing noise and improving the overall quality of the results.

### Splitting the Chain Stage

The Splitting the Chain stage involves splitting the chain of tasks across multiple agents, improving reasoning and reducing the complexity of the tasks.

Our Vulnerability Discovery Harness
------------------------------------

Our harness is designed to work in conjunction with Mythos Preview, leveraging its strengths to improve the overall efficiency and effectiveness of vulnerability research.

Example Harness
```markdown
# Recon Stage
agent: read_repository
input: repository_url
output: architecture_document, initial_queue

# Hunt Stage
agent: assign_tasks
input: initial_queue
output: task_assignments

# Adversarial Review Stage
agent: review_findings
input: task_assignments
output: reviewed_findings

# Splitting the Chain Stage
agent: split_chain
input: reviewed_findings
output: split_tasks
```
