# Enhancing SSH Security with Post-Quantum Key Exchange
=====================================================

GitHub is committed to ensuring the security of its users' data. As part of this effort, we are announcing the introduction of a post-quantum secure SSH key exchange algorithm to protect against future decryption attacks on quantum computers.

### What's Changing?

We are adding a new SSH key exchange algorithm, `sntrup761x25519-sha512` (also known as `sntrup761x25519-sha512@openssh.com`), to our SSH endpoints for accessing Git data. This change only affects SSH access and does not impact HTTPS access.

### Why Are We Making These Changes?

The new algorithm provides security against quantum computers by combining a post-quantum-secure algorithm, Streamlined NTRU Prime, with the classical Elliptic Curve Diffie-Hellman algorithm using the X25519 curve. This ensures that security won't be weaker than what the classical algorithm provides.

### What Is the Risk?

While today's key exchange algorithms are secure, an attacker could save encrypted sessions now and, if a suitable quantum computer is built in the future, decrypt them later. This is known as a "store now, decrypt later" attack.

### How Will This Affect You?

* If your Git remotes start with `https://`, you won't be impacted by this change.
* If your SSH client supports `sntrup761x25519-sha512@openssh.com` or `sntrup761x25519-sha512` (for example, OpenSSH 9.0 or newer), it will automatically choose the new algorithm by default if your client prefers it.
* If you use an older SSH client, your client should fall back to an older key exchange algorithm.

### How to Prepare

1. Check if your SSH client supports the new algorithm by running the command `ssh -Q kex`.
2. If you want to test which key exchange algorithm OpenSSH uses when connecting to GitHub.com, run the command `ssh -v git@github.com exit 2>&1 | grep 'kex: algorithm:'`.

### What's Next?

We'll continue to monitor the latest developments in security and update our offerings as new post-quantum algorithms become available, including those that comply with FIPS.

### Timeline

* September 17, 2025: The new algorithm will be enabled for GitHub.com and GitHub Enterprise Cloud with data residency (with the exception of the US region).
* GitHub Enterprise Server 3.19 will include this change.
