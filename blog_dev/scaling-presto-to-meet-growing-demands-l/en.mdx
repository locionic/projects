# Scaling Presto to Meet Growing Demands: Lessons Learned at Meta
===========================================================

Presto is a free, open-source SQL query engine that has been widely adopted at Meta for the past decade. As our usage of Presto grew, we encountered several challenges that required creative problem-solving to overcome. In this article, we will share four key lessons learned from scaling Presto to meet the growing demands of our organization.

### Scaling Presto Rapidly

One of the earliest challenges we faced was deploying new Presto releases to a large number of clusters while ensuring consistent availability and reliability. To address this, we implemented a load balancer called the Gateway, which routes Presto queries to the appropriate cluster. When a Presto cluster needs to be updated, it is first marked as drained from the Gateway, allowing existing queries to finish before the update is applied.

**Key Takeaway:** Implement a load balancer to route queries to available clusters and ensure consistent availability during updates.

### Automating Cluster Management

As our Presto footprint grew, we needed to automate the standup and decommissioning of clusters. We standardized our cluster configurations and implemented automation to generate configs from base templates. This allowed us to quickly spin up new clusters and integrate them with company-wide infrastructure services.

**Key Takeaway:** Automate cluster management to reduce manual errors and increase efficiency.

### Implementing Robust Load Balancing

As Presto usage increased, we encountered scalability issues with our load balancer. To address this, we implemented throttling and autoscaling features to prevent the Gateway from becoming overwhelmed. This ensured that the Gateway could withstand unexpected traffic patterns and maintain availability.

**Key Takeaway:** Implement robust load balancing with features like throttling and autoscaling to prevent scalability issues.

### Advice for Scaling Presto

Based on our experience, we recommend the following best practices for scaling Presto:

* Establish clear customer-facing SLAs to track important metrics like queueing time and query failure rate.
* Implement thorough monitoring and automated debugging to identify production issues early.
* Use good load balancing to distribute queries efficiently across clusters.
* Implement configuration management to simplify cluster configuration and reduce manual errors.

By following these best practices, you can ensure a successful Presto deployment that meets the growing demands of your organization.
