## Introduction

A fast website is not just a nice-to-have — it directly affects SEO rankings, user experience, and conversion rates. Next.js comes with many built-in performance features, but you need to know how to use them correctly.

In this post, I'll share the most impactful Next.js performance tips you can apply today.

---

## 1. Use the Next.js Image Component

Never use a plain `<img>` tag in Next.js. The built-in `Image` component automatically:

- Serves images in modern formats (WebP, AVIF)
- Resizes images for different screen sizes
- Lazy loads images by default
- Prevents layout shift with width/height
```tsx
// ❌ Bad
<img src="/hero.png" alt="Hero" />

// ✅ Good
import Image from 'next/image';

<Image
  src="/hero.png"
  alt="Hero"
  width={1200}
  height={600}
  priority  // add for above-the-fold images
/>
```

---

## 2. Use Static Generation Where Possible

Next.js App Router uses **React Server Components** by default, which render on the server and send HTML to the client. This is already fast, but static pages are even faster.
```tsx
// This page is statically generated at build time
export default function AboutPage() {
  return <h1>About Me</h1>;
}

// Force dynamic rendering only when needed
export const dynamic = 'force-dynamic';
```

Use static generation for pages that don't change often — home page, about, blog posts.

---

## 3. Optimize Fonts with next/font

Never load Google Fonts with a `<link>` tag. Use `next/font` instead — it downloads fonts at build time and serves them locally with zero layout shift:
```tsx
import { Inter } from 'next/font/google';

const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
});

export default function Layout({ children }) {
  return (
    <html className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

## 4. Code Split with Dynamic Imports

Large components that aren't needed on initial load should be dynamically imported:
```tsx
import dynamic from 'next/dynamic';

// Load component only when needed
const HeavyChart = dynamic(() => import('@/components/HeavyChart'), {
  loading: () => <p>Loading chart...</p>,
  ssr: false, // disable server rendering for client-only components
});
```

This reduces your initial bundle size and speeds up page load.

## 5. Add Proper Metadata

Good metadata improves SEO which brings more organic traffic — effectively improving your site's perceived performance from a user acquisition standpoint:
```tsx
// app/blog/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const post = await getPost(params.slug);
  return {
    title: post.title,
    description: post.summary,
    openGraph: {
      title: post.title,
      description: post.summary,
      images: [post.coverImage],
    },
  };
}
```

## 6. Use React Cache for Data Fetching

Avoid fetching the same data multiple times by using React's `cache` function:
```tsx
import { cache } from 'react';

export const getPost = cache(async (slug: string) => {
  const post = await db.posts.findOne({ slug });
  return post;
});

// Now calling getPost('my-post') multiple times
// only hits the database once per request
```

## 7. Minimize Client Components

In the App Router, components are server components by default. Only add `'use client'` when you actually need:

- `useState` or `useEffect`
- Browser APIs
- Event listeners
```tsx
// ✅ Server component — no 'use client' needed
export default async function BlogList() {
  const posts = await getPosts();
  return posts.map(post => <PostCard key={post.id} post={post} />);
}

// ✅ Client component — only for interactive parts
'use client';
export function LikeButton({ postId }) {
  const [liked, setLiked] = useState(false);
  return <button onClick={() => setLiked(!liked)}>Like</button>;
}
```

## 8. Analyze Your Bundle

Use the Next.js bundle analyzer to see what's making your app large:
```bash
npm install @next/bundle-analyzer
```
```js
// next.config.js
const withBundleAnalyzer = require('@next/bundle-analyzer')({
  enabled: process.env.ANALYZE === 'true',
});

module.exports = withBundleAnalyzer({});
```
```bash
ANALYZE=true npm run build
```

This opens a visual map of your bundle — find and eliminate large unnecessary dependencies.

---

## 9. Check Core Web Vitals

Use these free tools to measure your performance:

- **PageSpeed Insights** — pagespeed.web.dev
- **Vercel Analytics** — built into your Vercel dashboard
- **Chrome DevTools** → Lighthouse tab

Aim for:
- **LCP** (Largest Contentful Paint) under 2.5s
- **FID** (First Input Delay) under 100ms
- **CLS** (Cumulative Layout Shift) under 0.1

---

## Conclusion

Next.js gives you excellent performance out of the box, but small optimizations add up quickly. Start with these:

1. Use `next/image` for all images
2. Use static generation where possible
3. Load fonts with `next/font`
4. Dynamically import heavy components
5. Keep client components minimal

Apply these today and run a Lighthouse audit to see the improvements.
