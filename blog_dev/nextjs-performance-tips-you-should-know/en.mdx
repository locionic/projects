# Next.js Performance Optimization

A fast website directly boosts SEO, user experience, and conversions. Next.js has excellent built-in performance tools, but you must use them correctly.

<Callout type="idea" title="The Goal">
Aim for a Largest Contentful Paint (LCP) under 2.5s and Cumulative Layout Shift (CLS) near zero!
</Callout>

---

## Top Performance Tips

<Stepper>
  <Step title="1. Use the <Image> Component">
    Never use `<img>`. The Next.js `<Image>` component automatically serves WebP/AVIF, resizes dynamically, prevents layout shifts, and lazy-loads.
    ```tsx
    import Image from 'next/image';
    <Image src="/hero.png" alt="Hero" width={1200} height={600} priority />
    ```
  </Step>
  
  <Step title="2. Optimize Fonts with next/font">
    Avoid `<link>` tags for Google Fonts. Use `next/font` to download fonts at build time and serve them locally with zero layout shift.
  </Step>
  
  <Step title="3. Code Split with Dynamic Imports">
    Don't load heavy components immediately if they aren't visible on the initial load.
    ```tsx
    import dynamic from 'next/dynamic';
    const HeavyChart = dynamic(() => import('@/components/HeavyChart'), { ssr: false });
    ```
  </Step>

  <Step title="4. Minimize 'use client'">
    In the App Router, keep most components as Server Components. Only add `'use client'` when you absolutely need `useState`, `useEffect`, or browser APIs.
  </Step>
</Stepper>

## Caching and Analysis

<Callout type="info" title="React Cache">
Wrap your data fetching functions in React's `cache()` to prevent hitting your database multiple times during the same request cycle.
</Callout>

<Callout type="warning" title="Check Your Bundle">
Run `npm install @next/bundle-analyzer` to visually inspect what's bloating your app. Eliminate massive unnecessary libraries before deploying.
</Callout>
