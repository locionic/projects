I still remember the first time I wrote “Hello, world!” in Python and thought I was basically done with programming. Then I tried to do something real—read input, loop over it, make a tiny decision—and everything suddenly felt… messier. Not because Python is hard, but because “fundamentals” actually means a bunch of small pieces that have to click together.

The clean mental model is this: Python is a language for describing operations (math, text, data) and controlling what runs next (conditions and loops). Everything you build is just those two ideas stacked on repeat.

## Variables and basic types (the stuff you’ll touch constantly)
When you write:

```python
name = "Sam"
age = 17
price = 9.99
```

you’re creating variables—labels pointing at values. Python is dynamic, so you don’t declare types ahead of time. The tradeoff is simple: if you mix types without thinking, Python will break your code in subtle, annoying ways.

A few types you’ll bump into immediately:
- `int` for whole numbers  
- `float` for decimals  
- `str` for text  
- `bool` for True/False  

That “string vs number” thing trips people constantly. `"5" + "5"` becomes `"55"`, but `5 + 5` becomes `10`. Forget the conversion once, and you’ll get nonsense or a crash. It’s the most common early mistake.

## Input/output: printing is easy, formatting is where it gets real
Most beginners only use:

```python
print("Hello!")
```

Cool. Useful, but also kind of the warm-up lap.

The moment you accept user input, you’re dealing with strings all the time:

```python
name = input("Name: ")
age = int(input("Age: "))

print(f"{name} will be {age + 1} next year.")
```

That `int(...)` matters. `input()` always returns a string, even if the user types `17`. Converting early saves you from debugging weird type errors later.

## Operators and comparisons (this is what conditions run on)
Math is straightforward:

- `+ - * / %`
- `//` for floor division (`7 // 2 == 3`)
- `**` for powers (`2 ** 3 == 8`)

Then you get comparisons, which power decisions:

- `== != < <= > >=`

If you remember nothing else: use `==` for comparisons. Python won’t let you “almost” compare. It’ll stop you and point at the mistake, which is annoying—until it saves your time.

## Control flow: if statements and indentation (indentation is law)
Python’s control flow is easy, but indentation isn’t optional.

```python
temp = 30

if temp > 25:
    print("Hot")
else:
    print("Okay")
```

If you’re off by even a little indentation, Python changes what block belongs to what condition. No braces, no forgiveness—your editor and your habits matter.

You’ll also use `elif` when you have multiple cases:

```python
score = 82

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "C"

print(grade)
```

## Loops: repeating yourself without going insane
Loops are how you handle lists of things, repeated input, and “keep going until…” situations.

### `for` loops
Use when you know you’re iterating over a sequence:

```python
words = ["cat", "dog", "bird"]

for w in words:
    print(w)
```

### `while` loops
Use when you don’t know the number of iterations ahead of time:

```python
n = 3
while n > 0:
    print(n)
    n -= 1
```

If you mess up a `while`, you can accidentally make an infinite loop. Python won’t stop you. It’ll just keep running your code like you meant it.

## Lists: the first collection you should learn
Lists are where most beginners start living, because they’re the simplest way to store multiple values.

```python
nums = [1, 2, 3]
nums.append(4)
nums[0] = 10
```

Indexing matters. `nums[0]` is the first item, not the second.

And slicing is one of those features that feels like cheating when it clicks:

```python
nums = [10, 20, 30, 40, 50]
print(nums[1:4])  # [20, 30, 40]
```

You’ll use slicing constantly for “grab a chunk” logic.

## Functions: stop rewriting the same code
When you catch yourself repeating logic, you’re ready for functions. They take inputs, do something, and return results.

```python
def greet(name):
    return f"Hi, {name}!"

message = greet("Taylor")
print(message)
```

Mastering functions doesn’t just make code cleaner—it saves your sanity. You fix a bug once instead of hunting through five copies of the same logic.

## Errors: Python is strict, and you should learn to read it
Python gives you two big flavors of pain:
- syntax errors (your code can’t even run)
- exceptions (it runs, then crashes when something goes wrong)

Example:

```python
age = int("hello")
```

That triggers a `ValueError`. Learning to read error messages is *learning Python*. The first few times you’ll swear at the screen. After that, you’ll start using the traceback like directions.
