# Python Fundamentals: No Bullshit Guide

Everything you build in Python is just a combination of describing data (variables) and controlling what happens next (loops & conditions). Here are the foundational blocks.

<Callout type="idea" title="The Golden Rule of Python">
Indentation is law. Python uses whitespace (spaces/tabs) to define code blocks instead of `{}`. Get your indentation right, or nothing works.
</Callout>

---

## The Core Building Blocks

<Stepper>
  <Step title="1. Variables & Types">
    You don't declare types, but you must respect them. Mixing types blindly (like `"5" + 5`) is the #1 beginner mistake.
    - `int` (whole numbers), `float` (decimals)
    - `str` (text strings), `bool` (True/False)
  </Step>

  <Step title="2. Input & Formatting">
    `input()` always returns a string. If you need a number, you must convert it immediately using `int()` or `float()`.
    ```python
    age = int(input("Age: "))
    print(f"You will be {age + 1} next year.")
    ```
  </Step>

  <Step title="3. Control Flow (if/elif/else)">
    Make decisions using comparisons (`==`, `!=`, `<`, `>`). Always use `==` for checking equality!
    ```python
    if score >= 90:
        print("Grade: A")
    elif score >= 80:
        print("Grade: B")
    ```
  </Step>

  <Step title="4. Loops (for & while)">
    Repeat actions without going insane.
    - Use `for` when you know exactly how many times to loop (e.g., iterating a list).
    - Use `while` when looping until a condition changes. Watch out for infinite loops!
  </Step>
</Stepper>

---

## Collections & Reusability

<Callout type="info" title="Lists">
Lists (`[]`) are the easiest way to store multiple items. Remember that indexing starts at `0`.
```python
nums = [10, 20, 30]
print(nums[0])  # 10
```
</Callout>

<Callout type="info" title="Functions">
Stop rewriting the same code! Use `def` to create reusable blocks of logic.
```python
def greet(name):
    return f"Hi, {name}!"
```
</Callout>

<Callout type="warning" title="Embrace the Errors">
Python will yell at you with `SyntaxError` or `ValueError`. This is normal. Learning to read the Traceback is the real secret to mastering Python.
</Callout>
