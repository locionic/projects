Lần đầu tiên tôi viết code Python, tôi thấy mình như “bất khả chiến bại”. Tôi in ra “Hello, world!” rồi làm tiếp như thể tôi đã nắm vững ngôn ngữ này rồi.

Sau đó tôi thử nhập dữ liệu, lặp qua nó, và đưa ra một quyết định dù chỉ là đơn giản. Bỗng nhiên phần “cơ bản” không còn là lựa chọn nữa—nó là cả trò chơi. Không phải vì Python phức tạp, mà vì vài khái niệm nhỏ sẽ quyết định gần như tất cả những gì bạn làm sau đó.

Đây là mô hình tư duy mà tôi ước có ai đó đưa cho mình ngay từ ngày đầu: Python là ngôn ngữ để làm việc với các giá trị (chữ, số, danh sách) và quyết định điều gì sẽ xảy ra tiếp theo (điều kiện và vòng lặp). Hầu hết sự lẫn lộn của người mới đến từ việc không hiểu các mảnh này tương tác với nhau ra sao.

## Biến và các kiểu dữ liệu cơ bản (thứ bạn chạm liên tục)

Khi bạn viết:

```python
name = "Sam"
age = 17
price = 9.99
```

thì bạn đang tạo biến. Chúng chỉ là nhãn để gán cho giá trị. Python tự suy ra kiểu dữ liệu giúp bạn—tiện thật đấy—nhưng sẽ “dính” nếu bạn quên rằng `"17"` không giống `17`.

Bạn sẽ gặp những kiểu này liên tục:
- `int` cho số nguyên
- `float` cho số thập phân
- `str` cho chuỗi chữ
- `bool` cho True/False

Ví dụ nhanh để thấy vì sao kiểu dữ liệu quan trọng:

```python
print("5" + "5")  # "55"
print(5 + 5)      # 10
```

Cùng trông như một phép toán, kết quả lại hoàn toàn khác. Học để “đợi” điều đó, bạn sẽ tránh được rất nhiều lần ngồi gãi đầu vì không hiểu chuyện gì xảy ra.

## Nhập/xuất: `print()` thì dễ, `input()` cần được tôn trọng

In ra thì đơn giản:

```python
print("Hello!")
```

Nhưng ngay khi bạn dùng `input()`, bạn phải nhớ một quy tắc: `input()` luôn trả về một chuỗi ký tự.

```python
name = input("Name: ")
age = input("Age: ")

print(type(age))  # <class 'str'>
```

Nếu bạn cố biến input đó thành kiểu số mà không chuyển đổi, bạn sẽ gặp lỗi hoặc hành vi sai.

Khi bạn muốn làm việc kiểu số, hãy chuyển đổi đúng lúc:

```python
name = input("Name: ")
age = int(input("Age: "))

print(f"{name} will be {age + 1} next year.")
```

Cái `int(...)` này không phải nghi lễ rườm rà. Đó là cách bạn nói với Python: “Tôi kỳ vọng giá trị này sẽ là kiểu gì.”

## Toán tử và so sánh (đây là thứ để điều kiện chạy)

Toán tử dùng để tính toán, ghép lại và so sánh.

Phép toán số học:
- `+ - * /`
- `%` cho phần dư
- `//` cho phép chia lấy phần nguyên
- `**` cho lũy thừa

So sánh:
- `== != < <= > >=`

Đây là chỗ khác nhau giữa “trông có vẻ đúng” và “đúng thật”:

```python
temp = 30

if temp > 25:
    print("Hot")
else:
    print("Okay")
```

Python chỉ đưa ra quyết định dựa trên kết quả boolean của các phép so sánh như `temp > 25`.

## Luồng điều khiển: thụt lề là một phần của cú pháp

Nếu tôi chỉ được chọn một quy tắc trong phần này, thì đó phải là: thụt lề.

```python
temp = 30

if temp > 25:
    print("Hot")
else:
    print("Okay")
```

Những dấu cách đó không phải để cho đẹp. Chúng định nghĩa phần code nào thuộc về `if` và phần code nào thuộc về `else`.

Bạn cũng sẽ dùng `elif` khi cần nhiều điều kiện kiểm tra:

```python
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "D"
```

Viết các phép so sánh theo đúng thứ tự, Python sẽ làm đúng như ý bạn—không cần đoán mò.

## Vòng lặp: lặp lại mà không bị điên

Vòng lặp dùng để lặp lại công việc. Dùng `for` khi bạn duyệt qua một tập hợp. Dùng `while` khi bạn muốn tiếp tục cho đến khi điều gì đó thay đổi.

### Vòng lặp `for` (duyệt từng phần tử)

```python
words = ["cat", "dog", "bird"]

for w in words:
    print(w)
```

### Vòng lặp `while` (lặp đến khi điều kiện không còn đúng)

```python
n = 3

while n > 0:
    print(n)
    n -= 1
```

Cẩn thận với `while`: nếu điều kiện không bao giờ trở thành sai, Python sẽ cứ chạy mãi. Đây không phải “lỗi Python”—đó là “điều kiện vòng lặp của bạn không thay đổi”.

## Danh sách: kiểu collection mặc định bạn nên học trước

Nếu bạn mới với Python, danh sách là nơi bạn sẽ ở phần lớn thời gian.

```python
nums = [1, 2, 3]
nums.append(4)
nums[0] = 10
print(nums)  # [10, 2, 3, 4]
```

Bạn cũng sẽ cắt lát danh sách rất nhiều, vì nó cực kỳ tiện:

```python
nums = [10, 20, 30, 40, 50]
print(nums[1:4])  # [20, 30, 40]
```

Nếu bạn làm quen được với indexing (`nums[0]`) và slicing (`nums[1:4]`), bạn sẽ thấy tốc độ tiến lên nhanh ngay.

## Hàm số: đừng lặp lại logic

Hàm số giúp bạn đóng gói hành vi lặp lại thành một đơn vị có tên. Đây là cách để code không biến thành “mỳ spaghetti”.

```python
def greet(name):
    return f"Hi, {name}!"

msg = greet("Taylor")
print(msg)
```

Một thói quen nhỏ giúp ích: mỗi khi bạn thấy mình đang copy-paste logic, đó là dấu hiệu để tách ra thành một hàm.

## Lỗi: Python nghiêm túc, và bạn nên đọc nó

Lỗi của Python có hai nhóm lớn:

- Lỗi cú pháp (code không chạy được luôn)
- Exceptions (code chạy được, rồi đến lúc runtime thì vỡ)

Ví dụ như khi bạn cố chuyển đổi một chuỗi không phải số:

```python
age = int("hello")
```

Bạn sẽ nhận `ValueError`. Ban đầu có thể hơi bực, nhưng một khi bạn học cách đọc message lỗi, bạn sẽ thôi đoán mò và bắt đầu sửa thật sự.
