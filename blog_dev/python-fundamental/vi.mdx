# Nền Tảng Python: Ngắn Gọn & Thực Dụng

Mọi chương trình Python bạn viết đều xoay quanh việc quản lý dữ liệu (biến) và quyết định luồng chạy (vòng lặp & điều kiện). Dưới đây là những "viên gạch" cốt lõi nhất.

<Callout type="idea" title="Luật Bất Thành Văn Của Python">
Thụt lề (Indentation) là bắt buộc. Thay vì dùng ngoặc nhọn `{}`, Python dùng khoảng trắng (spaces/tabs) để xác định khối code. Bạn thụt lề sai, code sẽ lỗi.
</Callout>

---

## 4 Khối Xây Dựng Cơ Bản

<Stepper>
  <Step title="1. Biến & Kiểu dữ liệu">
    Bạn không cần khai báo kiểu (type), nhưng phải tôn trọng chúng. Ghép bừa bãi (như `"5" + 5`) là lỗi thường gặp nhất.
    - `int` (số nguyên), `float` (số thập phân)
    - `str` (chuỗi), `bool` (True/False)
  </Step>

  <Step title="2. Nhập dữ liệu (Input)">
    `input()` luôn trả về chuỗi (string). Nếu cần tính toán số học, phải ép kiểu ngay bằng `int()` hoặc `float()`.
    ```python
    age = int(input("Tuổi của bạn: "))
    print(f"Năm sau bạn sẽ {age + 1} tuổi.")
    ```
  </Step>

  <Step title="3. Rẽ nhánh (if/elif/else)">
    Dùng các phép so sánh (`==`, `<`, `>`) để ra quyết định. Nhớ dùng `==` (2 dấu bằng) khi muốn kiểm tra xem 2 giá trị có bằng nhau không!
    ```python
    if score >= 90:
        print("Điểm: A")
    elif score >= 80:
        print("Điểm: B")
    ```
  </Step>

  <Step title="4. Vòng lặp (for & while)">
    - Dùng `for` khi biết trước số lần lặp (vd: duyệt qua danh sách).
    - Dùng `while` khi lặp cho đến khi thỏa mãn một điều kiện. Cẩn thận vòng lặp vô hạn!
  </Step>
</Stepper>

## Gom Nhóm & Tái Sử Dụng

<Callout type="info" title="Danh sách (Lists)">
List (`[]`) là cách dễ nhất để chứa nhiều thứ. Nhớ rằng phần tử đầu tiên nằm ở vị trí `0`.
```python
nums = [10, 20, 30]
print(nums[0])  # In ra 10
```
</Callout>

<Callout type="info" title="Hàm (Functions)">
Đừng copy-paste code mãi! Dùng `def` để gom đoạn logic dùng chung lại thành một hàm.
```python
def greet(name):
    return f"Chào {name}!"
```
</Callout>

<Callout type="warning" title="Đối Mặt Với Lỗi">
Python sẽ phàn nàn bằng `SyntaxError` hoặc `ValueError`. Chuyện bình thường! Học cách đọc Traceback chính là chìa khóa để làm chủ Python.
</Callout>
