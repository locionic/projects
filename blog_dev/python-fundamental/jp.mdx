Pythonで初めてコードを書いたとき、「無敵だ」と感じました。`"Hello, world!"`を表示して、そのまま次へ。もう言語をマスターしたも同然だと思っていたんです。

でも、次に入力を読んで、それをループで回して、さらに簡単な判断まで入れようとした瞬間に、事態は変わりました。いわゆる「基礎」の部分が“オプション”じゃなくなったんです。基礎こそが、ゲームそのもの。Pythonが難しいからじゃありません。ちょっとした概念が、あなたがやること全部を支配しているからです。

そこで、1日目に誰かが渡してくれてたらよかった“頭の地図”がこれです。Pythonは、値（文字列、数値、リストなど）を扱うための言語で、次に何が起こるかを決めます（条件分岐とループ）。初心者がつまずく主な原因は、この部品同士がどう連動するかをちゃんと意識できていないことにあります。

## 変数と基本の型（ずっと触るやつ）

たとえば、こんなふうに書くとします。

```python
name = "Sam"
age = 17
price = 9.99
```

これは変数を作っています。値に対する“ラベル”みたいなものです。Pythonは型を自動で判断してくれるので便利なんですが、「"17"」は`17`とは別物だ、と忘れると一気にやられます。

頻繁に出会うのはだいたいこのあたり：

- `int`：整数
- `float`：小数
- `str`：文字列
- `bool`：True/False

型がなぜ大事か、サクッと例を見ましょう。

```python
print("5" + "5")  # "55"
print(5 + 5)      # 10
```

見た目は同じ操作に見えるのに、結果はまったく別。こうなることを最初から“想定できる”ようになれば、無駄に首をかしげる時間がかなり減ります。

## 入出力：`print()`は簡単、`input()`は敬意が必要

表示は簡単です。

```python
print("Hello!")
```

でも`input()`を使った瞬間、1つだけルールを思い出してください。`input()`は“必ず文字列を返す”ということです。

```python
name = input("Name: ")
age = input("Age: ")

print(type(age))  # <class 'str'>
```

入力を数値として扱おうとすると、エラーになったり、意図と違う動きになったりします。数値の計算をしたいときは、変換してください。

```python
name = input("Name: ")
age = int(input("Age: "))

print(f"{name} will be {age + 1} next year.")
```

この`int(...)`は“儀式”じゃありません。Pythonに「この値はこういう種類として扱ってほしい」と伝えているだけです。

## 演算子と比較（条件分岐が走るのはここ）

演算子は、計算したり、組み合わせたり、比較したりするためのものです。

算術：

- `+ - * /`
- `%`：余り
- `//`：床（切り捨て）除算
- `**`：べき乗

比較：

- `== != < <= > >=`

「それっぽい」から「ちゃんと正しい」へ切り替わる瞬間を見せます。

```python
temp = 30

if temp > 25:
    print("Hot")
else:
    print("Okay")
```

Pythonは判断を、こうした比較の“真偽値（boolean）”に基づいて行います。つまり`temp > 25`がTrueかどうかだけで決まる、ということです。

## 制御フロー：インデントが構文の一部

このセクションで1つだけ守るなら、それはインデントです。

```python
temp = 30

if temp > 25:
    print("Hot")
else:
    print("Okay")
```

このスペースは見栄えのための飾りじゃありません。`if`に属するコード、`else`に属するコードを区切っているんです。

複数条件なら`elif`も使います。

```python
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "D"
```

比較を書く順番をちゃんと揃えれば、Pythonはあなたの意図どおりに動きます。変な推測は不要です。

## ループ：同じことを繰り返して、頭をおかしくしない

ループは繰り返し処理を担当します。コレクションを順番に回したいなら`for`。何かが変わるまで（または条件が崩れるまで）続けたいなら`while`です。

### `for`ループ（アイテムを反復する）

```python
words = ["cat", "dog", "bird"]

for w in words:
    print(w)
```

### `while`ループ（条件が成り立たなくなるまで回す）

```python
n = 3

while n > 0:
    print(n)
    n -= 1
```

`while`は注意が必要です。条件が一度もFalseにならないなら、Pythonは当然のように永遠に回り続けます。これはPythonの問題じゃありません。「ループ条件が変わらない」あなたの問題です。

## リスト：最初に覚えるべきデフォルトのコレクション

Python初心者なら、まず生活する場所がリストです。

```python
nums = [1, 2, 3]
nums.append(4)
nums[0] = 10
print(nums)  # [10, 2, 3, 4]
```

それから、リストのスライスもめちゃくちゃ使います。便利すぎるので。

```python
nums = [10, 20, 30, 40, 50]
print(nums[1:4])  # [20, 30, 40]
```

インデックス指定（`nums[0]`）とスライス（`nums[1:4]`）に慣れたら、すぐに速度が上がります。

## 関数：同じロジックを何度も書くのをやめよう

関数は、繰り返し使う振る舞いを“名前付きのひとまとめ”にします。コードがカオスになってスパゲッティ化するのを防ぐ方法でもあります。

```python
def greet(name):
    return f"Hi, {name}!"

msg = greet("Taylor")
print(msg)
```

身につけると強い習慣が1つあります。自分でロジックをコピペしているのに気づいたら、それは関数に切り出せるサインです。

## エラー：Pythonは厳しい。そして書いてあることを読もう

Pythonのエラーには大きく分けて2種類あります。

- 構文エラー（コードがそもそも動かない）
- 例外（実行は進むけど、実行時に失敗する）

たとえば、数値にならない文字列を変換しようとすると：

```python
age = int("hello")
```

`ValueError`になります。最初はイラっとします。でも、エラーメッセージの読み方を覚えると、当てずっぽうで悩むのをやめて、ちゃんと直せるようになります。

## ちょっと現実チェック

ここまでを気持ちよくできるなら——変数に値を保存して、`if/elif/else`を使って、`for`と`while`でループして、リストを操作して、関数を定義して、基本的なエラーを読み解けるなら——それは「Pythonの基礎を学んでいるだけ」じゃありません。もう“本物のスクリプト”を作っている状態です。

もしあなたの今のレベル（まったくの初心者か、すでに小さなスクリプトを書いているか）と、目標（自動化／Web／データ／面接対策など）を教えてくれたら、その目標に合う短い基礎練習プランと、小さなエクササイズを提案できます。
