# Error Handling & Testing

Robust code handles errors gracefully and verifies behavior automatically. Python's exception system is more nuanced than just try/catch.

## Exception Hierarchy

```python
# Specific exceptions first, general last
try:
    result = risky_operation()
except ValueError as e:
    print(f"Bad value: {e}")
except (TypeError, KeyError) as e:
    print(f"Type or key error: {e}")
except Exception as e:
    print(f"Unexpected: {e}")
else:
    # Runs if NO exception occurred
    print(f"Success: {result}")
finally:
    # Always runs — cleanup
    cleanup()
```

## Custom Exceptions

```python
class ConfigError(Exception):
    """Raised when configuration is invalid."""
    pass

class ConfigNotFoundError(ConfigError):
    """Specific config error."""
    pass
```

## Testing with pytest

```python
import pytest

def test_addition():
    assert add(2, 3) == 5

# Parametrized tests
@pytest.mark.parametrize("a,b,expected", [
    (1, 1, 2), (0, 0, 0), (-1, 1, 0)
])
def test_add_params(a, b, expected):
    assert add(a, b) == expected

# Fixtures
@pytest.fixture
def db_connection():
    conn = create_test_db()
    yield conn  # teardown after test
    conn.close()
```

## Key Takeaways

- Use specific exception types, not bare `except:`
- `else` in try/except is underused — it clarifies intent
- pytest parametrization reduces test boilerplate
