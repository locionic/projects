# Variables, Types, and Control Flow

Python's type system is **dynamic** but not **weak**. Understanding the distinction — and how to use type hints effectively — is the foundation of writing Python that scales from scripts to systems.

## Variables Are References

In Python, variables are **names bound to objects**. This isn't an academic distinction — it affects how your code behaves.

```python filename="variables.py"
# Python doesn't "store" values in variables
# It binds names to objects in memory
a = [1, 2, 3]
b = a           # b references the SAME list
b.append(4)     # modifies both a and b!
print(a)        # [1, 2, 3, 4]

# To copy: use explicit copies
c = a.copy()    # shallow copy
d = a[:]        # also a shallow copy
```

<Callout type="warning" title="Common Pitfall">
Mutating a list you received as a function argument modifies the original. Always copy inputs you plan to mutate.
</Callout>

## Type Hints (Python 3.6+)

Type hints are **optional** but **strongly encouraged** for codebases larger than a single script.

```python filename="type_hints.py"
from typing import Optional, Union, List, Dict, Any

# Basic function signature
def greet(name: str, age: int = 0) -> str:
    return f"{name} is {age} years old"

# Optional vs Union
def find_user(user_id: int) -> Optional[dict]:
    # Optional[dict] means dict | None
    return db.lookup(user_id) or None

# Union for multiple types
def process(value: Union[str, int]) -> str:
    return str(value)

# Python 3.10+ cleaner syntax
def process_v2(value: str | int) -> str:
    return str(value)
```

<Flashcard term="Type hints are enforced at runtime" definition="False! Type hints are ignored by the Python interpreter. They're used by tools like mypy, pyright, and your IDE for static analysis." category="Python Types" />

<Flashcard term="Optional[X] vs X | None" definition="They're equivalent. Optional[X] is the older syntax (Python 3.6+), X | None is the newer syntax (Python 3.10+). Both mean 'X or None'." category="Python Types" />

## Control Flow Idioms

Python has a handful of control flow patterns that differ from C-like languages.

```python filename="control_flow.py"
# For-else: runs 'else' if loop didn't break
for item in collection:
    if matches(item):
        print("Found!")
        break
else:
    print("Not found — ran through entire loop")

# Walrus operator: assign and test in one expression
if (n := len(items)) > 10:
    print(f"Too many items: {n}")

# Match-case (Python 3.10+)
match status_code:
    case 200:
        print("OK")
    case 404:
        print("Not found")
    case _:
        print(f"Unknown: {status_code}")
```

## Key Takeaways

- Python variables are **references** to objects, not buckets
- Always use **type hints** in production code
- Learn the Python-specific idioms: for-else, walrus, match-case
- Copy before mutating inputs you don't own
