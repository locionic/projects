# Data Structures Deep Dive

Choosing the right data structure is the single most impactful optimization you can make. Python's built-in types are implemented in C and highly optimized — use them correctly.

## Time Complexity Reference

| Structure | Access | Search | Insert | Delete | Memory |
|-----------|--------|--------|--------|--------|--------|
| List | O(1) | O(n) | O(n) | O(n) | Low |
| Tuple | O(1) | O(n) | — | — | Lowest |
| Set | — | O(1) | O(1) | O(1) | High |
| Dict | O(1) | O(1) | O(1) | O(1) | High |
| Deque | O(1) | O(n) | O(1) | O(1) | Medium |

## When to Use What

```python
# List — ordered, indexed, allows duplicates
users = ["Alice", "Bob", "Charlie"]

# Set — unique items, fast membership testing
valid_ids = {101, 202, 303}
if user_id in valid_ids:  # O(1)
    print("Found!")

# Dict — key-value mappings, fast lookups
user_roles = {"alice": "admin", "bob": "user"}
role = user_roles.get("charlie", "guest")  # safe default
```

<Callout type="info" title="List vs Tuple Trade-off">
Tuples use less memory than lists and can be used as dict keys (they're hashable). Use tuples for fixed records, lists for variable-length collections.
</Callout>

## collections Module

```python
from collections import defaultdict, Counter, deque, OrderedDict

# defaultdict — auto-initialize missing keys
tags = defaultdict(int)
tags["python"] += 1  # no KeyError!

# Counter — count anything
word_counts = Counter("hello world hello".split())
print(word_counts.most_common(1))  # [("hello", 2)]

# deque — fast appends/pops from both ends
queue = deque([1, 2, 3])
queue.appendleft(0)
queue.pop()  # 3
```

<Flashcard term="dict keys must be hashable" definition="Lists, dicts, and sets are unhashable (mutable). Tuples, strings, numbers, and frozensets are hashable. Use frozenset if you need a set as a dict key." category="Python Data Structures" />

## Key Takeaways

- Sets and dicts give O(1) lookups — use them over lists for membership
- `collections` module has optimized variants for common patterns
- Tuples are lighter and hashable — prefer them for fixed records
