# Dataclasses, Pydantic & Attrs

Python 3.7+ `dataclasses` eliminate boilerplate for data containers. For validation, Pydantic adds runtime type checking.

## Basic Dataclass

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    label: str = ""

# Auto-generates __init__, __repr__, __eq__
p = Point(1.0, 2.0, "origin")
print(p)  # Point(x=1.0, y=2.0, label='origin')
```

## Advanced Dataclass Features

```python
from dataclasses import dataclass, field

@dataclass(frozen=True, order=True, slots=True)
class Config:
    host: str
    port: int = field(default=8080, metadata={"env": "APP_PORT"})
    tags: list[str] = field(default_factory=list)

# frozen=True → immutable (hashable)
# order=True → implements comparison operators
# slots=True → reduces memory (Python 3.10+)
```

## Pydantic for Validation

```python
from pydantic import BaseModel, Field, EmailStr

class User(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    email: EmailStr
    age: int = Field(ge=0, le=150)

user = User(name="Alice", email="alice@example.com", age=30)  # validates!
```

## Key Takeaways

- Dataclasses replace 80% of manual class boilerplate
- `frozen=True` + `slots=True` for immutable data objects
- Pydantic for any data entering your system (APIs, config files)
- Attrs is the precursor library — use dataclasses unless you need attrs-specific features
