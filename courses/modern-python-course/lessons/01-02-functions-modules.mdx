# Functions & Modules

Python functions are first-class objects with some unique behavior around argument passing that trips up engineers coming from other languages.

## Pass-by-Object-Reference

Understanding how Python passes arguments is critical:

```python
def append_to(element, target=[]):  # BAD: mutable default
    target.append(element)
    return target

# The default [] is created ONCE at function definition
print(append_to(1))  # [1]
print(append_to(2))  # [1, 2]  — BUG!
```

<Callout type="danger" title="Never use mutable defaults">
Default arguments are evaluated once at function definition time. Use `None` as sentinel instead:
```python
def append_to(element, target=None):
    if target is None:
        target = []
    target.append(element)
    return target
```
</Callout>

## Type-Annotated Functions

```python
from typing import List, Optional, Callable, TypeVar

T = TypeVar('T')

def first(items: List[T], default: Optional[T] = None) -> Optional[T]:
    """Return first item or default if empty."""
    return items[0] if items else default

# Callable type for higher-order functions
def apply_twice(fn: Callable[[int], int], value: int) -> int:
    return fn(fn(value))
```

<Flashcard term="Mutable default arguments" definition="Are evaluated once at function definition time, not each call. Always use None as default for mutable parameters, then reassign inside the function." category="Python Functions" />

<Flashcard term="*args and **kwargs" definition="*args collects extra positional args as a tuple. **kwargs collects extra keyword args as a dict. Use for wrapper/decorator functions." category="Python Functions" />

## Modules and Imports

```python
# my_module.py
__all__ = ['public_func']  # controls `from module import *`

def public_func():
    pass

def _internal():  # underscore = internal convention
    pass
```

## Key Takeaways

- Mutable defaults are evaluated once — use `None` instead
- Type annotations improve readability and tooling support
- Use `_` prefix for internal module members
