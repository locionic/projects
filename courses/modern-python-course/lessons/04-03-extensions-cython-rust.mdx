# C Extensions, Cython & Rust

When you've profiled and optimized everything, but it's still too slow: write a native extension.

## Options Overview

| Approach | Speedup | Complexity | Setup |
|----------|---------|------------|-------|
| Cython | 2-10x | Medium | Compile .pyx |
| C API | 10-100x | Hard | CPython API |
| Rust (maturin) | 10-100x | Medium | pyproject.toml |

## Cython Example

```python
# sum_cy.pyx
def fast_sum(int n):
    cdef int i
    cdef int total = 0
    for i in range(n):
        total += i
    return total
```

## Rust with Maturin

```rust
// src/lib.rs
use pyo3::prelude::*;

#[pyfunction]
fn fast_sum(n: u64) -> u64 {
    (0..n).sum()
}

#[pymodule]
fn my_extension(_py: Python, m: &PyModule) -> PyResult<()> {
    m.add_function(wrap_pyfunction!(fast_sum, m)?)?;
    Ok(())
}
```

<Flashcard term="When to write a native extension" definition="Only after profiling confirms Python is the bottleneck. The first question should always be 'can I use NumPy?' If yes, use NumPy. If no, consider Cython for moderate speedup, Rust/maturin for maximum performance with good ergonomics." category="Python Performance" />

## Key Takeaways

- Always profile first — you may not need native code
- NumPy already implements many common patterns in C
- Cython for moderate speedups with minimal changes
- Rust via maturin for maximum performance and safety
- Each extension adds build complexity — measure the benefit
