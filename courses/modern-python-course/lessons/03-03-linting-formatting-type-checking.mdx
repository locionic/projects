# Linting, Formatting & Type Checking

Modern Python projects use automated tools to enforce code quality. **ruff** replaces isort, flake8, and black. **mypy** catches type errors before runtime.

## Ruff Configuration

```toml
# pyproject.toml
[tool.ruff]
line-length = 100
target-version = "py311"

[tool.ruff.lint]
select = ["E", "F", "I", "N", "W", "UP"]
ignore = ["E501"]

[tool.ruff.format]
quote-style = "double"
```

## Mypy for Type Safety

```toml
[tool.mypy]
strict = true
ignore_missing_imports = true
```

## Pre-commit

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.4.0
    hooks:
      - id: ruff
      - id: ruff-format
  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.9.0
    hooks:
      - id: mypy
```

## Key Takeaways

- Ruff replaces flake8 + isort + black — one dependency, one config
- Mypy in strict mode catches real bugs before they reach production
- Pre-commit hooks make quality checks automatic
- CI should run the same checks as pre-commit
