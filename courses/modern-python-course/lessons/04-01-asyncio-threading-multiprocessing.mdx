# asyncio, Threading & Multiprocessing

Python offers three concurrency models. Choosing the right one is the difference between 10x speedup and 10x headache.

## When to Use What

| Model | Best For | Limit | Analogy |
|-------|----------|-------|---------|
| asyncio | I/O-bound, high concurrency | Single thread | 1 chef, 1000 orders |
| Threading | I/O-bound, moderate | GIL | 10 chefs, 1000 orders |
| Multiprocessing | CPU-bound | Memory | 10 chefs, 10 kitchens |

## asyncio Example

```python
import asyncio
import aiohttp

async def fetch(url: str) -> str:
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

async def main():
    urls = ["https://example.com"] * 10
    tasks = [fetch(url) for url in urls]
    results = await asyncio.gather(*tasks)

asyncio.run(main())
```

<Callout type="warning" title="asyncio gotchas">
- Blocking code (time.sleep, requests.get) blocks the entire event loop
- Use asyncio.sleep(), aiohttp for async HTTP
- CPU work inside a coroutine blocks the loop — move it to executor
</Callout>

## Threading vs Multiprocessing

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
import time

def io_work(n):
    time.sleep(0.1)  # Simulate I/O
    return n * 2

def cpu_work(n):
    return sum(range(n))  # CPU-intensive

# I/O-bound → threads
with ThreadPoolExecutor(max_workers=10) as ex:
    results = list(ex.map(io_work, range(20)))

# CPU-bound → processes
with ProcessPoolExecutor(max_workers=4) as ex:
    results = list(ex.map(cpu_work, [10_000_000] * 4))
```

## Key Takeaways

- asyncio for high-concurrency I/O-bound work
- Threading for moderate I/O-bound work (simpler mental model)
- Multiprocessing for CPU-bound work only
- Never mix asyncio and blocking code
