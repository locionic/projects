# Profiling & Optimization

Don't guess — profile. Python has excellent tooling for finding real bottlenecks.

## Profiling with cProfile

```python
import cProfile
import pstats

def slow_function():
    total = 0
    for i in range(10_000_000):
        total += i
    return total

cProfile.run('slow_function()', 'profile_stats')
p = pstats.Stats('profile_stats')
p.sort_stats('cumulative').print_stats(10)
```

## Common Optimizations

```python
# 1. Use local variable bindings
def slow(items):
    import math  # BAD: import inside loop
    return [math.sqrt(i) for i in items]

# 2. List comprehension over loops
squares = [i**2 for i in range(1000)]  # faster than explicit for-loop

# 3. Set membership over list
valid = {1, 2, 3}  # O(1) vs O(n) for list
if x in valid: ...

# 4. Cache expensive computations
from functools import lru_cache

@lru_cache(maxsize=128)
def expensive(n):
    return sum(range(n))
```

## Key Takeaways

- Always profile before optimizing
- cProfile shows function-level timing
- `lru_cache` is the simplest optimization for pure functions
- Local variable lookups are faster than global/attribute lookups
- Profile again after each optimization to verify improvement
