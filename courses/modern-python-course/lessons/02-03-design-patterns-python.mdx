# Design Patterns in Python

Gang of Four patterns often need less ceremony in Python. Some become one-liners; others are anti-patterns.

## Singleton — Use a Module

```python
# singleton.py — modules are singletons in Python
_cache = {}

def get(key: str):
    return _cache.get(key)

def set(key: str, value):
    _cache[key] = value
```

## Factory — Use a function or dict

```python
from typing import Callable

class Circle: ...
class Square: ...

SHAPES: dict[str, Callable] = {
    "circle": Circle,
    "square": Square,
}

def create_shape(kind: str):
    if kind not in SHAPES:
        raise ValueError(f"Unknown shape: {kind}")
    return SHAPES[kind]()
```

## Strategy — Use a function

```python
from typing import Callable

# Strategies are just callables
def quick_sort(data: list) -> list: ...
def merge_sort(data: list) -> list: ...

def sort_data(data: list, strategy: Callable[[list], list]) -> list:
    return strategy(data)

sort_data([3, 1, 2], quick_sort)
```

## Key Takeaways

- Modules replace Singletons — no class needed
- Functions replace Strategy interfaces — pass them directly
- Dicts replace Factory method hierarchies — use string mapping
- The simplest Python solution is often the most Pythonic
